\section{计算图上的反向传播与自动微分}

手推一个复杂模型时，最危险的做法是把整个表达式一次性展开。计算图把模型拆成基本运算，每个节点只需要知道自己的前向值和局部 VJP；反向传播按图的反方向累积敏感度。

考虑一个两层分类网络，batch 大小为 \(n\)：

\[
X\in\R^{n\times d},
\quad
W_1\in\R^{d\times h},
\quad
b_1\in\R^h,
\]

\[
W_2\in\R^{h\times K},
\quad
b_2\in\R^K.
\]

前向计算为

\[
Z_1=XW_1+\mathbf1b_1^T,
\]

\[
H=\phi(Z_1),
\]

\[
Z_2=HW_2+\mathbf1b_2^T,
\]

\[
L=\operatorname{CrossEntropy}(Z_2,Y).
\]

\subsection{反向从标量种子一开始}

损失是标量，所以反向种子是

\[
\bar L=1.
\]

softmax-cross-entropy 融合节点直接给出上一节的批量梯度

\[
\bar Z_2 = \frac1n(P-Y).
\]
经过第二个仿射层：

\[
\bar W_2 = H^T\bar Z_2,
\]
\[
\bar b_2 = \bar Z_2^T\mathbf1,
\]
\[
\bar H = \bar Z_2W_2^T.
\]
若激活函数逐元素作用，

\[
\bar Z_1 = \bar H\odot\phi'(Z_1).
\]
最后：

\[
\bar W_1 = X^T\bar Z_1,
\]
\[
\bar b_1 = \bar Z_1^T\mathbf1,
\]
\[
\bar X = \bar Z_1W_1^T.
\]
每个式子都只是局部线性映射的伴随。矩阵乘法前向为 \(AB\)，反向对 \(A\) 乘上右因子转置，对 \(B\) 乘上左因子转置；逐元素运算前向逐元素，反向也逐元素；广播前向复制，反向沿复制轴求和。

\subsection{自动微分不是数值微分}

有限差分通过多次重新运行程序估计斜率，会同时受到截断与舍入误差影响。自动微分把程序分解为已知基本算子，按链式法则组合它们的精确局部导数；没有差分步长。

它也不同于符号微分。符号系统尝试生成一个新的解析表达式，可能出现表达式膨胀；自动微分在实际执行轨迹上保存数值与局部运算，以计算 JVP 或 VJP。

``精确''仍指相对于所执行浮点程序和基本算子规则。前向若溢出、使用近似求解或经过不可微分支，反向会继承这些性质。

\subsection{正向模式与反向模式由维度决定}

对

\[
f:\R^n\to\R^m,
\]

正向模式一次传播一个输入方向，得到 \(Jv\)；反向模式一次传播一个输出余向量，得到 \(J^Tu\)。

\begin{itemize}
\tightlist
\item
  \(n\) 小、\(m\) 大时，正向模式常更经济；
\item
  \(n\) 大、\(m=1\) 时，反向模式一次得到全部梯度；
\item
  完整 Jacobian 需要多次模式调用；
\item
  Hessian-vector product 可由正反模式组合。
\end{itemize}

神经网络通常有大量参数和一个标量损失，因此 reverse mode 成为默认；这不是说反向模式对任何函数都更快。

\subsection{内存来自反向需要前向信息}

许多局部 VJP 需要前向中间值：\(\tanh\) 的反向需要输出或输入，矩阵乘法反向需要另一因子，softmax-cross-entropy 需要概率或 log-normalizer。因此反向模式必须保存计算图中的 activations，或在反向时重算它们。

Checkpointing 只保存部分节点，在反向阶段重算中间段，以额外计算换内存。可逆网络、算子融合和 custom VJP 也在调整同一取舍。

原地修改可能覆盖反向需要的值；动态控制流则要求系统记录实际走过的分支。自动微分框架的错误往往出在状态、别名、广播或自定义算子的语义上，链式法则本身反而很少是问题。

\subsection{为什么仍要做手推与梯度检查}

手推提供形状、符号和可简化结构。例如 softmax-cross-entropy 若作为两个独立节点机械反传，会产生不必要的中间量；知道它们合并为 \(P-Y\) 可以实现更稳定的融合算子。

梯度检查则提供独立证据。对小型、确定、光滑的测试输入，可比较自动微分方向导数

\[
v^T\nabla f(x)
\]

与中心差分

\[
\frac{f(x+hv)-f(x-hv)}{2h}.
\]

应使用较高精度、多个步长和多个方向，关闭 dropout 等随机状态，并避开 ReLU 零点与 max 并列点。检查通过不能证明所有输入正确，但检查无法发现故意注入的转置或广播错误，就说明测试没有判断力。

\subsection{反向传播的边界}

\begin{itemize}
\tightlist
\item
  ReLU 在零点没有唯一导数，框架使用约定值；
\item
  argmax、采样和离散索引通常不可直接求导；
\item
  截断迭代求解器的反向可以选择``对执行的有限步求导''或``对收敛方程隐式求导''，二者不是同一对象；
\item
  混合精度与非确定归约会影响梯度复现；
\item
  对复数、稀疏和受约束矩阵，还需额外的导数语义。
\end{itemize}

自动微分免去了重复手算，却没有免除对函数定义、形状、数值稳定性和可微性的理解。
