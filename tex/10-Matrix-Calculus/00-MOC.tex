\section{矩阵微积分：把复杂求导还原为线性传播}

机器学习和统计中的目标常写成紧凑的矩阵式：

\[
L(W) = \frac12\lVert XW-Y\rVert_F^2,
\]
\[
\ell(\Sigma) = \frac12\log\det\Sigma
+\frac12(x-\mu)^T\Sigma^{-1}(x-\mu).
\]
公式看起来比逐元素求和更清楚，一旦开始求导，却容易出现转置、布局和高阶张量混乱。查公式表能暂时得到答案，但模型稍微改变，记忆就失效；不同资料采用不同 Jacobian 约定时，甚至会出现看似矛盾的结果。

矩阵微积分真正需要的不是更多公式，而是一个稳定的推导原则：

\begin{quote}
导数是输入扰动到输出一阶变化之间的线性映射。
\end{quote}

Jacobian 是这条线性映射在坐标中的矩阵表示；梯度是使用内积把标量输出的线性泛函表示成与输入同形的向量或矩阵；Hessian 则描述梯度如何随方向改变。只要先抓住映射、扰动和形状，转置就不再靠猜。

\subsection{三个比公式更优先的问题}

每次求导前先问：

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  \textbf{谁是变量？} 同一个表达式对 \(x\)、\(A\) 或 \(B\) 求导会得到完全不同的线性映射。必须明确哪些量固定。
\item
  \textbf{输入和输出的空间是什么？} 向量、一般矩阵、对称矩阵或概率单纯形具有不同的允许扰动；结果形状应与映射方向一致。
\item
  \textbf{真正需要完整导数吗？} 大多数算法只需要 Jacobian-vector product、vector-Jacobian product 或 Hessian-vector product，无需构造巨大 Jacobian、Hessian或四阶张量。
\end{enumerate}

这三个问题往往能在计算前排除一半错误。

\subsection{从导数本体到记号约定}

导数的对象与记号先从 Fréchet 导数定义最佳线性近似：

\[
f(x+h)=f(x)+Df(x)[h]+o(\lVert h\rVert).
\]

本知识库采用

\[
dy=J_f(x)\,dx
\]

的 Jacobian 布局，即输出分量按行、输入分量按列。对标量函数，梯度保留为与输入同形的列向量，并满足

\[
df=\nabla f(x)^Tdx.
\]

其他资料若把 Jacobian 整体转置，数学映射没有变化，只是矩阵表示不同。阅读时应先翻译约定，再比较公式。

这一单元还会说明梯度依赖所选内积。欧氏梯度、预条件梯度和自然梯度可以表示同一个导数泛函，却使用不同几何把它转成向量。这让``梯度是什么''从偏导列表上升为线性代数问题。

\subsection{微分把复杂表达式拆成局部规则}

微分、链式法则与二阶信息给每个中间量加入扰动，并只保留一阶项：

\[
d(AB)=(dA)B+A(dB).
\]

对标量矩阵函数，Frobenius 内积定义

\[
df = \langle\nabla_Xf,dX\rangle_F
=\operatorname{tr}\bigl((\nabla_Xf)^TdX\bigr).
\]
推导的工作就是把所有依赖展开，再把 \(dX\) 整理到这个形式中。

链式法则则是线性映射复合。正向传播 JVP，把输入方向逐层推向输出；反向传播 VJP，把输出敏感度通过伴随映射逐层拉回。自动微分并没有使用另一套数学，它只是选择了不构造完整 Jacobian 的计算顺序。

二阶部分也保持算子视角。对矩阵变量，与其存储四阶 Hessian，不如直接计算

\[
\mathcal H_X[H] = D(\nabla_Xf)[H].
\]
这正是 Newton--Krylov、Gauss--Newton 和高维曲率分析真正使用的对象。

\subsection{常用公式都从恒等式现场推出}

Trace 方法与常用矩阵导数集中处理二次型、逆矩阵和 log-determinant，但目的不是重新制造公式表。

二次型通过变量每次出现的位置产生多条微分路径，因此一般有

\[
\nabla_x(x^TAx)=(A+A^T)x,
\]

只有 \(A\) 对称时才是 \(2Ax\)。

逆矩阵导数从

\[
XX^{-1}=I
\]

微分得

\[
d(X^{-1}) = -X^{-1}(dX)X^{-1}.
\]
实际计算会进一步把它改写成线性系统的隐式微分与伴随求解，避免显式形成逆。

log-determinant 则由 Jacobi 公式得到

\[
d\log\det X = \operatorname{tr}(X^{-1}dX).
\]
在正定锥上，它连接体积、Gaussian 似然、协方差和内点障碍；数值上应通过 Cholesky 对角线计算，而不是先求行列式。

\subsection{在模型中看见同一条反向结构}

从梯度推导到反向传播把工具用于三个连续案例。

最小二乘与 logistic 回归共享线性预测 \(z=Xw\)，差别只在输出空间的残差信号；反向都通过 \(X^T\) 把它拉回参数。softmax 交叉熵写成

\[
\operatorname{LSE}(z)-y^Tz
\]

后，梯度自然化简为 \(p-y\)，同时保留稳定 log-sum-exp 计算。两层网络则把这些局部规则组合起来，逐节点推导权重、bias 与输入梯度。

自动微分可以生成相同结果，但手推仍然提供四类不可替代的信息：

\begin{itemize}
\tightlist
\item
  结果应有的形状；
\item
  可以融合和稳定化的结构；
\item
  共享分支处为何必须累积梯度；
\item
  不可微、状态更新和近似求解处究竟在对什么求导。
\end{itemize}

\subsection{一套可重复的推导流程}

面对新表达式，可以按以下顺序工作：

先列出每个变量和中间量的形状，圈出真正变量。把标量目标改写成内积、范数平方或 trace；给表达式加入微分，并使用和、乘积、逆与链式规则展开。删除二阶小量后，通过 trace 循环与转置把目标变量的微分放到最后，匹配

\[
df=\operatorname{tr}(G^TdX).
\]

然后检查 \(G\) 是否与 \(X\) 同形、是否需要因对称或其他约束投影、标量特例是否一致。实现时再决定需要完整梯度、JVP、VJP 还是 HVP，并用一个小型、光滑、确定的例子做方向差分检查。

这个流程比背公式慢几行，却能迁移到未见过的模型。

\subsection{前置与后续}

本部分依赖 微积分 的一阶、二阶近似，也依赖 线性代数 的线性映射、伴随、内积、正定性与矩阵分解。数值分析 解释为什么求导公式中的逆应实现为线性求解，凸优化 则把梯度与 Hessian 转化为最优性和曲率条件。

这些工具将在 数理统计 的似然、信息矩阵和渐近理论，以及 综合应用 的线性模型、降维和神经网络中反复使用。

严格定义与布局可参考 Magnus 与 Neudecker 的 \emph{Matrix Differential Calculus}；应用顺序参考 Stanford CS229，并结合 The Matrix Calculus You Need for Deep Learning 的图形化解释。The Matrix Cookbook 适合查找候选恒等式，但每个结果仍应回到微分、形状与定义域独立验证。
