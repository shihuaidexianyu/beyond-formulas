点估计、区间和检验都在输出某种统计结论，但现实系统最终必须采取行动：报告一个数、选择一个假设、安排复查、拒绝自动判断，或把有限资源分配给一部分对象。只说估计量``准确''还不够；必须说明真实状态是什么、允许哪些行动、不同错误损失多少，以及评价是在每个状态下、先验平均下还是最坏情况下进行。

本单元把统计推断推进到决策层，并把``可信预测''拆成可分别检验的性质。概率校准不等于分类能力，预测区间覆盖不等于个体条件保证，模型不确定也不自动说明应该拒绝行动。只有把概率输出、覆盖程序和现实损失接在同一条链上，可信才不只是一个形容词。

六节依次展开：

\begin{enumerate}
\tightlist
\item
  状态、行动、损失与风险构成一个决策问题：用统一语言重写估计、检验和预测，并区分风险、支配与单次误差。
\item
  Bayes rule 最小化后验期望损失：说明 posterior mean、median 与 mode 分别由什么损失产生，以及 prior 如何进入行动而不只是进入估计。
\item
  Minimax 与可容许性比较整条风险曲线：研究没有可信先验时怎样控制最坏情况，并划清``不可被一致改进''与``实际值得采用''的距离。
\item
  Proper scoring rule 与校准衡量概率是否诚实：推导 Brier 与 log loss 的含义，区分 calibration、resolution 与 discrimination。
\item
  Conformal prediction 用可交换性换有限样本覆盖：从 split conformal 的秩论证得到预测集合，并说明边际覆盖、条件覆盖与分布漂移的边界。
\item
  拒绝预测与容量约束把概率变成行动：推导代价敏感阈值、abstention 和 top-budget 规则，再把静态决策连接到因果干预与序贯决策。
\end{enumerate}

本单元不承诺存在一个脱离任务的``最可信模型''。同一概率模型在 log loss 下可能很好，在罕见灾难的非对称损失下仍不可用；同一个 conformal 集合可以达到名义覆盖，却宽到没有决策价值。可信性的对象必须始终说清：哪一种随机机制、哪一个目标人群、哪一种行动和哪一种错误。

\subsection{怎么读这一章}

先从状态、行动与损失建立决策问题，再看 Bayes 与 minimax 用不同方式处理未知状态。后三篇转向概率预测：评分规则检查概率是否诚实，conformal prediction 给覆盖保证，拒绝预测与容量约束则把概率真正变成行动。不要把估计准确直接等同于决策正确。
