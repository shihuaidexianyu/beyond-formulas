\section{多重比较改变错误的计量单位}

单个 level 0.05 检验把一次第一类错误控制在 5\%。若同时做 100 个相互独立且都为真的原假设检验，每个仍用 0.05，至少出现一个假阳性的概率是

\[
1-(1-0.05)^{100}
\approx0.994.
\]

问题不在某个 \(p\) 值计算错误，而在错误控制的单位从``单个假设''变成了``整个假设家族''。

\subsection{Family-wise error rate}

FWER 定义为

\[
\Pr(V\ge1),
\]

其中 \(V\) 是被错误拒绝的真原假设数。Bonferroni 方法要控制 FWER 不超过 \(\alpha\)，就对 \(m\) 个检验分别使用

\[
\alpha/m.
\]

因为 union bound 给

\[
\Pr(\text{至少一个假阳性})
\le
\sum_{j=1}^m
\Pr(\text{第 }j\text{ 个假阳性})
\le\alpha.
\]

它不需要检验独立，但可能保守。Holm step-down 方法按 \(p\) 值排序逐步比较，在同样广泛条件下控制 FWER，且其拒绝集合总包含 Bonferroni 的拒绝集合，功效不会更低。

FWER 适合任何一个假阳性都代价很高的确认性研究。

\subsection{False discovery rate}

大规模探索中，可能愿意容忍少量假发现，以换取更多真发现。FDR 定义为

\[
\mathbb E
\left[
\frac{V}{\max(R,1)}
\right],
\]

其中 \(R\) 是总拒绝数。

Benjamini--Hochberg 方法把排序后的 \(p\) 值

\[
p_{(1)}\le\cdots\le p_{(m)}
\]

与

\[
\frac{k}{m}q
\]

比较，取最大的满足者并拒绝其之前的假设。在独立或某些正依赖条件下，它控制 FDR 不超过目标 \(q\)。

``FDR 5\%''不是说每个被拒绝假设有 5\% 概率为假，也不是保证每次实验的假发现比例都不超过 5\%；它是重复实验下比例的期望控制。

\subsection{假设家族必须由问题定义}

哪些检验算在同一个 family 中，取决于共同决策与报告语境。若研究者做了 50 个分析却只把最小的三个写进论文，不能把 family 事后定义成这三个。

多重性来源不仅是多个终点，还包括：

\begin{itemize}
\tightlist
\item
  多个亚组；
\item
  多个时间点；
\item
  多种模型规格；
\item
  两个尾部中事后选一个；
\item
  反复查看累计数据（即``\(p\) 值''一节中的可选停止问题）；
\item
  特征筛选后在同一数据上检验；
\item
  多个实验中只报告成功者。
\end{itemize}

隐藏选择使名义 \(p\) 值和区间失去校准，即使最终表格只有一行。

\subsection{同时置信区间与检验对应}

若希望一组区间以至少 \(1-\alpha\) 的概率同时覆盖所有参数，可用 Bonferroni 把每个区间置信水平提高到

\[
1-\alpha/m.
\]

这比逐个 95\% 区间更宽，但提供 family-wise coverage。对线性模型的所有对比、时间序列曲线或置信带，还可以使用相关结构得到比 Bonferroni 更紧的方法。

只对显著结果报告普通区间会产生选择偏差；选择性推断需要条件化选择事件或使用独立确认数据。

\subsection{设计阶段比事后修补更重要}

可靠流程包括：

\begin{itemize}
\tightlist
\item
  预先定义 primary endpoint 和假设家族；
\item
  把探索与确认分析分开；
\item
  对探索阶段使用 FDR，对关键结论使用独立验证；
\item
  记录所有分析路径，而不只保留显著结果；
\item
  在模型与数据处理选择中使用 nested 或 holdout 设计；
\item
  报告校正方法、依赖假设和未校正结果的用途边界。
\end{itemize}

多重校正不是对``坏数据挖掘''的万能惩罚。若假设由同一数据无限适应性生成，简单把最终数量代入 Bonferroni 也未必覆盖完整选择过程。

\subsubsection{留给读者}

某研究同时检验 20 个独立终点，得到最小 \(p\) 值 0.004。它能通过未校正 0.05 阈值，却不能通过 Bonferroni 0.0025。分别解释这两个决策控制的错误是什么。若这些终点只是探索并希望控制 FDR，还需要哪些其余 \(p\) 值信息？
