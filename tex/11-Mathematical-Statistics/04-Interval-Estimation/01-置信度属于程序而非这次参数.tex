\section{置信度属于程序，而非这次参数}

频率学派置信区间是一个由随机样本生成随机集合的程序：

\[
C(X)=[L(X),U(X)].
\]

若对每个参数值 \(\theta\)，

\[
\Pr_\theta\{\theta\in C(X)\}
\ge1-\alpha,
\]

则称它具有至少 \(1-\alpha\) 的覆盖率；若恰好相等，则是精确覆盖。

概率作用于尚未观测的随机区间 \(C(X)\)。重复抽样时，有些区间覆盖固定真值，有些不覆盖，长期比例约为 \(1-\alpha\)。

\subsection{观测后区间与参数都固定}

一旦数据已经观测，例如得到区间 \([0.86,0.97]\)，在经典频率学派模型中，参数 \(\theta\) 固定，端点也固定。事件``\(\theta\) 位于这个区间''已经真或假，不再具有由抽样随机性定义的 95\% 概率。

更准确的说法是：

\begin{quote}
我们使用了一个在假设模型下长期覆盖率为 95\% 的程序，它在这次数据上产生区间 \([0.86,0.97]\)。
\end{quote}

日常语言常把它简写为``95\% 置信区间''，但不应进一步解释成``参数有 95\% 概率在其中''。后者是 Bayesian 后验区间一类的陈述，需要参数概率模型。

\subsection{覆盖率是模型条件下的性质}

如果样本并非 iid、标准误估计错误、模型尾部不符或数据经过选择，名义 95\% 区间可能没有 95\% 实际覆盖。

覆盖率还可能是：

\begin{itemize}
\tightlist
\item
  对每个 \(\theta\) 都至少 95\% 的 uniform coverage；
\item
  只在大样本极限趋于 95\% 的 pointwise asymptotic coverage；
\item
  对某个参数子空间或条件事件的 coverage；
\item
  经过多重比较校正后的 simultaneous coverage。
\end{itemize}

只写``95\%''而不说明覆盖对象与条件，会掩盖重要差别。

\subsection{Bernoulli Wald 区间的刻意失败}

对样本比例 \(\hat\theta=S/n\)，中心极限定理给出 \(\hat\theta\) 的渐近正态性；把标准误中的未知 \(\theta\) 替换为 \(\hat\theta\)（plug-in 近似），得到 Wald 区间：

\[
\hat\theta
\pm
z_{1-\alpha/2}
\sqrt{
\frac{\hat\theta(1-\hat\theta)}n
},
\]

其中 \(z_{1-\alpha/2}\) 是标准正态的 \(1-\alpha/2\) 分位数，即 \(\Phi(z_{1-\alpha/2})=1-\alpha/2\)；Wald 区间的一般形式在渐近区间一节讨论。它简单，却在小样本或 \(\theta\) 接近 0、1 时表现很差。若 \(S=0\)，估计标准误为零，区间退化成 \([0,0]\)；这显然没有合理表达``零次成功只说明概率可能较小''，更不可能对所有小 \(\theta>0\) 保持 95\% 覆盖。

区间还可能超出合法参数范围 \([0,1]\)。简单截断能修复范围，却不自动修复覆盖率。

更好的选择包括 Wilson score、Clopper--Pearson、似然比或 Bayesian Beta 后验区间；它们在保守性、长度与解释上各有取舍。没有一个区间只凭``看起来合理''就获得覆盖保证。

\subsection{置信水平与区间长度的取舍}

在其他条件相同时，更高置信水平要求更宽区间；更多样本通常缩短区间。宽度还受数据噪声、设计效率和参数可识别性影响。

一个极窄区间可能来自：

\begin{itemize}
\tightlist
\item
  样本量很大；
\item
  观测噪声很小；
\item
  模型错误地忽略相关性；
\item
  只报告选择后的最有利结果；
\item
  把估计超参数当成已知；
\item
  数值软件漏掉一层不确定性。
\end{itemize}

所以``窄''不是自动的高质量标志。

\subsection{先看程序，再看这次端点}

评价区间方法时，应问：

\begin{itemize}
\tightlist
\item
  覆盖率是精确还是渐近？
\item
  对哪些参数值有效，边界表现怎样？
\item
  区间平均长度与最坏长度如何？
\item
  是否在看到数据、选择模型或选择终点后才决定报告对象？
\item
  聚类、时间依赖和有限总体设计是否进入标准误？
\item
  同时报告多个区间时是否需要 family-wise 或 false coverage 控制？
\end{itemize}

这些问题都发生在``这次区间是多少''之前。
