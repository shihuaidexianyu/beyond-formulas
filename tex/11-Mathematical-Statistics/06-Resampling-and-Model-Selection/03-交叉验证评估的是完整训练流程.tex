\section{交叉验证评估的是完整训练流程}

训练误差总偏向已经用于拟合的数据。要估计模型面对新数据的预测表现，必须让评价样本在训练时保持未见。

\(K\) 折交叉验证把数据分成 \(K\) 个互斥折。每次用 \(K-1\) 折训练，在剩下一折计算损失；轮流让每折做验证，最后平均：

\[
\widehat R_{\mathrm{CV}} = \frac1n
\sum_{i=1}^n
L
\bigl(
y_i,
\hat f^{(-k(i))}(x_i)
\bigr).
\]
\(\hat f^{(-k(i))}\) 表示没有使用第 \(i\) 点所在折训练的模型。

\subsection{被评估的是算法，不是一个固定拟合结果}

每个折都重新训练模型，所以 CV 评估的是``给定这种训练流程和约 \(n(K-1)/K\) 个样本，会得到怎样的预测器''，而不是最终在全部 \(n\) 个样本上重训出的某个固定参数向量。

训练样本稍少会带来偏差；不同折训练集高度重叠，使折损失相关，方差估计也不简单。\(K\) 大时训练规模更接近全数据，但计算更贵、折间相关更强；\(K\) 小时验证折更大但训练偏差更明显。

不存在对所有问题最好的 \(K\)。5 折或 10 折是常见折中，不是数学定律。

\subsection{预处理必须在每个训练折内完成}

若先用全数据标准化、填补缺失、筛选特征或做 PCA，再进行 CV，验证折信息已经进入预处理参数，导致乐观偏差。

正确流程是在每个折中：

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  仅用训练折拟合预处理；
\item
  用该预处理变换训练折与验证折；
\item
  在训练折拟合模型；
\item
  在验证折计算损失。
\end{enumerate}

所有利用标签或数据分布的步骤都属于训练 pipeline，包括异常值阈值、特征选择、类别平衡和早停。

``没有直接把验证标签送进模型''仍可能泄漏；全数据选择变量已经使用了它们。

\subsection{调参与最终评估需要嵌套}

若用同一组 CV 分数选择超参数，并把最小分数当成无偏泛化误差，选择会利用验证噪声，结果偏乐观。

Nested cross-validation 使用：

\begin{itemize}
\tightlist
\item
  内层 CV 选择超参数；
\item
  外层未见折评估``包含调参在内的完整流程''。
\end{itemize}

最终还可以在全训练数据上按选定规则重训一次，并只在独立测试集上做一次最终评估。测试集若被反复查看和用于改模型，就已经变成验证集，需要新的测试数据。

\subsection{切分必须尊重预测场景}

随机逐行切分隐含样本可交换——这与上一节置换检验要求排列尊重可交换结构是同一原则：重排必须匹配数据生成机制。以下数据需要特殊设计：

\begin{itemize}
\tightlist
\item
  时间预测：按时间先后切分，不能让未来进入训练；
\item
  同一患者多条记录：按患者分组，避免身份泄漏；
\item
  地理或设备簇：按部署单位分组；
\item
  稀有类别：可分层保持类别比例，但不能破坏组结构；
\item
  domain shift：验证集应模拟目标部署域，而非训练混合分布。
\end{itemize}

交叉验证只能估计与切分机制相符的泛化风险。若未来数据分布改变，随机 CV 的低误差没有保证。

\subsection{不确定性不能用普通独立样本公式}

各折损失共享大部分训练数据，因此不独立。把 \(K\) 个折分数当作 \(K\) 个独立观测计算标准误通常过于乐观。

可以报告逐样本 out-of-fold 损失、重复交叉验证的分布，或使用问题特定方差估计；比较两个算法时最好在相同折上做配对差异。无论方法怎样，模型选择的不确定性和数据集有限性都不应被一个小数点后四位的 CV 均值掩盖。

\subsection{评价指标也属于设计}

平方误差、绝对误差、log loss、AUC 与校准误差衡量不同目标。若先看结果再选择最有利指标，又产生多重选择。

训练损失、调参指标和最终业务代价可以不同，但关系应预先说明。例如不平衡分类中准确率可能很高却没有价值；概率模型只看 AUC 会忽略校准。

\subsubsection{留给读者}

一个数据集包含 1000 张图像，来自 100 位患者，每位患者 10 张。设计 5 折 CV 时，为什么逐图像随机分折会高估泛化？给出按患者分组的切分，并说明如果目标是预测``同一患者未来图像''而不是``新患者''，评价设计是否应改变。
