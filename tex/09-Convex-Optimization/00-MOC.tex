\section{凸优化：让局部信息获得全局意义}

许多决策都可以说成一句话：在允许的方案中，选择最好的一个。训练模型要选参数，资源调度要选产量，投资要选权重，控制系统要选输入。困难并不只是``求导并令它等于零''。

现实问题可能有约束，最优点可能在边界；目标可能不可微，梯度为零也可能是最大点或鞍点；变量可能有成千上万维，甚至一个局部极小点之外还藏着更深的谷地。一般优化中，找到一个不能继续局部改善的点，并不等于知道它是全局最好。

凸优化的特殊之处在于，它为问题加入了一种足够强、又能覆盖大量实际模型的几何结构：

\begin{itemize}
\tightlist
\item
  可行域中任意两个方案之间的线段仍然可行；
\item
  目标沿这条线段不会高于端点值的线性插值。
\end{itemize}

这两个条件共同排除了``必须先变差，才能到达更优谷地''的情形。若远处真有更优点，朝它走一小步就应立刻改善。因此，凸问题的局部最优必为全局最优。

这只是第一层收益。凸性还使切平面成为全局下界，使约束能够产生对偶证书，使 KKT 条件在适当条件下成为最优性的充要条件，并让算法的误差可以由原始---对偶间隙量化。凸优化是一套把建模、几何、证明与计算连起来的语言；它提供强结构，却不等于每个问题都``容易优化''。

\subsection{四个彼此不能替代的问题}

学习每个模型与算法时，都应同时追问：

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  \textbf{模型是否忠实？} 变量、目标和约束是否真正表达现实选择，单位与定义域是否正确？
\item
  \textbf{问题是否凸？} 目标曲率、约束方向和可行域几何是否满足严格定义，还是只凭图形猜测？
\item
  \textbf{候选解为何最优？} 一阶条件、KKT、对偶下界或其他证书能否排除更好的可行点？
\item
  \textbf{计算结果是否可信？} 原始残差、对偶残差、互补性和 gap 是否达到与数据精度相称的尺度？
\end{enumerate}

一个问题凸，不代表模型正确；一个算法收敛，不代表收敛到全局最优；求解器显示 optimal，也不代表统计假设和业务约束合理。把这四个层次分开，是使用优化工具时最重要的判断。

\subsection{从一次建模开始}

从现实选择到优化问题先把决策变量、目标、约束和可行域放进同一个生产决策中。这里会区分不可行、无界、最优值不取得和存在最优解，也会通过非凸地形说明局部信息为何通常不够。这样，凸性的出现有明确动机，而不是突然增加的一条定义。

接下来把凸性的两半分开理解。凸集合从方案混合出发，经过仿射集、半空间、分离超平面、凸锥和半正定锥，回答``两个可行方案之间能否安全移动''。凸函数再从弦、上境图、切平面和 Hessian 回答``移动过程中目标怎样变化''。严格凸与强凸也在这里被区分：前者主要给唯一性，后者提供统一曲率和定量误差控制。

\subsection{从结构识别到最优性证书}

掌握基本几何后，识别凸优化问题把线性规划、范数拟合、二次规划、几何规划和半正定规划串成一条建模主线。要理解 epigraph、对数换元与 Schur 补怎样把一个问题转成标准凸形式，以及变换必须保存哪些定义域和解；LP、QP、SOCP、SDP 等缩写只是模型名称。

然后进入 对偶。Lagrangian 的出发点非常具体：把合法符号的约束项加到目标上，再对变量取下确界，就得到一个对所有可行解都成立的下界。弱对偶永远成立；在凸性和 Slater 等条件下，下界能够碰到原始最优值。KKT 条件正是这个接触发生时的等号条件，而乘子又可以解释为约束的边际价格。

对偶让``找到一个好方案''升级为``同时说明它为什么已经足够好''。原始可行点给最小化问题的上界，对偶可行点给下界；两者相遇时，最优性不再依赖算法自我宣告。

\subsection{算法选择来自可用的计算原语}

优化算法不按名称罗列更新式，而按局部模型与结构逐步增强：

\begin{itemize}
\tightlist
\item
  梯度下降只用一阶信息，真正难点是步长与曲率尺度；
\item
  Newton 法用 Hessian 校正方向，每步更贵但局部极快；
\item
  次梯度法处理一般非光滑凸函数，但不保证逐步下降；
\item
  近端与坐标方法保留 \(L_1\)、投影和可分结构；
\item
  原始---对偶内点法沿中心路径逼近边界，并用 KKT 残差与 gap 控制高精度。
\item
  走出凸性后，下降引理只能保证接近驻点；随机梯度还会引入由步长与方差共同决定的噪声平台。
\end{itemize}

选择算法时应问：能否廉价计算全梯度？Hessian-vector product 是否可用？非光滑项有没有闭式 prox？投影是否简单？矩阵是否稀疏？需要多高精度和何种证书？没有一种方法能脱离这些问题被称为普遍最好。

\subsection{应用是模型假设的压力测试}

最后，建模案例用 Ridge、logistic 回归、Lasso、SVM 和均值---方差投资组合重新汇合全书内容。

这些案例会刻意保留理论之外的限制：可分 logistic 回归可能没有有限最优参数；Lasso 的精确零会在相关特征间不稳定；SVM margin 不是概率；投资组合求解误差可能远小于预期收益的估计误差。凸性保证模型内部的全局可解与可认证，却不能把错误数据变成正确结论。

\subsection{走出凸性时先降低结论强度}

非凸与随机优化把凸理论当作比较基准，而不是继续假装全局证书仍然存在。对光滑非凸目标，确定性梯度下降通常先得到驻点复杂度；Hessian 再区分负曲率鞍点与局部极小。若目标额外满足 PL 条件，才可能在不凸的情况下恢复函数值全局收敛。

SGD 部分把无偏性、方差、batch size 与步长放进同一条期望下降界；动量和 Adam 则被视为扩展状态空间中的动力系统。最后把优化误差、统计误差与近似误差分开，防止训练损失下降被误读成泛化保证。这一单元是通向深度学习训练与强化学习随机逼近的理论桥梁，不改变前面凸优化结论的适用范围。

\subsection{学完后应具备的工作方式}

面对一个新问题，可以按以下顺序思考：

先用自然语言说明可改变的决策、不可违反的边界和``更好''的含义，再写变量与单位。接着判断每个约束集合是否凸、目标是否凸，并尝试用已知原子和保凸运算证明，而不是依靠数值采样。若原式不凸，寻找 epigraph、透视、对数变量、Schur 补或凸松弛，同时明确变换是否等价。

模型确定后，先看是否存在可行点、最优值是否可能无界、解是否取得。再根据光滑性、强凸性、稀疏性和所需精度选择算法。求解结束时，不只保留参数向量，还应报告原始可行性、KKT 残差、对偶下界、gap 和对数据扰动的敏感性。

这样的流程把优化从``调用求解器''变成可解释、可审核的推理。

\subsection{前置与后续}

本部分依赖 线性代数 的内积、正定矩阵、特征值和分解，也依赖 微积分 的梯度、Hessian 与 Taylor 展开。数值分析 提供条件数、稳定线性求解和误差验证；这些内容决定算法公式能否可靠落地。

后续的 矩阵微积分 会让复杂模型的梯度推导更系统，数理统计 会解释经验风险与正则化的推断含义，机器学习与深度学习则把这些工具组合到更完整的问题中。

本部分的主要讲授顺序参考 Stanford EE364A Convex Optimization I 与 Boyd--Vandenberghe《Convex Optimization》。课程材料提供概念边界、标准例题和严谨定理；这里的章节则围绕读者的认知冲突、推理连续性和实际建模重新组织。
