\section{梯度下降真正难在步长}

梯度指出函数增长最快的方向，因此负梯度是最自然的局部下降方向：

\[
x_{k+1}=x_k-\alpha_k\nabla f(x_k).
\]

公式只有一行，但算法成败几乎都藏在步长 \(\alpha_k\) 中。步长太小，每次只移动一点；步长太大，会越过谷底来回振荡，甚至让目标不断增大。

\subsection{光滑性把``足够小''变成可计算条件}

假设梯度是 \(L\)-Lipschitz：

\[
\lVert\nabla f(x)-\nabla f(y)\rVert
\le L\lVert x-y\rVert.
\]

它保证函数在每一点都被一个二次模型从上方控制：由微积分基本定理，
\(f(y)-f(x)=\int_0^1\nabla f(x+t(y-x))^T(y-x)\,dt\)，
把被积函数中多出的 \(\nabla f(x+t(y-x))-\nabla f(x)\) 用 Lipschitz 条件放缩，即得

\[
f(y)
\le
f(x)+\nabla f(x)^T(y-x)
+\frac L2\lVert y-x\rVert^2.
\]

令 \(y=x-\alpha\nabla f(x)\)，得到

\[
f(x-\alpha\nabla f(x))
\le
f(x)
-\alpha\left(1-\frac{L\alpha}{2}\right)
\lVert\nabla f(x)\rVert^2.
\]

因此只要

\[
0<\alpha<\frac2L,
\]

右侧就严格下降，除非梯度已经为零。常用选择 \(\alpha=1/L\) 给出清楚的下降量：每一步 \(f\) 至少减少 \(\frac{1}{2L}\lVert\nabla f(x)\rVert^2\)。

这个推导说明步长上界来自最大曲率。梯度很大本身不是问题；真正危险的是在曲率陡峭方向上移动过远。

\subsection{狭长二次碗为什么让梯度下降曲折}

考虑

\[
f(x,y)=\frac12x^2+50y^2.
\]

Hessian 的特征值是 \(1\) 和 \(100\)，所以 \(L=100\)。若按陡峭的 \(y\) 方向选择稳定步长，沿平缓 \(x\) 方向每次只能前进很少；若按 \(x\) 方向选择大步长，\(y\) 会发散。

等高线是一组狭长椭圆，梯度垂直于等高线，迭代会横跨谷地左右摆动，缓慢沿长轴前进。对 \(m\)-强凸、\(L\)-光滑函数，条件数

\[
\kappa=L/m
\]

控制这种最坏曲率差异。固定步长 \(1/L\) 时，函数值或距离具有几何收敛，典型收缩因子与 \(1-m/L\) 有关；\(\kappa\) 越大，收敛越慢。

变量缩放和预条件并非美化输入，而是在改变算法看到的几何。若令 \(z=\sqrt{100}\,y\)，二次碗在新坐标中更接近圆形，同一步长能更均衡地处理各方向。

\subsection{\texorpdfstring{不知道 \(L\) 时怎样选择步长}{不知道 L 时怎样选择步长}}

固定步长简单，但往往不知道可靠的 \(L\)。回溯线搜索从一个候选步长开始，检查 Armijo 充分下降条件：

\[
f(x+\alpha p)
\le
f(x)+c\alpha\nabla f(x)^Tp,
\qquad0<c<1,
\]

若不满足就把 \(\alpha\) 乘以 \(\beta\in(0,1)\)，直到条件成立。对负梯度方向 \(p=-\nabla f(x)\)，右侧明确要求目标下降一定比例。

回溯的价值是自适应局部曲率，而且只需要函数值和方向导数。它不寻找一条射线上的精确最小值；精确线搜索本身可能比原问题还贵，也可能在噪声目标上没有意义。

实际实现还会考虑：

\begin{itemize}
\tightlist
\item
  初始步长是否沿用上一次成功值；
\item
  函数值是否因浮点误差或随机 mini-batch 而噪声化；
\item
  约束问题是否需要投影或可行方向；
\item
  目标计算是否昂贵，能否承担多次回溯；
\item
  下降条件使用的是全批目标还是随机估计。
\end{itemize}

\subsection{收敛速度依赖额外结构}

对一般 \(L\)-光滑凸函数，步长 \(1/L\) 的梯度下降典型函数值误差为

\[
f(x_k)-f^\star=O(1/k).
\]

若再有 \(m\)-强凸性，则得到线性或几何收敛：

\[
f(x_k)-f^\star
\le
\left(1-\frac mL\right)^k
\bigl(f(x_0)-f^\star\bigr).
\]

这里``线性收敛''不是说误差随 \(k\) 成线性函数，而是每次乘一个固定小于一的因子。术语容易误导，最好始终同时看公式。

Nesterov 加速利用一个带动量的辅助序列，可把一般光滑凸问题的速率改善到 \(O(1/k^2)\)。它表明一阶方法仍有更精细的最优设计；但加速序列对参数、噪声和重启可能更敏感，不能只凭渐近阶选择实现。

\subsection{什么时候可以停止}

相邻两次迭代几乎不动，可能因为接近最优，也可能因为步长太小。更有意义的证据包括：

\begin{itemize}
\tightlist
\item
  无约束凸问题的梯度范数；
\item
  约束问题的投影梯度或 KKT 残差；
\item
  已知强凸常数时，由梯度或函数值得到的误差界；
\item
  原始---对偶间隙；
\item
  验证集、统计误差或应用允许的精度。
\end{itemize}

优化误差不需要远小于数据噪声和建模误差。追求机器精度可能浪费大量计算，也可能放大病态方向。

本节的下降引理本身没有使用凸性；凸性主要负责把局部下降连接到全局函数值误差。``非凸与随机优化：能下降不等于找到全局解''会保留同一条下降不等式，却把结论降为驻点复杂度，并进一步加入鞍点、PL 条件与随机梯度噪声。
