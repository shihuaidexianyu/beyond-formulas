\section{对偶变量是约束的影子价格}
\label{sec:09-Convex-Optimization/05-Duality/04}

\begin{quote}
\textbf{先抓住这一点：}最优对偶变量近似回答“把这条约束放宽一点，最优目标能改善多少”。它不是约束本身的固定价格，而是当前模型、数据和最优点附近的边际价值。
\end{quote}

Lagrange 乘子不仅用于证明最优性。它还回答建模者最关心的一类问题：如果稍微放宽某条约束，最优目标能改善多少？

考虑带扰动的问题

\[
\begin{aligned}
p^\star(u,v)=
\min_x\quad&f_0(x)\\
\text{s.t.}\quad&
f_i(x)\le u_i,\\
&h_j(x)=v_j.
\end{aligned}
\]

\(u_i>0\) 表示放宽第 \(i\) 个不等式，\(v_j\) 改变等式右端。其 Lagrangian 可写为

\[
L(x,\lambda,\nu;u,v) = f_0(x)
+\lambda^{\trans}(f(x)-u)
+\nu^{\trans}(h(x)-v).
\]
若零扰动问题满足强对偶，最优乘子 \((\lambda^\star,\nu^\star)\) 给出

\[
p^\star(u,v)
\ge
p^\star(0,0)
-{\lambda^\star}^{\trans}u
-{\nu^\star}^{\trans}v.
\]

推导只需对扰动问题用一次第一节的弱对偶。由定义，扰动后的对偶函数只是零扰动版本的整体平移：

\[
\inf_x L(x,\lambda,\nu;u,v)
=g(\lambda,\nu)-\lambda^{\trans}u-\nu^{\trans}v.
\]

对偶可行性 \(\lambda\ge0\) 不随扰动改变，故 \((\lambda^\star,\nu^\star)\) 仍是扰动问题的合法乘子；弱对偶说 \(p^\star(u,v)\) 不低于右端在该乘子处的取值，而强对偶给出 \(g(\lambda^\star,\nu^\star)=p^\star(0,0)\)，合起来就是上面的下界。

这是一条关于扰动价值函数的全局仿射下界。若价值函数在零点可微，就有局部灵敏度

\[
\frac{\partial p^\star}{\partial u_i}(0)
=-\lambda_i^\star,
\qquad
\frac{\partial p^\star}{\partial v_j}(0)
=-\nu_j^\star.
\]

不等式乘子因此是放宽约束的边际价值。因为我们在做最小化，放宽上界通常使最优成本下降，所以导数是 \(-\lambda_i^\star\)。

\subsection{资源约束的单位帮助解释乘子}

假设目标单位是``元''，某条资源约束写成

\[
a_i^{\trans}x\le b_i
\]

且 \(b_i\) 的单位是``工时''。若改写为 \(a_i^{\trans}x-b_i\le0\)，对应乘子单位就是``元/工时''。它近似表示再增加一个工时能降低多少最优成本，或在利润最大化的符号约定下增加多少最优利润。

单位分析还能发现错误：若同一约束两边的量纲不一致，其乘子也不会有可解释单位。对偶变量的现实含义由原始约束的表达方式决定；把约束整体乘以 100，乘子会缩小 100 倍，但影子价格与约束变化量的乘积保持一致。

\subsection{影子价格是局部信息}

乘子给出小扰动附近的一阶变化，不能无条件外推到大幅政策改变。约束放宽后，活跃集合可能改变，价值函数出现折点，新的资源成为瓶颈。

若价值函数不可微，最优乘子可能不唯一；这时它们对应一个次梯度集合，而不是单一精确价格。冗余约束也可能导致多组乘子表达同一个最优性证书。

因此报告影子价格时应同时说明：

\begin{itemize}
\tightlist
\item
  约束的方向、尺度和单位；
\item
  乘子来自哪个最优解及何种符号约定；
\item
  灵敏度适用的是局部扰动；
\item
  当前约束是否活跃，乘子是否唯一；
\item
  模型参数变化是否会改变凸性或可行结构。
\end{itemize}

\subsection{原始---对偶间隙是算法的共同尺度}

对任意原始可行点 \(x\) 和对偶可行点 \((\lambda,\nu)\)，

\[
f_0(x)-g(\lambda,\nu)\ge0.
\]

这个差值把``还可能改善多少''变成可计算量——第二节的区间 \(10.1\le p^\star\le10.4\) 正是由这样一个差值夹出来的。一些算法主要更新原始变量，一些主要更新对偶变量，原始---对偶方法则同时推进两者；无论路径怎样，最终都希望：

\begin{itemize}
\tightlist
\item
  原始残差趋近零；
\item
  对偶残差趋近零；
\item
  互补性误差趋近零；
\item
  原始---对偶间隙趋近零。
\end{itemize}

单看目标在相邻迭代间变化很小，可能只是算法停滞；间隙很小则给出更直接的最优性信息。当然，只有在两侧近似可行且数值尺度合理时，gap 才能被正确解释。

\subsection{对偶让``解''变成``解释''}

原始解告诉我们应该怎样行动；对偶解说明哪些约束真正稀缺、候选解为何已经不能显著改善，以及改变哪项资源最有价值。两者共同构成一个可审计的决策结果。

这也解释了为什么对偶理论即使不直接用于求解仍然重要。它可以：

\begin{itemize}
\tightlist
\item
  给算法提供下界和停止证书；
\item
  给不可行问题提供分离或矛盾证书；
\item
  揭示正则化与约束形式之间的关系；
\item
  导出分解算法，使大问题按资源或数据块拆开；
\item
  将统计估计中的惩罚参数解释为某类约束价格。
\end{itemize}
