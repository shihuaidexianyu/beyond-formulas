\section{自动微分给出导数，梯度检查建立信任}

训练模型需要梯度。最直观的办法似乎是轻微改变参数，再观察损失变化：

\[
f'(x)\approx \frac{f(x+h)-f(x)}{h}.
\]

这个想法很自然，却不能直接支撑百万参数模型。每改变一个参数都要重新运行一次模型，成本随参数个数增长；更麻烦的是，\(h\) 既不能太大，也不能太小。自动微分解决的是``怎样高效得到程序的导数''，有限差分更适合解决另一个问题：``我们凭什么相信这段求导程序没有写错？''

先把两者分开，许多数值微分的困惑就会消失。

\subsection{有限差分为什么有一个最佳尺度}

若 \(f\) 足够光滑，Taylor 展开给出

\[
f(x+h)=f(x)+hf'(x)+\frac{h^2}{2}f''(x)+O(h^3).
\]

因此前向差分的截断误差是 \(O(h)\)。同时使用两侧函数值可以消去偶次项：

\[
\frac{f(x+h)-f(x-h)}{2h}
=f'(x)+O(h^2).
\]

这说明在精确算术中，缩小 \(h\) 会提高中心差分精度。但浮点数还带来另一股相反的力量：两个非常接近的函数值相减会丢失有效数字，舍入误差再被 \(1/h\) 放大，量级大致是 \(O(u/h)\)，其中 \(u\) 是机器精度。

中心差分的总误差可粗略写成

\[
C_1h^2+C_2\frac{u}{h}.
\]

令两项量级相等（也即对总误差关于 \(h\) 求极小），得到典型尺度 \(h\sim u^{1/3}\)；实际还要乘上与变量大小相称的尺度。于是``把步长设得极小''并非更严谨；它最终会让舍入误差主导。可靠检查通常会扫描几个数量级的 \(h\)，寻找误差先下降、后上升的区间。

\subsection{自动微分是在程序上执行链式法则}

自动微分不是有限差分，也不是把公式展开成庞大的符号表达式。它把程序拆成一系列基本运算，并沿计算图传播局部导数。

考虑

\[
y=\sin(x_1x_2)+x_2^2.
\]

程序可以拆成

\[
a=x_1x_2,\qquad b=\sin a,\qquad c=x_2^2,\qquad y=b+c.
\]

正向模式在给定输入方向 \(v\) 后，同时传播数值和切向量，最终得到 Jacobian-vector product，记作 \(Jv\)。若输入维度很小、输出很多，一次正向传播就能看到一个输入方向如何影响所有输出。

反向模式从输出端放入一个伴随量，按相反顺序传播，得到 vector-Jacobian product，通常写成 \(v^TJ\) 或其转置形式。对标量损失，令输出伴随为 \(1\)，一次反传便能得到所有输入分量的梯度。这正是神经网络参数很多、损失只有一个时，反向传播如此合适的原因。

在上面的例子中，记 \(\bar z\) 为输出 \(y\) 对中间量 \(z\) 的偏导（伴随量）。反向传播先得到

\[
\bar b=1,\qquad \bar c=1,
\]

再经 \(b=\sin a\) 的局部导数得到

\[
\bar a=\cos a,
\]

最后，\(\bar a\) 沿 \(a=x_1x_2\) 分给两个输入（局部导数分别为 \(x_2\) 与 \(x_1\)），\(\bar c\) 沿 \(c=x_2^2\) 流向 \(x_2\)（局部导数 \(2x_2\)），累积为

\[
\frac{\partial y}{\partial x_1}=x_2\cos(x_1x_2),
\qquad
\frac{\partial y}{\partial x_2}=x_1\cos(x_1x_2)+2x_2.
\]

这里的``精确''应理解为：没有有限差分的截断误差，并且严格按所执行程序的链式法则求导。基本运算仍在浮点数中完成，原程序中的溢出、消减和近似也会进入导数。自动微分不会把一个数值不稳定的前向计算自动变稳定。

\subsection{不必构造完整 Jacobian}

对映射 \(f:\mathbb R^n\to\mathbb R^m\)，完整 Jacobian 有 \(mn\) 个元素。许多应用真正需要的只是它对一个方向的作用：

\begin{itemize}
\tightlist
\item
  敏感度传播需要 \(Jv\)；
\item
  标量损失反传需要 \(J^Tv\)；
\item
  Hessian-vector product 可以通过求导模式组合得到；
\item
  隐式模型的反传往往只需求一个伴随线性系统。
\end{itemize}

先问``需要完整矩阵，还是只需要矩阵乘向量''能避免大量时间和存储浪费。这与下一节``需要的是解，而不是逆矩阵''的思想完全相同。

\subsection{怎样让梯度检查真正有判断力}

设自动微分给出梯度 \(g_{\mathrm{AD}}\)。逐坐标中心差分可以比较

\[
g_i^{\mathrm{num}}
=\frac{f(x+he_i)-f(x-he_i)}{2h}.
\]

参数很多时，更经济的做法是选一个方向 \(v\)，比较

\[
\frac{f(x+hv)-f(x-hv)}{2h}
\quad\text{与}\quad
v^Tg_{\mathrm{AD}}.
\]

一次随机方向检查会同时混合许多梯度分量，适合快速发现整体错误；它仍可能漏掉与所选方向近乎正交的错误，所以应使用多个方向，并在小问题上补充逐坐标检查。

比较时不能只看绝对差。一个常用的尺度化指标是

\[
\frac{\lVert g_{\mathrm{AD}}-g_{\mathrm{num}}\rVert}
{\max\{1,\lVert g_{\mathrm{AD}}\rVert,\lVert g_{\mathrm{num}}\rVert\}}.
\]

阈值没有脱离上下文的统一答案：float64 与 float32、平滑函数与尖点附近、数量级为 \(10^{-2}\) 与 \(10^8\) 的问题，本来就不应要求相同误差。

一个可信的梯度检查环境通常还需要：

\begin{itemize}
\tightlist
\item
  使用小而可读的输入，并优先采用较高精度；
\item
  固定随机种子，关闭 dropout，冻结会更新的统计量；
\item
  避开 ReLU 的零点、max 的并列点和 clip 边界等不可微位置；
\item
  尝试多个步长，而不是把一次比较当成裁决；
\item
  先故意注入符号、转置或广播错误，确认检查确实会失败。
\end{itemize}

最后一点尤其重要。测试能通过不等于实现处处正确；但如果一个故意的错误也能通过，这个测试就没有建立任何信任。

\subsection{不光滑处的分歧未必是程序错误}

以 \(f(x)=|x|\) 为例，在 \(x=0\) 处不存在唯一导数。对称差分给出零，自动微分系统也可能按约定返回零，但这不意味着它们发现了一个经典意义上的导数。带数据分支、离散采样、排序和原地状态更新的程序还可能需要自定义求导规则或重新表述。

因此梯度检查首先检查的是：在选定的光滑测试点、确定的程序状态和既定求导约定下，两个独立计算是否一致。它不能替代对模型可微性的数学判断。
