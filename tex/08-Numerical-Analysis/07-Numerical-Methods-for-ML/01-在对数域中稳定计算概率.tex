\section{在对数域中稳定计算概率}

设一个三分类模型给出的 logits（送入 softmax 之前的原始打分）是

\[
x=(1000,1001,999).
\]

按照 softmax 的定义，第一个类别的概率是

\[
p_1=\frac{e^{1000}}{e^{1000}+e^{1001}+e^{999}}.
\]

公式没有任何问题，但普通浮点数无法表示这些指数。分子和分母先溢出为无穷，随后无穷除以无穷产生 NaN。更高精度只能把失败推迟到更大的输入，不能改变算法本身的脆弱性。

这个失败暴露了一个关键事实：softmax 并不关心 logits 的绝对位置，只关心它们之间的差。给分子和分母同乘 \(e^{-c}\)，概率不变：

\[
\frac{e^{x_i}}{\sum_j e^{x_j}} = \frac{e^{x_i-c}}{\sum_j e^{x_j-c}}.
\]
因此可以自由选择 \(c\)。取 \(c=m=\max_j x_j\) 最有利，因为所有指数的自变量都不大于零，并且至少有一个恰好为零。上面的例子变成

\[
x-m=(-1,0,-2),
\]

于是只需计算

\[
(e^{-1},1,e^{-2})\approx(0.3679,1,0.1353).
\]

归一化后得到

\[
p\approx(0.2447,0.6652,0.0900).
\]

这不是对 softmax 的近似改写；在精确算术中，它与原式完全相同。变化的只是通往答案的计算路径。

\subsection{Log-sum-exp 是共同的核心}

概率模型经常需要计算

\[
\operatorname{LSE}(x)=\log\sum_{i=1}^n e^{x_i}.
\]

若先计算指数和再取对数，仍会遇到溢出。把最大值提出后，得到恒等式

\[
\operatorname{LSE}(x)
=m+\log\sum_{i=1}^n e^{x_i-m},
\qquad m=\max_i x_i.
\]

它还有一个有用的解释。平移后达到最大值的那项恰好是 \(e^0=1\)，其余各项都落在 \((0,1]\) 中，因此

\[
1\le \sum_i e^{x_i-m}\le n.
\]

三边取对数（中间一项正是 \(\operatorname{LSE}(x)-m\)），再加回 \(m\)，得到

\[
\max_i x_i
\le \operatorname{LSE}(x)
\le \max_i x_i+\log n.
\]

LSE 可以看作最大值的光滑版本：某一项远大于其他项时，它几乎就是最大值；多项接近时，额外的对数项记录了这些竞争者的共同贡献。对 \(x_i\) 求导还能得到

\[
\frac{\partial\operatorname{LSE}}{\partial x_i}
=\frac{e^{x_i}}{\sum_j e^{x_j}}
=p_i,
\]

也就是说，softmax 正是 LSE 的梯度。

\subsection{不要先得到概率，再对它取对数}

分类模型训练时，通常直接需要正确类别 \(y\) 的负对数概率，无须先单独算出 \(p_y\)。直接写成

\[
-\log p_y
=\operatorname{LSE}(x)-x_y
\]

比``先 softmax，再取 log''更可靠。如果某个极小概率已经下溢为零，再取对数只能得到负无穷；而 \(x_y-\operatorname{LSE}(x)\) 仍可能是完全可表示的有限数。

因此稳定的 log-softmax 应写成

\[
\log p_i=x_i-\operatorname{LSE}(x),
\]

交叉熵则直接由 logits 计算。成熟的数值库通常把这些步骤融合起来，不仅减少舍入路径，也避免生成一整张随后马上被取对数的概率表。

\subsection{下溢不总是灾难，语义不清才是}

减去最大值后，极小的项仍可能下溢为零。例如 \((0,-1000)\) 在常见精度下会得到概率 \((1,0)\)。这通常只是说第二个概率小到当前精度无法分辨，它对归一化和的贡献本来就可以忽略。真正危险的是：

\begin{itemize}
\tightlist
\item
  在平移前让大指数上溢；
\item
  需要对极小概率取对数，却先把它舍入成零；
\item
  在低精度中完成求和，使许多小贡献过早消失；
\item
  把一个没有合法概率语义的输入强行归一化。
\end{itemize}

最后一种情况常见于 mask。把无效位置设为 \(-\infty\) 后，只要一行中至少有一个有效项，稳定 softmax 就能正常工作；若整行都被 mask，最大值是 \(-\infty\)，相减会出现未定义的 \(-\infty-(-\infty)\)。此时不存在唯一``正确概率''，调用者必须明确这行应跳过、报错还是按约定填充。数值技巧不能替业务语义作决定。

混合精度训练还要注意：输入可以是较低精度，但最大值归约、指数和或损失累计往往值得放在更高精度中。减最大值解决了动态范围问题，却不会自动恢复低精度已经丢掉的有效数字。

\subsection{一个可迁移的检查方法}

实现稳定概率计算后，不要只测试几个正常输入。至少应检查：

\begin{itemize}
\tightlist
\item
  给所有 logits 加同一常数后，softmax 不变，LSE 只增加该常数；
\item
  每行概率非负且总和接近一；
\item
  极大、极小、相等 logits 和单个有效 mask 都返回有限结果；
\item
  log-softmax 与直接由 LSE 得到的结果一致；
\item
  全 mask、空归约和 NaN 的行为符合接口约定。
\end{itemize}

这些检查来自数学性质，因此比抄一组期望输出更能暴露轴、广播和 mask 错误。
