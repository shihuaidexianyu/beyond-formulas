机器学习常把数学公式直接交给计算机：把指数写成指数，把导数写成差分，把逆矩阵写成求逆。小例子上它们似乎都能运行，规模、数值范围或精度一变化，程序却会突然出现无穷、零、NaN，或者得到一个看似合理但并不可信的答案。

同一个数学量往往有多种计算形式，即使公式正确，也会由此产生数值问题。实数算术里等价的表达式，在浮点数里可能经历完全不同的中间量和舍入误差。数值计算真正训练的是一种翻译能力：先辨认我们究竟需要什么，再选择不制造多余困难的表达方式。

本单元用机器学习中最常见的三类计算来练习这种能力：

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  在对数域中稳定计算概率：从会溢出的 softmax 出发，推导 log-sum-exp、log-softmax 与稳定交叉熵。
\item
  自动微分给出导数，梯度检查建立信任：区分有限差分与自动微分，并说明怎样检查一段反向传播实现。
\item
  需要的是解，而不是逆矩阵：把回归、Gaussian 过程和隐式微分中的逆矩阵改写成结构化线性求解。
\end{enumerate}

这三节不是孤立的编程技巧。它们都在落实前面反复出现的同一原则：保存问题的数学结构，减少不必要的中间量，并用独立证据判断结果是否可信。读完后，可以继续把这些原则用于 凸优化、机器学习和深度学习。
