\section{割线法、收敛阶与混合策略}
\label{sec:08-Numerical-Analysis/02-Root-Finding/03}

\begin{quote}
\textbf{先抓住这一点：}没有导数时，割线法可以用最近两次函数值估计斜率。它通常比二分快，却不再自动守住含根区间。成熟求根器会先尝试快步，发现候选不安全时退回二分，而不是只忠于一种公式。
\end{quote}

\subsection{导数昂贵时，能否从函数值中学习斜率}

上一节 Newton 法的速度依赖 \(f'(x_k)\)。若函数是昂贵仿真、导数不可得或实现成本很高，可以用最近两个点的割线斜率

\[
\frac{f(x_k)-f(x_{k-1})}{x_k-x_{k-1}}
\]

近似导数。把它代入 Newton 步得到

\[
x_{k+1}
=x_k-f(x_k)
\frac{x_k-x_{k-1}}{f(x_k)-f(x_{k-1})}.
\]

这就是割线法。它也可看作对 \((x_{k-1},f(x_{k-1}))\) 与 \((x_k,f(x_k))\) 做线性插值，再取插值直线的零点。

每步通常只需一次新的函数计算，因为旧函数值可缓存。但若两个函数值太接近，分母会让步长巨大或失去可信度。

\subsection{它比线性快，却没有 Newton 那么快}

在足够光滑函数的简单根附近，割线误差满足

\[
e_{k+1}\approx C e_ke_{k-1}.
\]

这是一个可严格证明的渐近关系：把割线法看作两点线性插值后，用插值余项即可推出；此处先承认它，只看它的后果。

若假设渐近上 \(|e_k|\) 与 \(|e_{k-1}|^p\) 同阶，代入上式：左边 \(|e_{k+1}|\) 的阶是 \(|e_{k-1}|^{p^2}\)，右边 \(|e_k|\cdot|e_{k-1}|\) 的阶是 \(|e_{k-1}|^{p+1}\)，故指数需要满足

\[
p^2=p+1.
\]

正根为黄金比

\[
p=\varphi=\frac{1+\sqrt5}{2}\approx1.618.
\]

所以割线法是超线性收敛：误差比固定比例下降更快，却不及 Newton 的二次收敛。它用较低的单步信息成本换取稍低的渐近阶。

\subsection{收敛阶怎样统一比较}

若 \(x_k\to r\)，且存在 \(p\ge1\) 与 \(0<\lambda<\infty\) 使

\[
\lim_{k\to\infty}
\frac{|e_{k+1}|}{|e_k|^p}=\lambda,
\]

称序列具有 Q-阶 \(p\)。当 \(p=1\) 时还要求 \(\lambda<1\) 才称线性收敛；\(p>1\) 称超线性，\(p=2\) 称二次。

典型比较是：

\begin{itemize}
\tightlist
\item
  二分法的区间误差界每步乘 \(1/2\)，线性；
\item
  收缩不动点迭代通常线性；
\item
  简单根附近的割线法约 \(1.618\) 阶；
\item
  简单根附近的 Newton 法二次。
\end{itemize}

但收敛阶只描述接近根后的渐近误差，不等于总时间。导数计算可能比函数贵；二分法可能先花较少代价进入安全区域；高阶方法也可能在前几步走得很差。

\subsection{经验阶为何容易被误判}

若真根已知，可观察

\[
\frac{|e_{k+1}|}{|e_k|^p}
\]

是否趋向常数。真根未知时常用相邻迭代差替代误差，并由三到四个点估计阶，但这只在迭代已进入渐近区且误差模式规则时可靠。

早期迭代受全局几何影响，末期又会撞上舍入平台；在这两个区间拟合出来的``阶''都可能毫无意义。振荡、多步方法和有限终止也未必有简单 Q-阶。

\subsection{割线点不是括区点}

即使最初两点端点异号，普通割线法的后续点也不保证留在区间内，所以它没有二分法的根存在证书。噪声函数、近平坦区和多个根还会让斜率估计失真。

工程上常用混合方法：

\begin{enumerate}
\tightlist
\item
  始终维护异号括区；
\item
  尝试割线或更高阶逆插值步；
\item
  若候选超出括区、进展太小或数值不安全，就用二分；
\item
  更新括区并重复。
\end{enumerate}

Brent 类方法就是这种思想的代表。它们让快速插值承担大部分工作，同时保留二分法的最坏保证。成熟的算法设计要知道何时不应信任一种漂亮公式，而不会对它一味坚持。

\subsection{选择方法时真正比较什么}

\begin{itemize}
\tightlist
\item
  是否能找到可靠括区？
\item
  函数是否连续、光滑，是否含噪声？
\item
  导数能否准确、低成本获得？
\item
  需要确定误差证书还是只要快速近似？
\item
  是单次求根，还是同一模型要解很多次？
\item
  每次函数、导数评估和通信成本分别是多少？
\end{itemize}

若导数可由自动微分低成本获得，Newton 可能更合适；若函数是黑盒，割线或逆插值更有吸引力；若失败代价很高，括区混合方法通常是默认选择。
