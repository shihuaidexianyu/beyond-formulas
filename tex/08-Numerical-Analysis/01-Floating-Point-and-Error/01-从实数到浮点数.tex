\section{从实数到浮点数}

\subsection{\texorpdfstring{为什么 \(0.1+0.2\) 不一定恰好等于 \(0.3\)}{为什么 0.1+0.2 不一定恰好等于 0.3}}

十进制中的 \(0.1\) 是有限小数，但它的二进制展开无限循环。计算机只能保留有限位，因此实际参与运算的是附近的二进制有理数。问题不是计算机``算错了小学加法''，而是程序中的三个数从一开始就未必是数学上的 \(0.1,0.2,0.3\)。

浮点数可以看成有限位的科学计数法。一个规格化二进制浮点数写作

\[
x=(-1)^s(1.f)_2\times 2^e,
\]

其中 \(s\) 是符号，\(1.f\) 是有限精度的有效数，\(e\) 是有限范围的指数。指数让同样数量的有效位覆盖很宽的数量级；代价是可表示数在实轴上并不等距。

\subsection{精度近似恒定的是相对量}

在区间 \([2^e,2^{e+1})\) 内，相邻规格化数的间距固定为某个 \(2^e\) 的倍数；进入下一个数量级，间距也翻倍。所以浮点系统提供近似恒定的相对精度，而不是恒定的绝对精度。

以 IEEE 754 binary64 为例，有 53 个二进制有效位。\(1\) 附近的相邻数间距是

\[
2^{-52}\approx2.22\times10^{-16},
\]

但在 \(2^{53}\) 附近，相邻浮点数间距已经是 2。因此

\[
2^{53}+1
\]

不能与 \(2^{53}\) 区分，而较小范围内的许多整数却能精确表示。所谓``双精度能表示约 16 位十进制有效数字''，说的是有效位数量，不是所有 16 位十进制数都精确。

\subsection{machine epsilon 与 unit roundoff}

不同教材和软件对 machine epsilon 的叫法并不完全一致。最常见的两个量是：

\begin{itemize}
\tightlist
\item
  \(\varepsilon_{\mathrm{mach}}\)：从 1 到下一个更大浮点数的间距，binary64 中为 \(2^{-52}\)；
\item
  unit roundoff \(u\)：舍入到最近时的最大相对舍入误差，通常为 \(2^{-53}=\varepsilon_{\mathrm{mach}}/2\)。
\end{itemize}

在结果正常、没有上溢或下溢且采用舍入到最近时，常用模型是

\[
\operatorname{fl}(x)=x(1+\delta),
\qquad |\delta|\le u.
\]

对一次基本运算也常写

\[
\operatorname{fl}(a\circ b)
=(a\circ b)(1+\delta),
\qquad |\delta|\le u,
\]

其中 \(\circ\) 是加、减、乘或除，并假设精确结果处于正常范围。

这个模型来自``舍入到最近的绝对误差不超过半个 ulp''（ulp 即 unit in the last place，当前数量级下相邻浮点数的间距），再除以当前数值尺度。但它不是全实轴的无条件定理。

\subsection{零附近发生了什么}

最小规格化数与零之间若直接留下巨大空档，小数值会突然下溢为零。IEEE 754 使用次正规数填补这一区域，使间距逐渐延伸到零，称为 gradual underflow。代价是此处不再保持正常数那样的相对精度。

此外标准还定义：

\begin{itemize}
\tightlist
\item
  正零与负零，用于保留某些极限方向；
\item
  正负无穷，常由有限非零数除以零或溢出产生；
\item
  NaN，表示未定义或无效结果，例如 \(0/0\)；
\item
  多种舍入模式，默认通常是舍入到最近且 ties-to-even。
\end{itemize}

NaN 与任何值比较都不相等，包括它自身。无穷和 NaN 能让异常传播到后续计算，但不能替代输入检查。

\subsection{浮点运算不是实数代数}

每次运算后都可能舍入，因此结合律一般失效：

\[
(a+b)+c\ne a+(b+c).
\]

若 \(a\) 很大、\(b\) 很小，\(a+b\) 可能舍入回 \(a\)，小量完全消失。改变求和顺序会改变哪些低位先被丢掉。

乘法分配律也可能失效，理论上相等的两种表达式未必产生相同浮点结果。这不是允许我们随意忽略代数，而是提醒：代数改写本身可以成为数值算法设计的一部分。

\subsection{精确可表示与精确可计算是两件事}

\(1/8=0.001_2\) 可精确表示，\(1/10\) 不可以。但即使输入和精确答案都可表示，中间运算仍可能溢出或舍入；反过来，一连串近似中间值也可能最终舍入到正确答案。

所以判断一次计算是否可靠，不能只问输入能否精确表示，还要追踪：中间尺度、运算顺序、条件性与算法稳定性。

\subsubsection{一个小实验应看到什么}

在支持 IEEE binary64 的环境中，比较 1、\(2^{40}\) 和 \(2^{53}\) 附近的相邻浮点数。你应看到绝对间距随尺度增加，而相对间距大致稳定。再解释为什么用固定绝对容差比较所有数量级的数并不合理。
