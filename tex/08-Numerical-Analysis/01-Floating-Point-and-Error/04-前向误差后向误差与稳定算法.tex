\section{前向误差、后向误差与稳定算法}

\subsection{``答案错了多少''不是唯一的评价方式}

设问题是计算 \(y=f(x)\)，算法给出 \(\hat y\)。

前向误差直接比较输出：

\[
\|\hat y-f(x)\|.
\]

后向误差则问：能否找到一个邻近输入 \(x+\Delta x\)，使

\[
\hat y=f(x+\Delta x)
\]

精确成立？若所需的 \(\Delta x\) 很小，算法的结果就是一个轻微扰动问题的精确答案。

后向观点特别符合浮点现实：输入本身常已由测量和舍入得到，我们很难要求程序忠于一个无限精确、从未真正拥有的数；更合理的要求是，程序不要制造比原始数据尺度大得多的额外扰动。

\subsection{稳定性与条件性怎样接合}

若算法相对后向误差约为 \(O(u)\)，而问题相对条件数为 \(\kappa\)，一阶上通常有

\[
\text{相对前向误差}
\lesssim
\kappa\times\text{相对后向误差}
=O(\kappa u).
\]

所以：

\begin{itemize}
\tightlist
\item
  良态问题加后向稳定算法，通常得到高精度答案；
\item
  病态问题加后向稳定算法，前向误差仍可能很大；
\item
  良态问题却得到大误差，往往暴露算法不稳定或实现错误。
\end{itemize}

条件数评价问题，稳定性评价算法。二者相乘才接近最终可实现精度。

\subsection{线性系统中的后向误差}

对近似解 \(\hat x\)，残差 \(r=b-A\hat x\) 满足

\[
A\hat x=b-r.
\]

因此若只允许扰动右端，\(\hat x\) 是系统 \((A,b-r)\) 的精确解。相对残差可作为可计算的后向误差指标；但要判断前向误差，还必须乘上 \(A\) 的条件数。这正是上一节``小残差不保证小前向误差''在后向误差语言下的重述。

这解释了一个看似矛盾的现象：成熟线性代数库可能给出极小残差，却无法为极病态系统提供很多正确数字。算法已经很好，问题本身却没有允许更多精度。

\subsection{代数等价公式可以稳定性完全不同}

求二次方程

\[
ax^2+bx+c=0
\]

的根时，直接公式

\[
x_{1,2}=\frac{-b\pm\sqrt{b^2-4ac}}{2a}
\]

在 \(b^2\gg4ac\) 时，其中一个符号会让两个近似相等的大数相减，丢失有效位。可先稳定计算绝对值较大的根

\[
q=-\frac12\left(b+\operatorname{sign}(b)\sqrt{b^2-4ac}\right),
\]

再用根的乘积求

\[
x_1=\frac qa,\qquad x_2=\frac cq.
\]

这里没有改变数学问题，只是避免让一个根通过灾难性消去产生。稳定算法常来自对问题结构的重新表达。

另一个日常例子是计算 \(\sqrt{a^2+b^2}\)。直接平方可能先溢出，即使最终结果仍可表示。按较大绝对值缩放后计算

\[
m\sqrt{(a/m)^2+(b/m)^2},
\qquad m=\max(|a|,|b|),
\]

能控制中间尺度。高质量库函数 hypot 就体现了这种思想。

\subsection{稳定不等于收敛，也不等于没有误差}

数值分析中``稳定''有多个语境：

\begin{itemize}
\tightlist
\item
  浮点算法的后向或前向稳定性；
\item
  ODE 离散方法对扰动随时间传播的稳定性；
\item
  迭代法是否收敛到固定点；
\item
  统计估计在分布变化下是否稳健。
\end{itemize}

它们有关联，但定义不能互换。一个随步长 \(h\to0\) 收敛的离散公式，在固定有限精度下仍可能因过小步长遭受舍入；一个后向稳定直接求解器也不涉及迭代收敛率。

\subsection{如何评价一个数值算法}

面对算法输出，可以按顺序问：

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  问题采用什么输入与扰动模型，条件数多大？
\item
  输出的残差或其他可计算后向误差多大？
\item
  算法是否利用了缩放、主元、正交变换等稳定结构？
\item
  理论误差界中的维数、增长因子和高阶项是否仍小？
\item
  与更高精度参考或独立方法比较时，是否符合 \(O(\kappa u)\) 的预期？
\end{enumerate}

几个普通样例算对不能证明稳定；一个病态样例算错也不能证明算法不稳定。要有意识地分离问题敏感性与算法效应。
