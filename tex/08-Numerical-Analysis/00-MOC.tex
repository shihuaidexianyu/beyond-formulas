微积分和线性代数常在精确实数上讨论问题：方程有一个根，积分等于某个极限，线性系统存在唯一解。计算机面对的世界却不同。它只能保存有限位数字，只能做有限次运算，而观测数据本身也常带有误差。

因此，``知道答案在数学上是什么''与``能够可靠地算出答案''之间还隔着一门学问。数值分析研究怎样在有限信息和有限资源下得到可解释、可控制、可验证的近似；机械地把公式交给计算机无法保证这些性质。

一个很小的例子就能看出差别。表达式

\[
\frac{1-\cos x}{x^2}
\]

在 \(x\to0\) 时趋于 \(1/2\)。若直接在浮点数中计算，当 \(x\) 很小时，\(\cos x\) 与 \(1\) 几乎相等，相减会丢掉大量有效数字。利用恒等式 \(1-\cos x=2\sin^2(x/2)\) 后，同一个数学量可以更稳定地写成

\[
\frac12\left(\frac{\sin(x/2)}{x/2}\right)^2.
\]

数值分析关心的正是这种差异：公式等价，计算路径却不等价。

\subsection{贯穿全书的四个问题}

每遇到一种新方法，都可以用四个问题把它放回整体中：

\begin{enumerate}
\tightlist
\item
  \textbf{问题本身有多敏感？} 输入改变一点，精确答案会改变多少？这是条件性；病态问题无法仅靠更聪明的实现彻底治愈。
\item
  \textbf{我们用什么有限对象代替原问题？} 网格、插值多项式、差分公式和有限维基底都会引入离散化或截断误差。
\item
  \textbf{算法怎样传播舍入误差？} 一个良好条件的问题仍可能被不稳定算法算坏；后向稳定性说明算法是否近似解了一个邻近问题。
\item
  \textbf{凭什么相信计算已经足够好？} 残差、误差估计、步长加密、守恒量和独立实现提供证据；``程序返回了数字''从来不是证据。
\end{enumerate}

条件性属于问题，稳定性属于算法，精度则由二者和数据共同决定。把这三者混在一起，是学习数值方法时最常见的根本误解。

\subsection{怎么读这一章}

首先阅读 浮点数、误差与稳定性。这里建立后续所有章节都会使用的语言：机器精度、消减、条件数、前向误差、后向误差和稳定算法。没有这层理解，后面的公式很容易退化成算法清单。

接着从最容易看见迭代过程的一维问题开始。非线性方程求根比较二分法的可靠与 Newton 法的快速，借此理解局部收敛、初值依赖、停止条件和混合策略。它让``算法为什么成功或失败''第一次变得可观察。

当函数只通过有限个样本被认识时，问题从``找一个数''变成``重建一个对象''。插值与逼近从唯一插值多项式出发，经过 Runge 现象、节点选择和样条，说明增加次数为何不一定改善答案，也说明局部性和光滑性怎样成为可利用的结构。

有了插值观点，数值微分与数值积分就不再是一堆凭空出现的系数。差分和求积公式都可以从局部多项式推出；截断误差与舍入误差的竞争解释了为什么步长不能无限缩小，自适应求积则展示怎样让计算把资源放到真正困难的区域。

随后进入许多科学计算的核心：数值线性代数。这里把矩阵结构转化为算法选择，抽象线性代数的结论作为前提：为何分解优于求逆，QR 如何稳定地做最小二乘，Cholesky 怎样利用正定性，Krylov 方法为何只靠矩阵---向量乘法就能处理大型稀疏系统，以及 SVD 如何揭示数值秩。

常微分方程的数值解把误差推进到时间轴上。Euler 方法让局部误差如何积累为全局误差变得清楚；Runge--Kutta、自适应步长、稳定区域和刚性方程进一步说明，高阶并不自动等于高效，稳定性要求有时比局部精度更强。

对于周期、振荡和光滑结构，Fourier 与谱方法提供另一种观察方式：从时域转到频域后，微分可以变成乘法，卷积可以变成逐点乘积。采样定理划定离散数据能够保留的信息边界，FFT 解释快速计算，而谱方法展示光滑性如何换来极高精度。

最后，机器学习中的稳定计算把整门课的判断方式带回常见模型实现。稳定 softmax、自动微分与梯度检查、结构化线性求解看似分散，实质都要求区分数学对象与计算表达式，并为结果建立独立证据。

这条顺序适合第一次系统学习，但不是不可打破的锁链。完成浮点误差基础后，可以按问题需要进入求根、积分、线性代数或 ODE；遇到不明白的误差现象时，再回到四个贯穿问题定位它来自问题、离散化还是算法。

\subsection{学完后应形成的判断}

面对一个新的数值任务，成熟的第一反应不应是搜索某个现成函数，而应先做以下辨认：

\begin{itemize}
\tightlist
\item
  精确问题是什么，解是否存在且唯一？
\item
  输入是精确给定、测量得到，还是已经离散化？
\item
  问题是否病态，尺度是否合理，是否存在可利用的对称、稀疏、正定或光滑结构？
\item
  真正需要的是完整对象，还是某个作用结果，例如矩阵乘向量、线性系统的解或积分的一个数值？
\item
  主要误差来自数据、离散化、迭代未收敛还是浮点舍入？
\item
  有什么不依赖同一计算路径的证据能够验证答案？
\end{itemize}

这种判断比记住更多算法名称更可迁移。算法会变化，库接口会变化，但有限精度、问题敏感性和结构利用之间的关系不会消失。

\subsection{与前后内容的关系}

本部分默认读者已经具备 微积分 和 线性代数 的基本语言。微积分提供极限、Taylor 展开、积分和微分方程；线性代数提供子空间、正交、特征值和分解。数值分析则追问：当这些对象只能通过有限数据和有限运算接近时，哪些性质还能保存，误差怎样控制。

后续的 凸优化 会频繁调用线性求解、条件数和迭代收敛；数理统计、机器学习与深度学习也会依赖稳定概率计算、最小二乘和数值验证。

本部分的讲授主线主要参考 MIT 18.330 Introduction to Numerical Analysis，数值线性代数补充参考 MIT 18.335；定义、例题与进一步推导可结合 Fundamentals of Numerical Computation 和 Nick Trefethen 的数值分析讲座 阅读。课程材料决定内容边界和推导节奏，但各章均围绕本知识库的认知主线重新组织。
