\section{欧拉回路与一笔画}
\label{sec:04-Discrete-Mathematics/03-Graph-Theory/07}

\begin{quote}
\textbf{先抓住这一点：}走遍每条边的全局问题，被``每次进入必须成对离开''压缩成局部度数奇偶。充分性证明又能直接变成拼接回路的构造算法。
\end{quote}

\subsection{七桥问题：一次散步引出的新学科}

18 世纪的 Königsberg（今俄罗斯 Kaliningrad）被 Pregel 河分成四块陆地：河心的两座岛屿与两岸，由七座桥相互连接。当地流传一个散步难题：能否从某处出发，把每座桥恰好走一遍，最后回到出发点？

许多人反复尝试，走法换了一条又一条，始终无人成功。但``没人走出来''不等于``走不出来''：尝试只能覆盖有限条路线，无法穷举所有可能。要断言一件事``不可能''，靠试是不够的，必须给出理由。这正是数学登场的缝隙。

Euler 在 1736 年给出的理由，第一步是抽象。他注意到陆地有多大、桥有多长、走得快慢全都无关紧要，唯一要紧的是``哪块陆地与哪块陆地之间有桥、有几座''。于是把每块陆地缩成一个顶点，每座桥画成一条边：Königsberg 变成一张只有 \(4\) 个顶点、\(7\) 条边的图。顶点的位置、边的曲直都失去意义，连接模式被完整保留。一个看似关于城市与散步的问题，答案只依赖``谁与谁相连''——图论作为一门学科由此诞生。

有两点值得记下。其一，这座桥图不是简单图：有陆地之间并排着两座桥，即存在平行边。本节允许平行边（多重图），度数按相连边数计算，每条平行边各算一次。其二，四块陆地的度数分别是 \(5,3,3,3\)，全是奇数——这就是全部关键信息。

\subsection{把``走不走得通''翻译成度数}

沿桥行走经过一块陆地时，必然沿一座桥进入、沿另一座桥离开，桥梁被进出成对地消耗。于是：

\begin{itemize}
\tightlist
\item
  中途经过的陆地，桥数必须能两两配对，即度数为偶数；
\item
  若起点与终点不同，起点多出一次``只出不进''，终点多出一次``只进不出''，恰好这两处度数为奇；
\item
  若起点与终点相同，首次离开与最后返回也配成一对，起点度数同样是偶数。
\end{itemize}

四块陆地的度数全是奇数，落不进上面任何一条。散步难题因此有了确定的否定答案：这样的走法不存在。结论不靠更多的尝试，而靠一次奇偶计数。

把问题一般化。经过每条边恰好一次的闭合迹称为欧拉回路（Euler 回路），不要求闭合的称为欧拉通路；中文里更形象的说法是一笔画：笔尖不离纸，每条线恰好描一次。\hyperref[sec:04-Discrete-Mathematics/03-Graph-Theory/02]{``路径、环与连通性''}曾直接给出过这个问题的判据，本节补上它的来历、完整证明与用法。

\subsection{定理：连通，且度数全为偶数}

欧拉定理：无向图中所有非零度顶点属于同一连通分量时，它存在欧拉回路，当且仅当每个顶点的度数为偶数。

这里特意没有粗略写成``图连通''。一个顶点若完全没有关联边，它不会妨碍走遍其余边；要求孤立点也连进来反而过强。若讨论的图没有孤立点，条件当然就简化为通常所说的``连通且度数全偶''。若图一条边也没有，空的闭合游走通常被视作平凡情形，是否称它为欧拉回路取决于教材约定，不影响非空图的判据。

必要性就是上面的计数。沿欧拉回路行进，每次进入一个顶点都沿另一条没走过的边离开，关联边一进一出配成对；起点的首次出发与最后返回也配成一对。所以每个顶点度数为偶数。

充分性需要把回路构造出来。设图连通且度数全偶。从任一顶点 \(v_0\) 出发，沿尚未走过的边随意行进，走过的边随即作废。关键观察是：除 \(v_0\) 外，任何顶点一旦被走进，其偶数度数保证还剩一条没走过的边可以走出，所以行进只可能在回到 \(v_0\) 时停下——得到一条闭合回路 \(C\)。

若 \(C\) 已用完所有的边，证明结束。否则删去 \(C\) 的边：沿途顶点每被经过一次就删去两条关联边，剩余图的度数仍是偶数。由原图连通，剩余边中必有一条的端点落在 \(C\) 上——否则从剩余边通往 \(v_0\) 的路径会在进入 \(C\) 前断掉。从这样的顶点 \(u\) 出发，在剩余图中重复同样的行进，必又回到 \(u\)，得到新回路 \(C'\)。把 \(C'\) 在公共顶点 \(u\) 处拼进 \(C\)：沿 \(C\) 走到 \(u\) 时，先绕 \(C'\) 一圈再继续。合并后的回路更长，剩余度数依旧全偶，过程可以不断重复，直到没有边剩下。最终的闭合回路经过每条边恰好一次，即欧拉回路。

有两点值得强调。其一，判据只数度数，与图怎么画、边是曲是直毫无关系；自环给度数贡献 \(2\)，也不影响奇偶判断。其二，这个证明本身就是算法：不断找回路、拼接回路。判定与构造在这里是同一件事，欧拉回路不仅存在，而且可以高效地找出来。

\subsection{把存在性证明真正执行出来}

上面的构造称为 Hierholzer 算法。实际执行时，不必每找一个小回路就把整条路线重新复制。可以维护一个栈：

\begin{enumerate}
\tightlist
\item
  把任一非零度顶点压栈；
\item
  若栈顶还有未使用的关联边，就沿这条边前进，把新顶点压栈并删除该边；
\item
  若栈顶已经无边可走，就把它弹出并写到答案序列末尾；
\item
  重复直到栈空，最后反转答案序列。
\end{enumerate}

为什么``无路可走''时才输出顶点？因为那一刻说明从它出发的所有边都已经被嵌进某个闭合段，顶点在最终路线中的后继位置已经确定。栈里尚未弹出的部分则是还可能被新回路插入的骨架。这个数据结构恰好实现了证明中的``找到回路再拼接''。

用邻接表存图，并让每条边在被使用时只删除一次，整个过程的时间复杂度为

\[
O(|V|+|E|).
\]

这与读入图本身的成本同阶。若只需要判断是否存在，检查非零度顶点的连通性并统计奇度顶点也只需线性时间。这里的``容易''不只是有简洁定理，还包括能在输入规模的线性时间内构造答案。

\subsection{一个小例子：为什么不能只靠贪心不回头}

考虑两个三角形只共享一个顶点 \(v\) 的图。所有非共享顶点度数为 \(2\)，\(v\) 的度数为 \(4\)，所以一定存在欧拉回路。若从一个三角形出发，沿它绕回 \(v\)，眼前看似已经闭合，但另一个三角形还没有走。这不是失败：从 \(v\) 再绕第二个三角形，把新回路插入第一次经过 \(v\) 的位置即可。

这个例子解释了为什么``一直选一条没走过的边''还不是完整算法。随意行进能保证形成闭合回路，却不保证第一次闭合就覆盖全图；必须保留在旧回路上继续插入新回路的机制。Hierholzer 算法的栈正是在记这些待拼接位置。

另一个常见算法是 Fleury 算法：每一步尽量不走割边，除非已经没有别的选择。直觉是不要过早把剩余图切断。它容易手算，却需要反复判断割边，朴素实现比 Hierholzer 慢得多。二者都能找到欧拉路线，但前者更适合解释``不要把自己困住''，后者更适合真正计算。

构造完成后也应检查答案，而不是只相信算法名称。若输出顶点序列

\[
v_0,v_1,\ldots,v_m,
\]

则相邻顶点必须对应原图中尚未被使用的某一条边，\(m\) 必须等于 \(|E|\)；闭合情形还要有 \(v_m=v_0\)。多重图中仅核对端点对不够，因为同一对端点之间可能有多条不同的边，程序或手算都应给边独立编号。这个检查是线性的，却能抓住漏边、重边和错误闭合三类常见失误。

还可以反过来理解握手定理为何总在本节出现。每条边有两个端点，所以奇度顶点总成偶数个；欧拉开放通路只容许其中两个充当边界。图内部所有进出都相互抵消，只有路线的边界留下失衡。这个``内部抵消、边界留下''的思想还会在流、散度与离散 Stokes 型公式中反复出现。

\subsection{不回到起点：恰有两个奇度顶点}

不要求闭合时，进出计数只放开起点与终点两处，所以奇度顶点个数必为 \(0\) 或 \(2\)——握手定理已保证它是偶数（见\hyperref[sec:04-Discrete-Mathematics/03-Graph-Theory/01]{``图的语言与度数''}）。

对应的定理是：无向图中所有非零度顶点属于同一连通分量时，它存在非闭合的欧拉通路，当且仅当奇度顶点恰有两个；此时通路的两个端点必为这两个奇度顶点，别无选择。

证明用一个``补边''技巧：在两个奇度顶点 \(s,t\) 之间添一条新边（相邻就再添一条平行边），所有度数变为偶数，新图有欧拉回路；从回路中删掉这条新边，剩下的正是从 \(s\) 到 \(t\) 的欧拉通路。反方向由端点计数直接得到。

完整图景因此是：

\begin{itemize}
\tightlist
\item
  奇度顶点 \(0\) 个：可以一笔画并回到起点，起点任意；
\item
  奇度顶点 \(2\) 个：可以一笔画，但起点与终点被钉死在这两处；
\item
  奇度顶点超过 \(2\) 个（必为至少 \(4\) 个的偶数）：无论如何起笔都不可能。
\end{itemize}

Königsberg 有四块奇度陆地，落在第三类：连``不要求回家''的走法也不存在。

这也给出一套不会漏条件的判断顺序：

\begin{enumerate}
\tightlist
\item
  先忽略孤立点，检查所有含边顶点是否连通；若不连通，一支笔不可能从一个含边分量跳到另一个；
\item
  再数奇度顶点；数量为 \(0\) 时有闭合回路，为 \(2\) 时有开放通路，更多时不存在；
\item
  若要构造，闭合情形从任一非零度顶点出发，开放情形必须从两个奇度顶点之一出发。
\end{enumerate}

只检查奇偶而忘记连通性会得到假结论：两个互不相连的三角形中每个顶点度数都是 \(2\)，却不可能用一条连续路线走遍两个分量。只检查连通性也不够，Königsberg 正是连通但奇偶失败。

\subsection{有向图里，``进出配对''变成入度等于出度}

若道路是单行的，边有方向，度数奇偶便不再是正确语言。闭合行程每进入顶点一次就要沿有向边离开一次，因此必要条件变为

\[
\deg^+(v)=\deg^-(v)
\qquad\text{对每个顶点 }v.
\]

此外，所有非零度顶点必须落在同一个强连通结构中：沿边方向既能从一处走到另一处，也能回来。更精确地说，删去孤立点后，有向图存在欧拉回路，当且仅当每点入度等于出度，并且所有含边顶点属于同一个强连通分量。

开放的有向欧拉通路允许两个端点失衡：起点满足 \(\deg^+=\deg^-+1\)，终点满足 \(\deg^-=\deg^++1\)，其余顶点入度等于出度；再配合相应的可达性条件即可。无向图中的``两个奇点''在有向图里被翻译成``一处多出一次离开，一处多出一次进入''。本质没有变，仍是在给每次进入寻找配对的离开。

\subsection{应用：巡视、布线，以及必须重复时}

欧拉回路回答``能不能不重复''。街道巡视、铲雪车路线、电路板走线、展馆参观流线，都可以先检查连通性与度数奇偶，答案立即得出，无需任何搜索。

现实更常遇到判据失败的情形：邮递员必须把每条街道至少走一遍，并希望总路程最短。度数不全为偶时，重复不可避免，问题变成``至少重复哪些边''。这就是中国邮路问题，因管梅谷 1962 年的研究得名。核心观察是：把一条边多走一遍，等价于给它的两个端点各添一条平行边；要让全图度数变偶，重复的边恰好要把奇度顶点两两配对。于是最优方案是：在全部奇度顶点之间求一组配对，使配对两端最短路长度之和最小，再把相应的最短路各重复一遍。奇偶计数再一次把``走法''问题化归成纯粹的图上计算。

这里也要看清模型边界。欧拉模型要求``服务的是边''：每条街、每段管线或每条通信链路都要被覆盖。若任务是访问每个客户一次，而客户在顶点上，欧拉回路就答错了问题；若道路必须重复且边权可能表示时间、距离或风险，单纯数重复边数也不等于最小化真实成本；若道路有方向，还必须使用有向版本的平衡条件。

一个实用的建模检查是先问：不可遗漏的对象究竟是边还是顶点？一旦答案是边，度数与进出配对往往是正确入口；一旦答案是顶点，下一节的哈密顿困难便会出现。
