信息一旦离开发送者，就可能被噪声改变、被网络丢弃，或者只剩下若干不完整的片段。最直接的办法是重复发送，但``多抄几遍''只制造了冗余，并没有解释冗余应当怎样组织，也没有告诉接收者哪一份可信。

编码理论研究的是一个更精确的问题：

\begin{quote}
怎样以尽可能少的额外符号，使接收者能够发现错误、定位错误，甚至恢复原始信息？
\end{quote}

这条问题线索会把离散数学中的几个看似分散的主题连接起来。有限域提供可逆的离散算术；多项式插值把少量信息扩展成许多彼此制约的取值；Hamming 距离把``能纠正多少错误''变成几何问题；稀疏图则允许接收者在巨大码字空间中进行局部、迭代的推断。

建议按下面的顺序阅读：

\begin{enumerate}
\tightlist
\item
  有限域与多项式插值：先建立后续编码真正依赖的代数工具。
\item
  码距决定检测与纠错能力：理解所有纠错码共同的几何原理。
\item
  Reed--Solomon 编码：看多项式怎样把这个原理变成可计算的方案。
\item
  稀疏图码与消息传递：当最优解码太昂贵时，怎样利用局部结构逼近它。
\item
  Fountain 码与无速率传输：当主要困难从``符号被改坏''变成``数据包不知丢了哪些''时，编码思路怎样改变。
\end{enumerate}

这几节并不试图覆盖全部现代编码理论。主线是让读者形成三个可以迁移的判断：

\begin{itemize}
\tightlist
\item
  冗余的价值不在数量，而在不同符号之间形成了什么约束；
\item
  能否纠错首先由码字之间的最小距离决定，解码算法只是实现这种能力的方式；
\item
  不同信道需要不同结构：错误、擦除和软噪声不能被当成同一种问题。
\end{itemize}

学完后，可以继续阅读信息论部分的\hyperref[ch:058]{``有噪信道编码：噪声中为什么仍能可靠通信''}，在那里我们会问另一个层次的问题：可靠传输的极限究竟在哪里，而不只是某个具体码怎样工作。

\subsection{怎么读这一章}

不必每次都从有限域顺读到 Fountain 码。可以先按眼前的问题选择入口：

\begin{itemize}
\tightlist
\item
  若想知道``冗余为什么真的能够纠错''，读\hyperref[sec:04-Discrete-Mathematics/05-Coding-and-Polynomial-Methods/02]{``码距决定检测与纠错能力''}；先抓住 Hamming 球互不相交这幅图景，再决定是否回补线性码公式。
\item
  若想看一个确定、可手算的代数方案，依次读\hyperref[sec:04-Discrete-Mathematics/05-Coding-and-Polynomial-Methods/01]{``有限域与多项式插值''}与\hyperref[sec:04-Discrete-Mathematics/05-Coding-and-Polynomial-Methods/03]{``Reed--Solomon 编码''}；主线只是``低次多项式不能拥有太多根''。
\item
  若关心现代长码怎样在现实成本下解码，直接读\hyperref[sec:04-Discrete-Mathematics/05-Coding-and-Polynomial-Methods/04]{``稀疏图码与消息传递''}；其中的 Tanner 图建立在\hyperref[ch:029]{``图论：局部邻接怎样形成全局网络''}的二分图语言上。
\item
  若问题是丢包率未知、广播反馈昂贵，读\hyperref[sec:04-Discrete-Mathematics/05-Coding-and-Polynomial-Methods/05]{``Fountain 码与无速率传输''}；这里处理的是已知丢失位置的擦除，而不是未被发现的篡改。
\end{itemize}

第一次阅读可以在能回答下面的问题时停下：信道究竟造成错误、擦除还是软噪声？当前结论描述的是存在性边界、确定性保证还是高概率表现？把这两件事说清，比记住具体码名更重要。
