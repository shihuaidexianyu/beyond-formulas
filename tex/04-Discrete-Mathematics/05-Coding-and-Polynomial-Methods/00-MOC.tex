信息一旦离开发送者，就可能被噪声改变、被网络丢弃，或者只剩下若干不完整的片段。最直接的办法是重复发送，但``多抄几遍''只制造了冗余，并没有解释冗余应当怎样组织，也没有告诉接收者哪一份可信。

编码理论研究的是一个更精确的问题：

\begin{quote}
怎样以尽可能少的额外符号，使接收者能够发现错误、定位错误，甚至恢复原始信息？
\end{quote}

这条问题线索会把离散数学中的几个看似分散的主题连接起来。有限域提供可逆的离散算术；多项式插值把少量信息扩展成许多彼此制约的取值；Hamming 距离把``能纠正多少错误''变成几何问题；稀疏图则允许接收者在巨大码字空间中进行局部、迭代的推断。

建议按下面的顺序阅读：

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  有限域与多项式插值：先建立后续编码真正依赖的代数工具。
\item
  码距决定检测与纠错能力：理解所有纠错码共同的几何原理。
\item
  Reed--Solomon 编码：看多项式怎样把这个原理变成可计算的方案。
\item
  稀疏图码与消息传递：当最优解码太昂贵时，怎样利用局部结构逼近它。
\item
  Fountain 码与无速率传输：当主要困难从``符号被改坏''变成``数据包不知丢了哪些''时，编码思路怎样改变。
\end{enumerate}

这几节并不试图覆盖全部现代编码理论。主线是让读者形成三个可以迁移的判断：

\begin{itemize}
\tightlist
\item
  冗余的价值不在数量，而在不同符号之间形成了什么约束；
\item
  能否纠错首先由码字之间的最小距离决定，解码算法只是实现这种能力的方式；
\item
  不同信道需要不同结构：错误、擦除和软噪声不能被当成同一种问题。
\end{itemize}

学完后，可以继续阅读信息论部分的``有噪信道编码''，在那里我们会问另一个层次的问题：可靠传输的极限究竟在哪里，而不只是某个具体码怎样工作。
