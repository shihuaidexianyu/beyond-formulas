\section{稀疏图码与消息传递}

最小距离告诉我们可靠纠错在原则上可能，最大似然解码则告诉我们理论上应该选择哪个码字：给定接收结果 $y$，寻找使 $P(y\mid c)$ 最大的合法码字 $c$。但一个长度几千、信息位几百的线性码可能拥有天文数量的码字。逐个比较虽然定义正确，却不是可运行的方法。

LDPC 码选择了另一条路：它把合法性拆成许多简单、稀疏、彼此重叠的局部校验，避免使用少数复杂的全局条件。于是一次巨大的搜索被改造成节点之间反复交换局部证据。

\subsection{稀疏校验矩阵就是一张图}

低密度奇偶校验码（LDPC code）由一个稀疏的校验矩阵 $H$ 定义：

\[
Hc^\mathsf T=0\pmod2.
\]
``低密度''表示矩阵中绝大多数元素为零，每个校验方程只涉及少量变量，每个变量也只参与少量校验。

把 $H$ 画成 Tanner 图：

\begin{itemize}
\tightlist
\item
  每一列对应一个变量节点，即一个码字符号；
\item
  每一行对应一个校验节点，即一个奇偶约束；
\item
  若 $H_{ji}=1$，就在校验 $j$ 与变量 $i$ 之间连边。
\end{itemize}

例如约束

\[
c_1+c_2+c_4=0,\qquad
c_2+c_3+c_4=0
\]
会形成两个校验节点。$c_2,c_4$ 同时连接到两个约束，因此一个局部判断可以经它们传播到图的另一部分。

图不是矩阵的装饰性画法。它暴露了算法可以利用的条件独立结构：每个节点只需与邻居通信，不必直接面对整个码字空间。

\subsection{硬判决为什么丢掉了有用信息}

假设信道输出一个实数。收到 (+0.1) 和 (+5) 时，硬判决都可能把它们解释为比特 $0$，但两者置信度显然不同。若立刻压成 $0/1$，解码器会丢失``前者非常可疑、后者几乎确定''的信息。

软判决常用对数似然比表示：

\[
L_i=\log\frac{P(c_i=0\mid y_i)}{P(c_i=1\mid y_i)}.
\]
$L_i$ 的符号表示更偏向哪个比特，绝对值表示信心强弱。消息传递算法在节点间交换可累积的证据，最终答案要等这些证据汇总后才能形成。

\subsection{一轮消息传递在做什么}

在变量节点到校验节点的方向，变量 $c_i$ 汇总信道证据以及来自其他校验的消息，然后告诉某个校验：``在不使用你这条消息的前提下，我目前更相信自己是什么。''

在校验节点到变量节点的方向，某个奇偶约束查看其他相邻变量的意见，推断要使约束成立，目标变量更可能取什么值。

这种排除接收方自身旧消息的做法叫传递外在信息。若把对方刚告诉自己的内容原样放大再返还，图中会迅速形成虚假的自我强化。

以擦除信道为例，消息甚至不需要概率。若校验

\[
c_1+c_2+c_3=0
\]
中 $c_1=1,c_2=0$ 已知而 $c_3$ 擦除，那么校验节点立刻推出 $c_3=1$。新恢复的变量又可能让另一个校验只剩一个未知量，这样便产生剥离式连锁恢复。

\subsection{为什么树上精确、有环时只是近似}

若 Tanner 图是一棵树，从叶子向内、再向外传递消息，可以在有限轮内汇总互不重复的证据，所得边缘概率是精确的。这与动态规划在树上有效是同一个原因：切断一条边后，两侧真正条件独立。

实际高性能码不可能是一棵有限树，因为树会有大量低度叶节点，也难以形成需要的码率与距离。Tanner 图必然含环。此时同一份证据可能绕一圈后以``新消息''的面貌返回，独立性假设不再严格成立，所以 belief propagation（置信传播）是近似算法。

长环通常问题不大，因为在前几轮中局部邻域看起来仍像树；短环则让证据过早重复。某些小型错误结构还会形成 trapping set（陷阱集）：每个局部节点都得到彼此支持的错误意见，使迭代停在非码字或错误码字附近。

因此 LDPC 设计不只是随机摆放一些 $1$。行列度分布、短环数量、图扩张性和具体信道都会影响性能。

\subsection{Turbo 码的共同思想}

Turbo 码通常把两个卷积码通过交织器连接。交织器把同一批信息位打乱顺序，使两个组成解码器从不同邻域观察它们。两个解码器反复交换外在信息：一个解码器难以判断的位，可能在另一个排列下得到强证据。

它与 LDPC 的具体图结构不同，但教学上应把它们放在一起理解。二者都通过以下局部计算避免直接枚举全局码字：

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  把全局约束分解成局部组成部分；
\item
  在局部计算软证据；
\item
  反复交换尚未被对方使用的信息；
\item
  期望迭代逐步逼近全局一致解释。
\end{enumerate}

``Turbo''的性能来自交织、软输入软输出解码与迭代外在信息的共同作用，并不存在一个单独的神秘公式。

\subsection{接近最优不等于没有条件}

LDPC 和 Turbo 码能在许多信道上接近 Shannon 极限，但这句话有明确边界：

\begin{itemize}
\tightlist
\item
  性能依赖码长，短码未必表现出渐近优势；
\item
  解码器需要合理的信道概率模型；
\item
  迭代次数、量化精度和硬件结构会改变结果；
\item
  belief propagation 不是一般含环图上的最大似然保证；
\item
  误码平台常由 trapping sets 等有限长度结构造成。
\end{itemize}

这正体现了``理论能力''和``可计算实现''的区别：码距或信道容量给出边界，图结构和算法决定我们能否在现实成本内接近它。

下一节会换一个信道模型。若数据包要么完整到达、要么明确丢失，而且发送者事先不知道接收者会丢多少包，那么固定码率本身就成了麻烦。Fountain 码让编码器可以像水源一样不断产生新的编码包，直到接收者收够为止。

\subsubsection{自己补完}

考虑三个校验：

\[
x_1+x_2+x_3=0,\qquad
x_2+x_4=0,\qquad
x_3+x_4+x_5=0.
\]
已知 $x_1=1,x_2=0,x_5=1$，而 $x_3,x_4$ 擦除。

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  按``只剩一个未知量的校验优先''进行剥离；
\item
  写出每一步新恢复的变量；
\item
  若删去第二个校验，说明为何恢复会停住。
\end{enumerate}
