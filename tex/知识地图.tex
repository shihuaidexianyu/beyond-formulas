这本书不是一条只有单一起点和终点的先修链。它更像一张有主干、有汇合点、也允许回返的网络。为了先看清全貌，可以把十六个内容 Part 压缩成六层：

\begin{center}
对象与命题 $\longrightarrow$ 结构与变化 $\longrightarrow$ 不确定性与数据\\
$\longrightarrow$ 计算与选择 $\longrightarrow$ 学习与表示 $\longrightarrow$ 轨迹与行动
\end{center}

箭头表示最常见的理解方向，而不是不可逆的课程顺序。学习会不断沿箭头向前组合知识，也会在遇到失败时向后追查前提。

\subsection{真正重要的是汇合点}

单独知道每个 Part 的主题仍不等于看见体系。许多核心方法恰好出现在学科交界处：

\begin{itemize}
\item
  \textbf{线性模型}把线性代数的投影、概率模型的噪声、统计中的估计与优化中的目标函数合成同一个对象；
\item
  \textbf{反向传播}把微积分的链式法则、线性代数的伴随映射、矩阵微积分的微分记法与数值计算的稳定性连接起来；
\item
  \textbf{变分推断}同时使用概率模型、KL divergence、Monte Carlo 估计和优化，缺少其中任一层都容易把下界误当成 posterior 本身；
\item
  \textbf{Bellman 方程}把条件期望、Markov 结构、动态规划、不动点、优化与反馈控制压缩成一条递推关系；
\item
  \textbf{可信评价与统计决策}横跨统计推断、机器学习与现实行动，要求区分损失、风险、校准、覆盖、拒绝、分布偏移和真正关心的后果；
\item
  \textbf{因果推断}把概率图、随机试验、潜在结果与现实决策连接起来，要求把观察关联、干预分布、识别假设和有限样本估计逐层分开。
\end{itemize}

这些汇合点也是最值得回读前置章节的地方。若一个方法只能在所属章节中复述，却不能拆回它调用的数学结构，就还没有真正进入可迁移的知识网络。

\subsection{三条贯通路径}

若目标已经明确，可以先沿一条纵向路径建立主干，再补旁支。

\paragraph{机器学习路径。}
数学语言负责把任务与假设说清楚；线性代数和微积分提供表示与局部变化；概率论和数理统计处理噪声、估计与泛化证据；数值分析、凸优化和矩阵微积分把目标变成可靠算法。机器学习 Part 最终将这些部分重新合成“问题定义---数据---表示---目标---求解---评价”的闭环。

\paragraph{深度学习路径。}
这条路径建立在机器学习路径之上，但更依赖函数复合、Jacobian、伴随映射、随机优化和浮点稳定性。进入具体架构后，卷积会回到线性算子与对称性，注意力会回到矩阵运算与概率加权，生成模型会回到潜变量、信息量和近似推断。架构名称会变化，这些结构关系更稳定。

\paragraph{强化学习路径。}
概率论与随机过程先给出条件期望、Markov 性和长期分布；优化与数值迭代解释策略改进和不动点逼近；动力系统与控制说明状态、反馈、稳定性和模型已知时的最优决策。强化学习改变的不是最终数学目标，而是信息条件：转移规律未知，只能通过交互样本逐步逼近。

三条路径并非三个封闭专业。深度学习可以成为强化学习的函数逼近器，序列决策会反过来破坏机器学习常用的独立同分布假设，而统计与数值可靠性始终横穿其中。

\subsection{从知识地图回到一次实际判断}

面对新问题时，可以沿下面的循环定位所需知识：先确定对象、目标与可观察信息，再选择结构和概率假设；随后推导目标与算法，检查数值实现；最后用有限数据评价结论，并把发现的失败归因到定义、模型、求解、数据或决策层。若结果不可信，就沿地图退回相应层次修改，而不是只更换一个算法名称。

这也是全书最终希望形成的能力：不仅知道一个公式位于哪一章，还知道它在完整判断链中承担什么责任，以及链条断裂时应当回到哪里。
