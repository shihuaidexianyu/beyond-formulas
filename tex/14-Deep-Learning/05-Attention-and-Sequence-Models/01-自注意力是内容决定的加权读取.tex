\section{Self-Attention 是内容决定的加权读取}
\label{sec:14-Deep-Learning/05-Attention-and-Sequence-Models/01}

设序列表示为

\[
X\in\R^{T\times D}.
\]

若逐位置应用共享函数 \(h_t=f(x_t)\)，第 \(t\) 个输出无法随其他位置改变。若所有位置共享同一个全局平均，又无法针对当前问题选择不同来源。注意力的基本动作是：每个位置先提出查询，再与所有可读位置的键比较，最后按比较结果汇聚值。

\begin{quote}
\textbf{先抓住这一点：}注意力把固定的读取规则改成由当前内容产生的读取规则。Query 决定在找什么，Key 决定怎样匹配，Value 决定实际取回什么。
\end{quote}

\subsection{Query、Key、Value 分开匹配与读取}

单头自注意力使用三组线性投影：

\[
Q=XW_Q,\qquad K=XW_K,\qquad V=XW_V,
\]

其中

\[
W_Q,W_K\in\R^{D\times d_k},
\qquad
W_V\in\R^{D\times d_v}.
\]

于是 \(Q,K\in\R^{T\times d_k}\)，\(V\in\R^{T\times d_v}\)。分数矩阵

\[
S=\frac{QK^{\trans}}{\sqrt{d_k}}
\in\R^{T\times T}
\]

的元素 \(S_{ts}\) 衡量第 \(t\) 个查询与第 \(s\) 个键的匹配。沿每一行对可读位置 \(s\) 做 Softmax：

\[
A_{ts}=\frac{\exp(S_{ts})}{\sum_{r=1}^{T}\exp(S_{tr})},
\qquad
Z=AV.
\]

因此

\[
z_t=\sum_{s=1}^{T}A_{ts}v_s
\]

是值向量的凸组合。权重由 Query 与 Key 决定，实际被取回的内容由 Value 决定。把三者分开，允许模型用一种特征判断相关性，用另一种特征传递信息。

\subsection{为什么除以 \texorpdfstring{\(\sqrt{d_k}\)}{sqrt(dk)}}

若 \(q_t\) 与 \(k_s\) 的各坐标近似独立、均值为零、方差为一，则点积

\[
q_t^{\trans}k_s=\sum_{j=1}^{d_k}q_{tj}k_{sj}
\]

的方差约为 \(d_k\)。维度增大时，未缩放分数的典型绝对值按 \(\sqrt{d_k}\) 增长，Softmax 更容易进入接近 one-hot 的饱和区，梯度集中在极少位置。除以 \(\sqrt{d_k}\) 使分数量级在维度变化时较稳定。

这是一条初始化尺度解释，不是对所有训练后分布都精确成立的定理。归一化、权重尺度和数据相关性仍会改变实际分数。

\subsection{Softmax 的轴决定语义}

注意力必须对每个查询 \(t\) 沿键位置 \(s\) 归一化，使

\[
\sum_s A_{ts}=1.
\]

若误沿查询轴归一化，得到的是``每个键把质量分给哪些查询''，不再是每个输出位置从哪里读取。矩阵尺寸仍然合法，程序甚至可能训练，却已经改变了模型语义。

数值实现应按行减去最大分数：

\[
\softmax(s)_j
=\frac{\exp(s_j-m)}{\sum_r\exp(s_r-m)},
\qquad m=\max_rs_r.
\]

平移不改变 Softmax，却能避免大正数指数溢出。若整行都被 mask，则分母没有合法项；实现必须定义这种输入是否允许，而不能期待 Softmax 自动给出有意义答案。

\subsection{Mask 是可读关系，不是普通特征}

因果注意力要求位置 \(t\) 只能读取 \(s\le t\)。可把不允许的位置加上 \(-\infty\)：

\[
\widetilde S_{ts}=
\begin{cases}
S_{ts},&s\le t,\\
-\infty,&s>t.
\end{cases}
\]

Softmax 后被屏蔽位置权重为零。padding mask 则防止真实 token 读取补齐位置。两种 mask 都在规定图上的可达关系，却服务于不同信息边界。

手机输入法在补全下一个词时，只能读取已经键入的前缀。训练材料虽然是完整句子，但若当前位置可以偷看后面的词，模型在练习中像是每次都猜对，真正逐字输入时却失去这份答案。自回归语言模型里的因果 mask，正是把``尚未键入''的位置从当前查询能够读取的 Key 和 Value 中拿掉。

在病情时间序列中预测下一小时风险时，同一规则意味着未来化验结果不能进入当前位置。若因果 mask 与目标错开一个位置，模型可能在训练和随机切分验证中获得异常漂亮的 loss，因为答案已经沿注意力边泄漏进输入；逐步部署时未来信息消失，性能才突然崩塌。因果 mask 因而是在声明预测时刻真正可用的信息，而不只是为了得到一个三角矩阵。

有限精度实现常用很大的负数代替 \(-\infty\)。这个值必须与 dtype 匹配；过小可能仍留下非零概率，过大则可能在后续运算产生 NaN。mask 应在 Softmax 前作用于分数，而不是在归一化后把权重清零却不重新归一化。

\subsection{多头让不同子空间拥有不同读取规则}

多头注意力为每个头 \(h\) 使用独立投影，得到

\[
Z^{(h)}
=\softmax\left(
\frac{Q^{(h)}K^{(h)\trans}}{\sqrt{d_k}}
\right)V^{(h)}.
\]

各头输出拼接后再线性映射：

\[
Z=\operatorname{Concat}(Z^{(1)},\ldots,Z^{(H)})W_O.
\]

多头并不保证每个头学到可命名的语言关系；它只是允许多个低维匹配与读取机制并行存在。头数增加而总宽度固定时，每个头的维度会减小，表达分工与单头容量之间存在权衡。

\subsection{没有位置结构时，注意力不知道顺序}

设 \(P\) 是置换矩阵，把序列位置重新排列。若不加入位置相关信息，线性投影与注意力满足

\[
\operatorname{Attn}(PX)=P\operatorname{Attn}(X).
\]

原因是 \(Q,K,V\) 同时被 \(P\) 置换，而

\[
(PQ)(PK)^{\trans}=P(QK^{\trans})P^{\trans}.
\]

行 Softmax 和加权求和保留相同置换。模型知道内容集合及其相互关系，却不知道哪个 token 是第一个。位置编码、相对位置偏置、旋转位置结构或因果 mask 都是在不同方式下打破或限制这种对称。

因果 mask 提供``不能看未来''的偏序，但仅凭它不一定能表达精确距离；位置结构仍决定模型怎样区分相邻与遥远历史。

\subsection{全局读取的代价是二次矩阵}

分数矩阵含 \(T^2\) 个元素，时间和显存的主项通常随序列长度二次增长。自回归生成时，过去位置的 Key 与 Value 在每一步不会改变，可以缓存它们，避免重复计算历史投影；但新 Query 仍需与全部历史 Key 比较，因此单步读取成本随上下文长度增长。

稀疏、局部或低秩注意力试图减少这项代价，同时也限制了哪些位置能直接交互。是否值得交换，取决于任务所需的长程关系，而不是``更快''这一项指标。

\subsection{注意力权重不是天然解释}

\(A_{ts}\) 确实是当前层中 Value 的线性组合系数，但它不等于输入对最终预测的唯一因果贡献。后续层可以重新混合，Value 投影改变被传递的内容，不同参数也可能产生相近输出而权重不同。

因此注意力图可以帮助检查 mask、位置和读取模式，却不能单独证明模型依据某个词作出决定。结构正确、数值稳定、预测有效和解释可信仍是四种不同结论。
