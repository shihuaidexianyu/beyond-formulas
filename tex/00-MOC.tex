公式往往是推理的终点，却只是使用的起点。一个梯度更新式可以写在一行里，但这一行没有说明变量属于什么空间、目标函数为什么这样定义、导数在何处存在、浮点计算是否稳定，也没有说明训练数据能否支持部署时的结论。只记住这一行，能够复现一次计算；把它背后的结构重新展开，才可能把方法迁移到新的问题。

Beyond Formulas 不是要绕开公式。恰恰相反，它希望让公式恢复完整含义：公式压缩了哪些对象和关系，依赖哪些条件，由什么推理得到，怎样被可靠地计算，又需要什么证据才能用于判断和决策。全书反复追问六件事：

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  \textbf{对象是什么：}变量、函数、分布、空间与数据分别属于什么类型；
\item
  \textbf{结论为何成立：}定义、假设和推导怎样把前提连接到结论；
\item
  \textbf{结果能否算出：}算法是否收敛，误差是否放大，有限精度会改变什么；
\item
  \textbf{证据支持什么：}有限样本允许怎样的统计陈述，哪些解释超出了数据；
\item
  \textbf{边界在哪里：}条件改变、分布偏移或系统进入反馈回路时，方法怎样失效；
\item
  \textbf{结构是什么：}这个现象背后的低维结构、对称性、不变量是什么？为什么世界倾向于呈现这种结构？
\end{enumerate}

前五个问题处理``数学如何支撑计算与判断''，第六个问题处理``数学如何揭示世界的本质''。两者互补：知道怎么算，才能动手；知道为什么是这样，才能理解。AI 可解释性需要的正是这种双重能力——既会推导反向传播，也理解为什么深度网络能捕捉数据的低维结构。

因此，这里不把``数学基础''看成学完便可以离开的预备课。数学语言、空间结构、不确定性、数值计算、优化、学习与控制会在同一个实际问题中反复相遇；后面的应用也会迫使我们回到前面，重新检查一个量词、一种收敛或一次矩阵分解。

这套体系也不宣称提前收录未来需要的每一个主题。它提供的是一个可扩展骨架：遇到新知识时，能够判断它解决哪类困难，依赖哪些已有结构，应当接入哪条主线，以及它与相邻方法的区别。书后的边界可以继续向外生长，但判断知识位置的坐标系应当保持稳定。

接下来的``全书知识地图''解释各 Part 怎样组成这个坐标系；``怎样选择学习路线''则给出四种实际走法。前者回答``知识之间是什么关系''，后者回答``我现在应该从哪里进入''。
