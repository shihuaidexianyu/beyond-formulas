\section{消元与解的结构：从计算答案到看见约束}

方程组写成 \(Ax=b\) 后，困难并没有自动消失。我们仍然需要判断它是否相容、有哪些变量能够自由选择，以及怎样描述全部解。

Gaussian 消元的核心动作很朴素：用一个方程消去其他方程中的某个未知量。真正重要的是，每一步都只把原方程换成等价表达，因此不会改变解集；与此同时，独立约束、冗余约束和矛盾会逐渐显现出来。

\subsection{哪些操作不会改变解集}

对增广矩阵 \([A\mid b]\) 可以做三类初等行操作：

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  交换两行；
\item
  把一行乘以非零常数；
\item
  把一行的倍数加到另一行。
\end{enumerate}

每一种操作都可逆。交换可以再交换回来，非零倍缩放可以乘以倒数撤销，加上一行的倍数可以减去同样的倍数撤销。因此，新旧方程组具有完全相同的解。

行操作改变的是``用哪组方程描述约束''，不是约束本身。后面把行操作写成矩阵时，这一点会变成初等矩阵的可逆性。

\subsection{主元是新信息出现的位置}

从左到右处理矩阵时，在当前尚未处理的部分选择一个非零元素作为主元，用它把同列下方元素消为零。重复后得到行阶梯形。

例如

\[
\left(
\begin{array}{cc|c}
1&2&3\\
2&5&8
\end{array}
\right)
\]

用第二行减去第一行的两倍：

\[
\left(
\begin{array}{cc|c}
1&2&3\\
0&1&2
\end{array}
\right).
\]

第二行带来了一个新的独立约束 \(y=2\)；回代第一行得到 \(x=-1\)。两个未知量都对应主元，因此没有自由变量，解唯一。

主元个数就是矩阵的秩（这里以此作为秩的工作定义）。它衡量矩阵中真正独立的方向或约束数量，这与矩阵有多少行列是两个问题。

\subsection{自由变量从哪里来}

考虑

\[
\begin{aligned}
x+2y-z&=1,\\
2x+4y+z&=5,\\
-x-2y+2z&=0.
\end{aligned}
\]

增广矩阵为

\[
\left(
\begin{array}{ccc|c}
1&2&-1&1\\
2&4&1&5\\
-1&-2&2&0
\end{array}
\right).
\]

做

\[
R_2\leftarrow R_2-2R_1,
\qquad
R_3\leftarrow R_3+R_1,
\]

得到

\[
\left(
\begin{array}{ccc|c}
1&2&-1&1\\
0&0&3&3\\
0&0&1&1
\end{array}
\right).
\]

后两行表达同一个约束 \(z=1\)。第一列和第三列出现主元，第二列没有主元，所以 \(y\) 可以自由选择。令 \(y=t\)，则第一行给出

\[
x+2t-1=1,
\]

于是

\[
\begin{pmatrix}x\\  y\\  z\end{pmatrix} = \begin{pmatrix}2\\0\\1\end{pmatrix}
+t\begin{pmatrix}-2\\1\\0\end{pmatrix}.
\]
这个表达式比``有无穷多个解''提供了更多信息：第一项是一个特解，第二项描述了所有不改变右端的移动方向。

\subsection{为什么全部解是``特解加零空间''}

满足 \(Az=0\) 的全体向量构成 \(A\) 的零空间，记作 \(\nullsp(A)\)。它对加法和数乘封闭（\(A(z_1+z_2)=0\)，\(A(cz)=0\)），因此是一个穿过原点的子空间。

假设 \(x_p\) 是 \(Ax=b\) 的一个特解。若 \(z\in\nullsp(A)\)，那么

\[
A(x_p+z)=Ax_p+Az=b.
\]

所以 \(x_p+z\) 仍是解。反过来，若 \(x\) 也是一个解，则

\[
A(x-x_p)=Ax-Ax_p=b-b=0,
\]

因此 \(x-x_p\in\nullsp(A)\)。于是所有解恰好是

\[
x_p+\nullsp(A).
\]

非齐次解集通常是零空间的平移，因而一般不穿过原点，也就不构成子空间。这个结构会在微分方程、优化约束和统计模型中反复出现。

\subsection{矛盾怎样在消元中暴露}

把上例第三个方程的右端从 \(0\) 改为 \(1\)：

\[
-x-2y+2z=1.
\]

同样做 \(R_3\leftarrow R_3+R_1\)，第三行给出 \(z=2\)，第二行却给出 \(z=1\)。继续消元会出现

\[
\begin{pmatrix}0&0&0&|&1\end{pmatrix},
\]

也就是荒谬的等式 \(0=1\)。这说明 \(b\) 不在 \(A\) 的列空间中——这正是上一节列视角中可解条件 \(b\in\operatorname{Col}(A)\) 被违反的情形。

用秩表达，相容条件为

\[
\rank(A)=\rank([A\mid b]).
\]

若增广列制造了额外主元，右端包含了列空间无法合成的方向，系统无解。

\subsection{三种解情形怎样统一判断}

对 \(m\times n\) 系统 \(Ax=b\)，先判断是否出现矛盾行：

\begin{itemize}
\tightlist
\item
  出现 \([0\ \cdots\ 0\mid c]\) 且 \(c\ne0\)：无解；
\item
  没有矛盾，且每个未知量列都有主元：解唯一；
\item
  没有矛盾，但存在非主元列：有自由变量，因此有无穷多解。
\end{itemize}

齐次系统 \(Ax=0\) 永远至少有零解。若 \(A\) 的列数大于秩，就存在自由变量，也就存在非零齐次解。

这里最值得避免的误解是：方程数等于未知量数并不保证唯一解。方阵也可能有重复或矛盾的约束；唯一性取决于是否每列、每行都能找到主元。

\subsection{行阶梯形与简化行阶梯形}

Gaussian 消元只消去主元下方，得到行阶梯形（REF），随后通过回代求解。Gauss--Jordan 消元继续把主元归一化并消去上方元素，得到简化行阶梯形（RREF）。

给定矩阵的 RREF 唯一，普通 REF 则可能因主元选择不同而不同。手工研究解集时 RREF 很直观；实际求解大型方阵时，通常停在上三角形并回代，因为继续消去上方会增加计算量。

行操作保持方程解集，也保持行空间（各行张成的空间：行只被替换为彼此的可逆线性组合，张成不变），却不保持列空间中的具体列向量。它会改变列本身，因此不能从化简后矩阵直接拿出原矩阵列空间的基；只能利用主元位置回到原矩阵选择对应列。

\subsection{精确代数与浮点计算的差别}

在精确算术中，只要主元非零就可以使用。但在浮点计算中，用一个非常小的数作主元会产生很大的乘子，放大舍入误差。

例如，本来可以交换一行后使用大小接近 \(1\) 的主元，却坚持除以 \(10^{-16}\)，中间结果会达到 \(10^{16}\) 的量级，许多有效数字可能丢失。实际算法通常采用部分选主元：在当前列剩余候选中选择绝对值较大的元素换到主元位置。

因此，行交换不只是为了避开零，它还是数值稳定性的基本防线。下一节中的分解通常写成 \(PA=LU\)，其中 \(P\) 就记录这些交换。
