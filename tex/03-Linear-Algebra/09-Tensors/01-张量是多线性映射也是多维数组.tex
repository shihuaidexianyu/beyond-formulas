\section{张量是多线性映射，也是多维数组}

一段彩色视频可存成四维数组

\[
X\in\R^{T\times H\times W\times C},
\]

四个索引分别表示时间、纵向位置、横向位置和颜色通道。若把后三个索引合并，当然也能得到 \(T\times(HWC)\) 矩阵；若把时间和高度合并，又会得到另一张矩阵。数值没有丢，结构却取决于摊平方式。矩阵只知道``第几行、第几列''，不知道一列编号里混合了空间与颜色。

这会制造一个认知冲突：张量若只是``更多索引的数组''，那么换一种编号或坐标后，它还是不是同一个张量？要回答，必须先把坐标表格与它表示的多线性对象分开。

\subsection{从双线性型重新看矩阵}

设 \(V,W\) 为有限维实向量空间。一个双线性映射

\[
B:V\times W\to\R
\]

对每个输入分别线性：

\[
B(\alpha v_1+\beta v_2,w)
=\alpha B(v_1,w)+\beta B(v_2,w),
\]

并对第二个变量满足同样性质。选定 \(V\) 的基 \(e_1,\ldots,e_m\) 与 \(W\) 的基 \(f_1,\ldots,f_n\)，记录

\[
b_{ij}=B(e_i,f_j),
\]

就得到一个 \(m\times n\) 数组。若

\[
v=\sum_i x_i e_i,
\qquad
w=\sum_j y_j f_j,
\]

多线性立即推出

\[
B(v,w)
=\sum_{i,j}b_{ij}x_i y_j
=x^\trans [b_{ij}]y.
\]

矩阵 \([b_{ij}]\) 是 \(B\) 在所选基下的坐标表示。换基后数字会变，双线性映射 \(B\) 本身不变。二次型、内积和协方差结构都可以从这个角度理解。

\subsection{从两个输入推广到多个输入}

给定向量空间 \(V_1,\ldots,V_k\)，一个 \(k\) 阶协变张量可看成多线性映射

\[
\mathcal T:
V_1\times\cdots\times V_k
\longrightarrow\R.
\]

它对每一个输入单独线性。若第 \(r\) 个空间选基

\[
e^{(r)}_1,\ldots,e^{(r)}_{n_r},
\]

张量由所有基向量组合上的取值

\[
t_{i_1\cdots i_k}
=
\mathcal T\left(
e^{(1)}_{i_1},\ldots,e^{(k)}_{i_k}
\right)
\]

完全决定。这些分量排成形状

\[
n_1\times n_2\times\cdots\times n_k
\]

的多维数组。于是``张量是多线性映射''与``张量是多维数组''并不矛盾：前者是不依赖坐标的对象，后者是选基后保存它的分量。

工程文献还常把向量空间张量积

\[
V_1\otimes\cdots\otimes V_k
\]

中的元素直接称为张量。在有限维 Euclidean 空间中，内积把向量与线性泛函自然对应，两种语言可以互相翻译；若没有内积或进入微分几何，向量输入与对偶空间输入必须严格区分。本单元停在有限维实内积空间，不把这种方便误当成所有语境下的同一。

\subsection{阶与维度不是同一个数}

张量的阶（order）是索引方向的个数，维度则是每个方向可取多少个坐标：

\begin{itemize}
\tightlist
\item
  标量没有自由索引，可看作零阶张量；
\item
  向量有一个索引，是一阶张量；
\item
  矩阵有两个索引，是二阶张量；
\item
  \(T\times H\times W\times C\) 视频数组有四个索引，是四阶数据张量。
\end{itemize}

不要把阶与矩阵的秩（rank）混淆。一个三阶张量可能具有不同的 CP 秩；``三阶''只表示它有三个模（mode），没有说明信息复杂度。物理文献也常把 order 称为 rank，因此跨领域阅读时必须先确认术语。

\subsection{秩一张量由外积产生}

给定向量

\[
a^{(1)}\in\R^{n_1},\ldots,
a^{(k)}\in\R^{n_k},
\]

它们的外积

\[
\mathcal X
=
a^{(1)}\otimes\cdots\otimes a^{(k)}
\]

定义为

\[
x_{i_1\cdots i_k}
=
a^{(1)}_{i_1}\cdots a^{(k)}_{i_k}.
\]

它称为秩一张量，因为每个模的变化完全分离。三阶情形

\[
x_{ijk}=a_i b_j c_k
\]

可读成：第一个方向只由 \(a\) 控制，第二个方向只由 \(b\) 控制，第三个方向只由 \(c\) 控制，三者通过乘法组合。

矩阵的秩一形式 \(uv^\trans\) 正是二阶外积 \(u\otimes v\)。SVD

\[
A=\sum_r\sigma_r u_rv_r^\trans
\]

因此已经是一种``秩一成分之和''；高阶张量分解会保留这个想法，却失去奇异向量正交、奇异值排序等部分幸运结构。

\subsection{模乘法：只沿一个索引做线性变换}

设三阶张量 \(\mathcal X\in\R^{I\times J\times K}\)，矩阵 \(M\in\R^{P\times I}\) 只作用在第一个模，定义

\[
\mathcal Y=\mathcal X\times_1 M,
\qquad
y_{pjk}=\sum_{i=1}^{I}m_{pi}x_{ijk}.
\]

第二、第三个索引保持不动。类似地可以定义 \(\times_2N\)、\(\times_3P\)。多个模上的变换彼此独立：

\[
\mathcal X
\times_1 M
\times_2 N
\times_3 P
\]

表示分别换三个方向的坐标或压缩三个方向。矩阵乘法只有一个输入索引与一个输出索引；模乘法明确说线性映射作用在哪一条轴上。

例如视频的颜色通道可沿第四色模乘一个 \(3\times3\) 矩阵做颜色变换，而不混合时间与空间位置；沿时间模乘 Fourier 或小波变换，则只重组帧序列。保留模的身份，使变换的含义可追踪。

\subsection{缩并：让一对索引相加消失}

矩阵乘法

\[
C_{ik}=\sum_j A_{ij}B_{jk}
\]

可看成沿共同索引 \(j\) 的缩并（contraction）：两个二阶张量各拿出一个索引配对求和，结果仍有两个自由索引。内积

\[
\ip{x}{y}=\sum_i x_i y_i
\]

则把两个一阶张量的唯一索引缩并掉，得到标量。

对三阶张量，固定或求和某个索引会产生不同对象。例如

\[
M_{ij}=\sum_k x_{ijk}
\]

沿第三模聚合成矩阵；若另有向量 \(c\)，

\[
M_{ij}=\sum_k x_{ijk}c_k
\]

是张量与向量沿第三模的缩并。Einstein 求和约定会把重复索引默认求和，但初学时显式写出求和更能防止把自由索引与被缩并索引混淆。

\subsection{换基时，分量必须成组变化}

双线性型的矩阵换基不是逐元素变换。若同一空间的新基坐标满足 \(x=Sx'\)，则

\[
B(v,w)=x^\trans Ay
=(x')^\trans S^\trans ASy',
\]

所以新坐标矩阵为

\[
A'=S^\trans AS.
\]

三阶协变张量在三个基变换 \(S,R,U\) 下，分量按

\[
t'_{abc}
=
\sum_{i,j,k}
s_{ia}r_{jb}u_{kc}\,t_{ijk}
\]

联动变化。公式的具体转置方向取决于基矩阵约定，但不变原则是：每个索引都必须按自己的基变换一次。只旋转数组的一条轴、却声称整个几何对象未变，通常是不完整的。

这正是``任意多维数组''与``张量''之间的边界。数据处理中人们常宽松地把任何多维数组叫张量，因为计算接口只关心形状；数学上，一个数组要表示不依赖坐标的张量，还必须说明每个索引代表哪个空间，以及换基时怎样变换。

\subsection{对称性减少独立分量}

若三阶张量满足

\[
t_{ijk}=t_{jik},
\]

它在前两个输入交换时不变，称在这两个指标上对称；若交换任意两个指标都不变，则为完全对称张量。三阶导数

\[
\frac{\partial^3 f}
{\partial x_i\partial x_j\partial x_k}
\]

在足够光滑时就是完全对称的三阶数组。反对称张量则在交换指标时变号，行列式与有向体积和这种结构相连。

对称性不是排版方便，而是减少独立自由度并改变适合的分解。下一篇先处理一般数据张量的 CP 与 Tucker 分解；若已知对称、非负、稀疏或守恒结构，分解时还应把这些条件显式保留，不能期待无约束算法自动发现。
