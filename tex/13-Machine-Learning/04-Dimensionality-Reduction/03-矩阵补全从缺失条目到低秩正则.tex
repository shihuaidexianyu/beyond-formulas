\section{矩阵补全从缺失条目到低秩正则}
\label{sec:13-Machine-Learning/04-Dimensionality-Reduction/03}

\begin{quote}
\textbf{先抓住这一点：}矩阵补全能从少量条目恢复其余部分，是因为假设矩阵近似低秩且观测位置足够分散。缺失机制偏向特定用户或项目时，低秩本身并不能保证可恢复。
\end{quote}

推荐系统中的用户---物品矩阵大多是空的。我们知道少数评分，却想预测尚未发生的评分。若完整矩阵近似低秩，似乎可以借助潜在偏好恢复空白；但``低秩''本身远远不够。

先看最直接的反例：如果某个用户一条评分都没有，那么无论其他行多完整，都无法仅由矩阵判断这个用户喜欢什么。甚至在一个 \(2\times2\) 矩阵中只观察对角线，许多秩一矩阵也能匹配相同观测：

\[
\begin{pmatrix}1&2\\2&4\end{pmatrix}
\quad\text{与}\quad
\begin{pmatrix}1&1\\4&4\end{pmatrix}
\]

都是秩一矩阵，对角线完全相同，未观测位置却各不相同。算法不能从不存在的信息中恢复唯一答案。

\subsection{从秩最小化到核范数}

令真实矩阵为 \(M\)，观测位置集合为 \(\Omega\)，\(P_\Omega\) 只保留这些位置的条目、其余位置置零。理想化的无噪声问题是

\[
\min_X\rank(X)
\quad\text{s.t.}\quad
P_\Omega(X)=P_\Omega(M).
\]

秩是离散且非凸的，直接优化通常困难。常见凸松弛把秩换成核范数

\[
\|X\|_*=\sum_j\sigma_j(X),
\]

得到

\[
\min_X\|X\|_*
\quad\text{s.t.}\quad
P_\Omega(X)=P_\Omega(M).
\]

在有噪声时，记 \(Y\) 为实际观测到的带噪数据（只在 \(\Omega\) 上有条目），更常写为

\[
\min_X
\frac12\|P_\Omega(X-Y)\|_F^2+\lambda\|X\|_*.
\]

核范数是奇异值的 \(\ell_1\) 范数，因而像 Lasso 令系数稀疏一样，鼓励奇异值变为零。它的近端算子正是对奇异值作软阈值：先对矩阵作 SVD，把每个 \(\sigma_j\) 变为 \((\sigma_j-\tau)_+\)（\(\tau\) 为阈值参数，\((\cdot)_+\) 表示取正部），再重组矩阵。

另一条路线直接写 \(X=UV^{\trans}\)，优化

\[
\frac12\|P_\Omega(UV^{\trans}-Y)\|_F^2
+\frac{\lambda}{2}
\left(\|U\|_F^2+\|V\|_F^2\right).
\]

这种因子化方法更适合大规模计算，却需要预先选择潜在秩，而且目标对 \(U,V\) 非凸。即便 \(X\) 能较好估计，因子本身也不唯一：对任何可逆矩阵 \(R\)，\(UR\) 与 \(VR^{-\top}\) 可产生同一个乘积。

\subsection{可恢复性需要结构与观测共同配合}

精确恢复定理通常同时要求：

\begin{itemize}
\tightlist
\item
  矩阵确实足够低秩；
\item
  奇异向量不过度集中在少数坐标上，即具备某种 incoherence；
\item
  观测足够多且在行列中分布较均匀；
\item
  噪声或误差满足相应控制条件。
\end{itemize}

为什么需要 incoherence？若秩一矩阵只有左上角一个巨大非零条目，那么除非恰好观察该位置，低秩不能告诉我们它存在。低秩描述全局自由度，incoherence 则保证信息没有藏在极少数条目里。

推荐数据还面临更根本的问题：缺失通常不是随机的。用户更可能给喜欢或极不喜欢的内容评分，平台又会先展示算法认为合适的物品。于是``未观测''本身包含选择机制。矩阵补全可以在某种分布下预测缺失条目，却不能仅凭低秩假设保证这些预测是用户在随机展示下的无偏偏好。

\subsection{验证方式必须模拟真正缺失的方式}

把所有已观测评分随机拆分为训练和测试，可能让同一用户和同一物品在两边大量出现，于是只测到``熟悉用户对熟悉物品''的插值能力。若部署关心新用户、新物品或未来时期，应按用户、物品或时间构造相应切分。

一次可信的矩阵补全分析至少包括：

\begin{enumerate}
\tightlist
\item
  先画观测 mask，检查整行、整列缺失和曝光偏差；
\item
  与全局均值、用户偏置和物品偏置等简单基线比较；
\item
  在训练观测上调节秩或 \(\lambda\)，在符合部署情境的验证集上评估；
\item
  分层报告活跃用户、冷启动用户、热门物品和长尾物品误差；
\item
  用合成低秩矩阵检验实现能否在已知条件下恢复，并改变 mask 观察敏感性。
\end{enumerate}

如果某些行没有任何信息，合理结论可能是``不可由当前矩阵识别，需要侧信息或主动收集''，而不是继续增加优化轮数。

\subsubsection{反例构造}

给定只观察第一行的 \(3\times3\) 矩阵，构造两个不同的秩一补全，使它们在观测位置完全一致、未观测位置却差异很大。然后逐步增加分散于各行各列的观测，思考哪些自由度被排除。最后比较随机按条目切分与按用户留出的验证误差，并解释两者分别回答什么问题。
