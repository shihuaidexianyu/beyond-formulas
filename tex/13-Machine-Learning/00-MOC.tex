前面的学科分别讨论向量空间、概率、优化、数值误差、统计推断和信息量。真实建模却不会按课程边界出题。一个模型可能在纸面上有漂亮的最优解，却因矩阵病态而算不准；可能在验证集上预测良好，却没有可解释的因果含义；也可能优化完全收敛，却只是在精确拟合一个错误的损失函数。

机器学习的任务，就是把这些原本分开的判断重新放到同一条问题链上，并用有限数据选择一条能在新对象上继续工作的规则。

\subsection{一个模型必须连续回答的六个问题}

首先要明确希望改变或预测什么，数据又是怎样被观察到的。这个阶段决定样本单位、部署分布和评价目标，不能由算法代替。

接着要选择表示。线性代数告诉我们特征张成什么空间、哪些方向重复；降维与低秩建模进一步追问高维观测是否由少数潜在方向产生。

第三步是写出模型和损失。概率论区分联合、条件与边缘分布；线性模型展示平方误差、Bernoulli 似然和正则化如何从假设中产生；概率模型与隐变量则处理标签或状态无法直接观察的情形。

第四步是推断或优化。凸优化说明何时局部信息足以保证全局结论，矩阵微积分把复杂函数的梯度传回参数，采样与近似推断面对解析计算不可得的 posterior 和期望。

第五步是确认计算结果值得相信。数值分析区分问题本身的敏感性与算法误差；深度学习进一步把 shape、反向传播、随机优化和浮点稳定性放进很深的复合函数。

最后才是统计评估。数理统计告诉我们估计量在重复抽样中怎样波动，数据切分与重抽样又在估计什么；信息论学习则从概率编码、最大熵、描述长度与表示压缩的角度审视目标。

这六步并非一次性流水线。残差结构可能迫使我们重新选择表示，数值不稳定可能暴露不可识别性，部署误差也可能说明训练分布或损失定义错了。建模是带反馈的推理过程。

\subsection{怎么读这一章}

若希望建立一条完整路线，应先读\hyperref[ch:109]{``学习问题与可信评价''}，固定数据、目标、风险和评价纪律，再从\hyperref[ch:110]{``线性模型''}观察一条透明的建模链。之后不必按目录顺序逐章读完，可按正在回答的问题选择路线：

\begin{itemize}
\tightlist
\item
  \textbf{预测与数据几何：}\hyperref[ch:111]{``树、集成与邻域''}（局部分区与度量） \(\rightarrow\) \hyperref[ch:112]{``降维与低秩建模''}（表示与重构） \(\rightarrow\) \hyperref[ch:113]{``聚类与数据几何''}（无标签分区）；
\item
  \textbf{潜变量与不确定性：}\hyperref[ch:114]{``概率模型与隐变量''}（生成假设） \(\rightarrow\) \hyperref[ch:115]{``概率图与序列模型''}（条件独立） \(\rightarrow\) \hyperref[ch:116]{``采样与近似推断''}（目标量无法解析计算）；
\item
  \textbf{函数空间与泛化：}\hyperref[ch:117]{``核方法与 Gaussian process''}（相似性与函数先验） \(\rightarrow\) \hyperref[ch:118]{``统计学习理论''}（样本外风险） \(\rightarrow\) \hyperref[ch:119]{``信息论学习''}（编码、约束与复杂度）；
\item
  \textbf{从观察到行动：}\hyperref[ch:120]{``因果推断''}先检查干预目标能否识别；若模型进一步涉及能量、配分函数或平均场，再读\hyperref[ch:121]{``统计物理与自由能''}。
\end{itemize}

第一次阅读的最小目标不是记住所有算法，而是对每章能说出三件事：它加入了什么结构假设，它用什么证据选择或拟合，以及哪种分布变化、不可识别性或计算误差会使结论失效。推导细节可在真正调用时回补。

也可以按问题进入：

\begin{itemize}
\tightlist
\item
  特征共线、模型系数不稳定：从\hyperref[ch:110]{``线性模型''}的正则化开始；
\item
  高维数据需要压缩、去噪或补全：进入\hyperref[ch:112]{``降维与低秩建模''}；
\item
  需要局部规则、非线性分区或集成：进入\hyperref[ch:111]{``树、集成与邻域''}；
\item
  没有标签却希望寻找分组结构：进入\hyperref[ch:113]{``聚类与数据几何''}；
\item
  类别或时间状态不可见：进入\hyperref[ch:114]{``概率模型与隐变量''}与\hyperref[ch:115]{``概率图与序列模型''}；
\item
  积分、posterior 或抽样分布无法直接得到：进入\hyperref[ch:116]{``采样与近似推断''}；
\item
  深度模型能运行却无法解释梯度或训练故障：继续进入\hyperref[ch:122]{``深度学习''}；
\item
  想理解交叉熵、复杂度、压缩与表示之间的关系：进入\hyperref[ch:119]{``信息论学习''}；
\item
  模型已经输出概率，却不知道怎样设阈值、拒绝或给出覆盖保证：回到\hyperref[ch:092]{``统计决策与可信预测''}；

  想判断一个预测关系能否支持干预或政策选择：进入\hyperref[ch:120]{``因果推断''}，先区分关联、识别与估计。
\end{itemize}

\subsection{贯穿全篇的判断边界}

在每个应用中，都应把下面几种``正确''分开：

\begin{itemize}
\tightlist
\item
  公式正确：推导在所写假设下成立；
\item
  数值正确：算法以足够精度求出了目标量；
\item
  统计正确：抽样、验证和不确定性分析与数据生成机制相符；
\item
  决策正确：指标与真实行动成本一致；
\item
  解释正确：预测相关性没有被夸大为因果或潜变量的唯一现实含义。
\end{itemize}

机器学习是一次回到第一性问题的练习：我们究竟观察到了什么，额外假设了什么，计算了什么，又凭什么相信结论能用于尚未见到的世界。算法名称只是这套问题中的工具索引。若目标从预测转向主动改变世界，还必须把观察规律、干预机制与反事实目标分开。

\subsection{一种贯穿视角：学习中的低维结构}

许多成功的学习方法都在利用某种有效低维性：数据虽复杂，与当前任务相关的变化、参数或表示却可能由较少的自由度控制。这是一类有力的结构假设，而不是所有机器学习问题的定义。它体现在：

\begin{itemize}
\tightlist
\item
  \textbf{充分统计量}：在特定参数模型中，某些统计量可以包含样本中关于参数的全部信息（Part 11），使原始数据可被压缩而不损失该模型下的参数信息；
\item
  \textbf{流形假设}：某些高维数据可能集中在低维流形附近（Part 12），但是否存在、维数多少以及任务是否只依赖该流形，都需要额外证据；
\item
  \textbf{信息瓶颈}：它把表示学习写成压缩输入与保留任务相关信息之间的权衡（Part 07 的互信息）；这是一种建模视角，不自动构成对泛化的保证；
\item
  \textbf{泛化的有效复杂度}：若任务相关规律能由较小的函数类、较低秩表示或较短描述捕捉，有限样本才更可能约束未见输入上的行为；但结论仍取决于数据分布、学习算法与复杂度度量。
\end{itemize}

因此，低维性应当被当作可检验的归纳偏置：它可以解释方法为何有效，也必须允许数据证明自己并不低维、低维结构与目标无关，或部署环境改变了原有结构。
