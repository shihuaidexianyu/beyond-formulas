\section{图结构把联合分布拆成局部因子}
\label{sec:13-Machine-Learning/07-Graphical-and-Sequential-Models/01}

\begin{quote}
\textbf{先抓住这一点：}图模型用局部因子限制高维联合分布，并让变量消元复用中间结果。图本身不会自动带来稀疏、可识别或因果语义。
\end{quote}

\(d\) 个离散变量若各有 \(K\) 个取值，任意联合分布需要记录 \(K^d-1\) 个自由概率。维度稍大就无法存储，更不用说由有限数据可靠估计。概率图模型用条件独立把联合分布限制为局部因子的乘积；图只是表达这种结构的工具，本身不会减少复杂度。

\subsection{Bayesian network 用有向无环图规定生成顺序}

设有向无环图中节点 \(X_i\) 的父节点集合为 \(\operatorname{pa}(i)\)。Bayesian network 的联合分布分解为

\[
p(x_1,\ldots,x_d)
=\prod_{i=1}^{d}
p\bigl(x_i\mid x_{\operatorname{pa}(i)}\bigr).
\]

这比普通概率链式法则更强。链式法则允许 \(X_i\) 条件于排序中所有更早变量；图分解只保留父节点，相当于声明

\[
X_i\perp
\text{非后代且非父节点}
\mid X_{\operatorname{pa}(i)}.
\]

缺边的含义来自整个分解和条件化集合，不能简单读成``两个变量不相关''。

\subsection{链、共同原因和碰撞点有不同条件化行为}

三节点结构已经包含最容易混淆的情况。

在链

\[
X\longrightarrow Z\longrightarrow Y
\]

和共同原因

\[
X\longleftarrow Z\longrightarrow Y
\]

中，给定 \(Z\) 会阻断 \(X\) 与 \(Y\) 之间的路径，因此图蕴含 \(X\perp Y\mid Z\)。

碰撞结构

\[
X\longrightarrow Z\longleftarrow Y
\]

恰好相反：若 \(X,Y\) 的先验独立，未观察 \(Z\) 时路径被碰撞点阻断；一旦给定 \(Z\) 或其后代，\(X\) 与 \(Y\) 反而可能相关。这是 selection bias 的基本图形。看到两个变量在分层后相关，不能自动断言其中一个造成另一个。

d-separation 把这套规则推广到任意 DAG：给定集合 \(C\) 后，若所有路径都被非碰撞条件节点阻断，或含有未被观察及无被观察后代的碰撞点，则相应条件独立由图分解保证。

\subsection{Markov random field 用无向势函数表达相容性}

无向图没有生成方向。若最大 clique 集合为 \(\mathcal C\)，分布可写成

\[
p(x)
=\frac1{Z}
\prod_{C\in\mathcal C}\psi_C(x_C),
\]

其中 \(\psi_C\ge0\) 是势函数，

\[
Z=\sum_x\prod_{C\in\mathcal C}\psi_C(x_C)
\]

是配分函数。连续变量时求和换成积分。

势函数不是局部概率，不要求各自归一化。它衡量局部配置的相对相容性；只有除以全局 \(Z\) 后才得到概率。参数学习常需计算

\[
\nabla_\theta\log Z(\theta)
=\E_{p_\theta}[T(X)],
\]

于是一个看似局部的模型在学习时又遇到全局期望。这正是 MCMC、变分推断和对比近似会出现的原因。

无向图的局部 Markov 性质是：给定节点的邻居后，它与其余节点条件独立。与 DAG 不同，无向路径只需检查是否经过条件集合，没有 collider 的反向开启现象。

\subsection{同一独立结构可能有多张图}

不同 DAG 可以蕴含完全相同的条件独立集合，形成 Markov equivalence class。仅凭观察分布通常无法确定边的全部方向；若两个变量始终一起变化，方向、共同原因和选择机制可能都与数据相容。

因此 Bayesian network 中的箭头首先表示概率分解方向，不自动表示因果。把它解释为干预关系，需要额外的因果 Markov、faithfulness、无隐藏混杂或实验设计等条件。一个预测良好的图模型也可能拥有错误的因果方向。

\subsection{推断是对因子乘积消元}

给定观测 \(x_E\)，边缘概率需要对未查询变量求和：

\[
p(x_Q\mid x_E)
\propto
\sum_{x_H}
\prod_a\psi_a(x_a).
\]

变量消元依次把包含某变量的因子相乘并对该变量求和。不同消元顺序产生不同中间因子大小，复杂度由图的 treewidth 控制，而不只由节点数控制。链和树上可以高效执行消息传递；稠密图或存在大环时，中间因子可能指数增长。

HMM 的 forward--backward、树上的 sum-product、Kalman filter 都可以看成利用特定图结构重复消去局部变量。图模型的统一价值，是把模型假设与推断复杂度同时画出来：一条边既改变条件独立，也可能改变计算代价。

\subsection{图只是模型，不是数据事实的照片}

有限样本很难区分接近独立与真正独立，隐藏变量又能让观测图出现额外边。学习得到一张稀疏图，不代表缺失边对应现实中绝对没有作用；正则化、搜索空间和评分准则都参与了结构选择。

使用图模型时，应分别说明：图声明了哪些独立，参数规定了哪些局部分布，推断算法精确或近似计算了什么，以及数据是否足以辨认这些结构。图像直观不能替代这四层条件。Machine Learning Part 末尾的\hyperref[ch:120]{``因果推断：预测世界不等于改变世界''}会进一步为箭头加入结构方程与干预语义，并说明哪些因果量能够由观察分布识别。
