\section{把数据升到高维为什么有用}
\label{sec:13-Machine-Learning/09-Kernel-Methods-and-Gaussian-Processes/01}

到目前为止，学过的预测模型——\hyperref[sec:13-Machine-Learning/02-Linear-Models/01]{``线性回归从投影到概率模型''}中的直线与平面、\hyperref[sec:09-Convex-Optimization/07-Applications/03]{``最大间隔与支持向量机''}中的分离超平面——都在原始特征空间里画线性边界。这一节从两个反例开始：存在连``分对四个点''都做不到的线性模型；而把同样的数据映射到更高维后，一张平面就能干净地分开。升维确实有用，但显式升维的账单很快会付不起，这正是本单元后面各节的动机。

\begin{quote}
\textbf{先抓住这一点：}升维不是凭空把问题变简单，而是把原空间中的非线性关系改写成新特征上的线性关系。它提升表达力，也同步提升记忆噪声的能力。因此“训练数据可分”只说明表示足够丰富，不说明边界会泛化。
\end{quote}

\subsection{一维就已经会失败}

先看一个一维例子。四个样本 \(x=-2,-1,1,2\)，标签按``绝对值是否大于 \(1.5\)''给出：\(-2\) 与 \(2\) 为正类，\(-1\) 与 \(1\) 为负类。一维的线性分类器形如 \(f(x)=\sign(wx+b)\)，它的正类区域只能是半条直线：要么 \(x\) 大于某阈值，要么小于某阈值。但正类集合 \(\{-2,2\}\) 分居负类两侧，任何半直线只要同时包含 \(-2\) 与 \(2\)，就必然把 \(-1\) 或 \(1\) 也吞进去。不存在任何 \((w,b)\) 完成分类。

补救很廉价：增加一个特征，令 \(\phi(x)=(x,x^2)\)。在新空间里取 \(f(x)=x^2-\tfrac94\)，则 \(|x|>1.5\) 时 \(f>0\)，否则 \(f<0\)，四个点全部分对。正类区域在原空间是``两端''，在新空间是一条水平线上方——非线性边界被折进了新特征里。

\subsection{四个点难住整张平面}

一维的失败也许还显得特殊，二维的 XOR（异或）则是彻底的死局。四个样本的特征取值为 \(0\) 或 \(1\)，标签由 XOR 规则给出：

\[
(0,0)\mapsto -1,\quad
(1,1)\mapsto -1,\quad
(0,1)\mapsto +1,\quad
(1,0)\mapsto +1.
\]

任何线性分类器都形如 \(f(x)=\sign(w_1x_1+w_2x_2+b)\)。假设它分对了全部四个点，那么正类两样本给出

\[
w_2+b>0,\qquad w_1+b>0,
\]

负类两样本给出

\[
b<0,\qquad w_1+w_2+b<0.
\]

把前两个不等式相加得到 \(w_1+w_2+2b>0\)，把后两个相加得到 \(w_1+w_2+2b<0\)。同一个量不可能既正又负，因此不存在任何 \((w_1,w_2,b)\) 完成分类。这已经证明解不存在，远比``还没找到''更强：两条对角线各属一类，任何直线都会切错至少一块。

更值得记住的是失败的原因。XOR 的标签取决于两个特征的联合取值：单独看 \(x_1\) 或 \(x_2\)，与标签毫无关系——在每个特征的任一取值下，正负类各占一半；合在一起却完全决定标签。线性模型只能表达``每个特征各记一笔账、加起来''的函数，而这里恰恰没有可单独记的账。

\subsection{一个三维映射让平面重新够用}

既然缺少``联合取值''的信息，就把它补成一个新特征。定义

\[
\phi(x_1,x_2)=(x_1,\;x_2,\;x_1x_2).
\]

四个样本在新空间 \(\R^3\) 中成为

\[
\text{负类： }(0,0,0),\ (1,1,1);
\qquad
\text{正类： }(0,1,0),\ (1,0,0).
\]

取

\[
f(x)=1.5\,x_1+1.5\,x_2-3\,x_1x_2-1,
\]

逐点代入：\((0,0)\) 处为 \(-1\)，\((1,1)\) 处为 \(1.5+1.5-3-1=-1\)，\((0,1)\) 与 \((1,0)\) 处均为 \(1.5-1=0.5\)。负类全负、正类全正：在 \(\phi\) 给出的三维空间里，一张平面完成了分类。

这里发生了两件事。第一，``非线性问题''只是换到了另一个空间，并没有消失：新空间里的分离面对应原空间中的曲线 \(1.5x_1+1.5x_2-3x_1x_2-1=0\)，那是一条双曲线而非直线。第二，模型对参数仍然是线性的——写成 \(f=\ip{w}{\phi(x)}+b\)，\(w=(1.5,1.5,-3)\) 以线性方式出现。这正是\hyperref[sec:13-Machine-Learning/02-Linear-Models/01]{``线性回归从投影到概率模型''}末尾强调过的：线性指对参数线性，对原始变量可以任意弯曲。升维没有改变模型的种类，只改变了模型看到的数据表示。

\subsection{高维为什么更容易分开}

XOR 不是孤立的巧例。Cover 定理（1965）给出一般性陈述：对处于一般位置的点，线性分类器能够实现的二分类方式数量会随特征维数上升；当特征维数接近样本数时，几乎所有标签方式都可分。这里的“一般位置”排除了特殊的仿射退化，计数结论也不是说任意给定的非线性映射都会让数据可分。

直觉并不神秘。\(d\) 维空间中，\(n\) 个一般位置的点能被超平面实现的标签划分（dichotomy）数目为

\[
2\sum_{k=0}^{d}\binom{n-1}{k},
\]

而全部标签方式共 \(2^n\) 种。当 \(d\) 远小于 \(n\) 时，可实现的比例微乎其微；当 \(d\) 接近 \(n-1\) 时，求和逼近 \(2^{n-1}\)，几乎每种标签划分都有某张超平面能实现。维数每增加一，空间就多一些``余地''让点错开。

这句直觉必须立刻配上警告：分得开不等于分得好。若任意标签都能被分开，那么把噪声当标签也照样分得开——表达力的增长同时是记忆能力的增长。``可分性变容易''与``泛化变容易''是两件事，后者需要容量控制来平衡，\hyperref[ch:118]{``统计学习理论：训练误差对真风险有多乐观''}一章处理的正是这个差额。升维只解决了前半件事。

\subsection{显式构造特征的账单}

既然升维有用，直接把``所有可能的交叉特征''都造出来似乎是一条路。设原始输入 \(x\in\R^d\)，构造全部不超过二次的特征：

\[
\phi(x)=\bigl(1,\;x_1,\dots,x_d,\;x_1^2,\;x_1x_2,\;\dots,\;x_d^2\bigr).
\]

特征总数为

\[
\binom{d+2}{2}=\frac{(d+2)(d+1)}{2}\approx \frac{d^2}{2}.
\]

\(d=1000\)（一张小图片、一段稀疏文本都远不止此）时约五十万维。这还只是二次：全部不超过五次的特征数为

\[
\binom{d+5}{5}\approx \frac{d^5}{120},
\]

\(d=1000\) 时约 \(8\times 10^{12}\) 维。一条样本的显式特征就要占据几十 TB 内存，内积、求解、存储全部不可行。手工挑选少量交叉项可以缓解，却又回到``靠人猜哪些交互重要''的老问题——猜错的部分，模型永远补不回来。

账单的根源在于我们坚持显式构造 \(\phi(x)\)：先把每个样本变成一根高维向量，再在向量上做运算。如果后续算法根本不需要 \(\phi(x)\) 本身，只需要两个样本之间的某种``相似程度''，也许就能绕开整张账单。

\subsection{出路：不构造特征，只算内积}

回看已经学过的推导，会发现一个共同的形态。\hyperref[sec:09-Convex-Optimization/07-Applications/03]{``最大间隔与支持向量机''}的对偶问题里，数据只以 \(x_i^{\trans}x_j\) 的形式出现；\hyperref[sec:13-Machine-Learning/02-Linear-Models/02]{``正则化从 Ridge 到 Lasso''}的一阶条件里，数据只以 \(X^{\trans}X\) 与 \(X^{\trans}y\) 的形式出现——同样全是内积的组合。把 \(x\) 换成 \(\phi(x)\) 后，这些算法只需要下面的内积，无须显式构造 \(\phi(x)\) 本身：

\[
k(x,z)=\ip{\phi(x)}{\phi(z)}.
\]

若有办法不算 \(\phi\) 而直接算出 \(k(x,z)\)，就能把整张维度账单换成一次便宜的函数求值：多项式特征的内积只需要 \(O(d)\) 次运算，与特征空间是五十万维还是 \(8\times10^{12}\) 维无关。下一节把这个想法变成定义，并检查它到底能覆盖多少已学过的方法。
