\section{核技巧：只通过内积看世界}
\label{sec:13-Machine-Learning/09-Kernel-Methods-and-Gaussian-Processes/02}

上一节的结论是：升到高维有用，但显式构造 \(\phi\) 付不起账单。本节把上一节末尾的想法变成正式定义，然后做两件事：盘点哪些已学过的算法只通过内积接触数据；给``随手写出的相似度函数''一个判据——它什么时候真的是某个特征空间里的内积。

\begin{quote}
\textbf{先抓住这一点：}核技巧不是“任何算法加一个相似度函数”，而是把一个只依赖内积的算法搬到隐式特征空间。真正的门槛有两个：算法必须只通过内积接触数据，所用函数必须对任意有限样本产生半正定 Gram 矩阵。
\end{quote}

\subsection{核函数：不算特征，只算相似}

设存在一个特征空间 \(\mathcal{F}\)（一个内积空间）与映射 \(\phi:\mathcal{X}\to\mathcal{F}\)。称

\[
k(x,z)=\ip{\phi(x)}{\phi(z)}_{\mathcal{F}}
\]

为由 \(\phi\) 诱导的核函数。核技巧的全部内容是一句话：凡是以内积形式使用数据的算法，把每个内积 \(\ip{x}{z}\) 换成 \(k(x,z)\)，就等价于把算法原样搬进 \(\mathcal{F}\) 运行，而全程不需要知道 \(\phi\) 的坐标。

这听起来像偷换概念，关键在于``只需要内积''这一前提到底覆盖多少东西。下面盘点三个已学过的对象。

\subsection{支持向量机：对偶里只有内积}

\hyperref[sec:09-Convex-Optimization/07-Applications/03]{``最大间隔与支持向量机''}中，硬间隔 SVM 的驻点条件给出 \(w=\sum_i\alpha_iy_ix_i\) 与 \(\sum_i\alpha_iy_i=0\)。把 \(w\) 的形式代回原始问题消去 \((w,b)\)，剩下的对偶问题只含乘子：

\[
\max_{\alpha}\ \sum_{i=1}^{n}\alpha_i
-\frac12\sum_{i=1}^{n}\sum_{j=1}^{n}
\alpha_i\alpha_j\,y_iy_j\,\ip{x_i}{x_j},
\qquad
\alpha_i\ge0,\ \sum_{i=1}^{n}\alpha_iy_i=0,
\]

预测函数同样只用内积：

\[
f(x)=\sign\Bigl(\sum_{i=1}^{n}\alpha_iy_i\,\ip{x_i}{x}+b\Bigr).
\]

目标、约束、预测三处都只用内积。把每个 \(\ip{x_i}{x_j}\) 换成 \(k(x_i,x_j)\)，得到的就是在特征空间里求最大间隔超平面；决策边界回到原空间，则弯曲成 \(k\) 决定的形状。上一节的 XOR 用 \(\phi(x)=(x_1,x_2,x_1x_2)\) 可分，对应的核是

\[
k(x,z)=x_1z_1+x_2z_2+x_1x_2\,z_1z_2,
\]

直接代入计算即可，三维向量 \(\phi\) 从头到尾不必出现。

\subsection{Ridge 回归：同一个解的 Gram 形态}

\hyperref[sec:13-Machine-Learning/02-Linear-Models/02]{``正则化从 Ridge 到 Lasso''}的一阶条件是

\[
(X^{\trans}X+\lambda I)\,w=X^{\trans}y.
\]

它有一个不那么显眼却等价的形式。把解写成训练样本的线性组合 \(w=X^{\trans}\alpha\)（为什么可以这样写，下一节的表示定理会给出一般答案），代入得

\[
X^{\trans}XX^{\trans}\alpha+\lambda X^{\trans}\alpha=X^{\trans}y,
\]

只要

\[
(XX^{\trans}+\lambda I)\,\alpha=y
\]

成立，上式就成立。记 Gram 矩阵 \(G=XX^{\trans}\)，\(G_{ij}=\ip{x_i}{x_j}\)，则

\[
\alpha=(G+\lambda I)^{-1}y,
\qquad
f(x)=\ip{w}{x}=\sum_{i=1}^{n}\alpha_i\,\ip{x_i}{x}.
\]

同样只剩内积。这个形式还透露一个结构性事实：无论特征有多少维，\(w\) 永远躺在训练样本张成的空间里，对偶系数 \(\alpha\in\R^n\) 才是问题的真正自由度。把 \(G\) 换成核矩阵 \(K\)，就是本单元最后一节的核岭回归。

\subsection{PCA：协方差可以换成 Gram}

\hyperref[sec:13-Machine-Learning/04-Dimensionality-Reduction/01]{``PCA 从最大方差到最小重构误差''}中，主方向是 \(X_c^{\trans}X_c\) 的特征向量，其中 \(X_c\) 是中心化后的数据矩阵。但 \(X_c^{\trans}X_c\in\R^{p\times p}\) 与 Gram 矩阵 \(X_cX_c^{\trans}\in\R^{n\times n}\) 共享全部非零特征值：若 \(u_j\) 是 \(X_cX_c^{\trans}\) 对应特征值 \(\sigma_j^2\) 的单位特征向量，则 \(v_j=X_c^{\trans}u_j/\sigma_j\) 是 \(X_c^{\trans}X_c\) 对应同一特征值的单位特征向量。因此主方向与主成分得分都可以完全从 Gram 矩阵恢复；进一步把 Gram 矩阵换成核矩阵，就是核 PCA 的起点。这里只指出这扇门，不走进去。

\subsection{三个常见核}

\textbf{线性核。} \(k(x,z)=\ip{x}{z}\)，对应 \(\phi\) 为恒等映射。它是最基本的检验：核化后的算法必须退化回原来的线性算法。

\textbf{多项式核。} \(k(x,z)=(1+\ip{x}{z})^{p}\)，对应``全部不超过 \(p\) 次单项式''的特征空间。以 \(d=2,p=2\) 为例，令 \(x=(x_1,x_2)\)，\(z=(z_1,z_2)\)，直接展开：

\[
(1+\ip{x}{z})^{2}
=\bigl(1+x_1z_1+x_2z_2\bigr)^2
=1+2x_1z_1+2x_2z_2+x_1^2z_1^2+x_2^2z_2^2+2x_1x_2\,z_1z_2.
\]

取

\[
\phi(x)=\bigl(1,\ \sqrt2\,x_1,\ \sqrt2\,x_2,\ x_1^2,\ x_2^2,\ \sqrt2\,x_1x_2\bigr),
\]

逐项核对可见 \(\ip{\phi(x)}{\phi(z)}\) 与上式完全相同。核值等于六维内积，这不是魔术，只是组合数的记账：\((1+\ip{x}{z})^p\) 展开后的每一项，天然是``\(x\) 的某个单项式乘 \(z\) 的同一单项式''，系数收进根号即可。但两种算法的计算量完全不同：左边一次内积加一次幂，\(O(d)\)；右边要先构造约 \(d^p/p!\) 维的向量再做内积。

\textbf{Gaussian/RBF 核。}

\[
k(x,z)=\exp\bigl(-\gamma\norm{x-z}^{2}\bigr),\qquad \gamma>0.
\]

把 \(\norm{x-z}^{2}=\norm{x}^{2}+\norm{z}^{2}-2\ip{x}{z}\) 代入，再对 \(\exp(2\gamma\ip{x}{z})\) 做 Taylor 展开，可以看到它对应无穷维的特征空间——不受次数限制的全部单项式都带着衰减系数出现。直觉上，\(\gamma\) 是相似度的衰减速度：\(\gamma\) 大时，只有紧邻的点核值明显非零，学到的函数可以剧烈起伏；\(\gamma\) 小时，远处的点也互相影响，函数被迫平滑。这个旋钮直接控制有效复杂度，最后一节还会回到它。

\subsection{什么时候可以自己设计核}

反过来问：写一个直觉上合理的相似度函数 \(s(x,z)\)，什么时候能当核用？答案是正定性判据（在机器学习语境中常被宽泛地称为 Mercer 条件；经典 Mercer 定理还包含定义域、连续性与测度等附加条件）：

\[
k\ \text{是某个特征空间的内积}
\iff
\text{对任意有限点集 } \{x_1,\dots,x_n\},
\ K=\bigl[k(x_i,x_j)\bigr]_{i,j}\ \text{半正定}.
\]

必要性一行就能验证：若 \(k(x,z)=\ip{\phi(x)}{\phi(z)}\)，则对任意 \(v\in\R^n\)，

\[
v^{\trans}Kv
=\sum_{i=1}^{n}\sum_{j=1}^{n}v_iv_j\,\ip{\phi(x_i)}{\phi(x_j)}
=\norm{\sum_{i=1}^{n}v_i\,\phi(x_i)}^{2}\ge0.
\]

充分性——从半正定的 \(k\) 真的造出一个特征空间——正是下一节构造 RKHS 的内容。

这个判据让``设计核''成为可检验的工程，而不是玄学。封闭性性质提供了积木：两个核的和仍是核，两个核的积仍是核，核乘正常数仍是核——对应的 Gram 矩阵分别保持半正定。于是可以按数据的不同侧面分别构造相似度再组合。反过来，一个凭直觉写出却不满足半正定的``相似度''，会让 SVM 对偶目标失去凹性、让``范数''不再是范数——\hyperref[sec:09-Convex-Optimization/07-Applications/03]{``最大间隔与支持向量机''}末尾对非 PSD 相似度的警告正来自这里。

\subsection{核技巧没回答的问题}

到这里，核技巧仍只是一个算法层面的替换规则。它没有回答：\(k\) 对应的特征空间到底长什么样，里面的``函数''是什么对象，\(\norm{w}^{2}\) 换成核语言后惩罚的究竟是什么。不回答这些问题，``加正则项控制复杂度''就只是一句口号。下一节把这个空间严格构造出来。
