\section{KNN 把度量直接变成归纳偏置}

K-nearest neighbors 几乎没有显式训练：给定查询 \(x\)，找到训练集中距离最近的 \(k\) 个点，再平均它们的响应或投票类别。计算简单不等于假设少；关于相似性的全部判断都被压进了距离和特征表示。

\subsection{邻域平均估计局部条件均值}

回归 KNN 写成

\[
\widehat f_k(x)
=\frac1k\sum_{i\in\mathcal N_k(x)}y_i,
\]

其中 \(\mathcal N_k(x)\) 是 \(x\) 的 \(k\) 个近邻。若邻域足够小，其中样本近似来自 \(X=x\) 附近；若样本又足够多，平均可以降低噪声。因此一致性需要两个相反趋势：\(k\to\infty\) 以获得更多样本，同时 \(k/n\to0\) 使邻域半径缩小。

固定很小的 \(k\) 保留局部细节但方差大；很大的 \(k\) 平均更稳定，却把不同区域混在一起。\(k\) 是平滑尺度，不只是投票人数。

\subsection{特征尺度会重新定义谁是邻居}

Euclidean 距离

\[
d(x,x')^2
=\sum_{j=1}^{p}(x_j-x_j')^2
\]

让数值尺度大的坐标主导。把米换成毫米会使对应差异放大一千倍，最近邻关系可能完全改变。标准化、Mahalanobis 距离、学习表示或问题特定度量都在重新规定相似性。

无关特征也会贡献距离噪声。即使每个无关坐标影响很小，许多坐标的平方差累积后仍可能淹没有用信号。特征选择和度量学习对 KNN 不是附加优化，而是模型定义的一部分。

\subsection{高维中局部需要指数级样本}

在单位超立方体 \([0,1]^p\) 中，一个边长为 \(r\) 的局部立方体体积为 \(r^p\)。希望它平均包含 \(k\) 个均匀样本，大致需要

\[
nr^p\approx k,
\qquad
r\approx\left(\frac{k}{n}\right)^{1/p}.
\]

维度 \(p\) 增大时，即使 \(n\) 很大，\(r\) 也不再小；所谓近邻可能离查询点仍很远。若想保持相同局部半径，样本量必须随维度指数增长。这是维数灾难的几何形式。

实际数据若集中在低维流形上，有效维度可能小于坐标数，但这是一项需要验证的结构假设。先用全数据学习降维表示再交叉验证，也会发生表示泄漏；降维必须嵌入每个训练折。

\subsection{距离集中削弱远近对比}

许多高维分布中，不同坐标距离贡献相加，最近距离和最远距离的相对差异会缩小。所有点看起来都差不多远时，“取最近几个”对小扰动十分敏感。

这不表示任何高维 KNN 都必然失败。稀疏表示、合适度量或强低维结构可以恢复邻域意义；但成功来自结构，而不是 KNN 绕过了维数灾难。

\subsection{预测边界与计算边界}

分类 KNN 的概率估计是邻域类别频率。类别不平衡时，多数类更容易占据邻域，可使用距离权重或代价敏感决策，但这些修改又改变了估计对象。查询落在训练支持之外时，算法仍会返回某些“最近”点，却没有机制警告这些点实际很远。

朴素查询需比较全部 \(n\) 个样本，成本为 \(O(np)\)。树索引在低维有效，高维中常退化；近似最近邻以召回误差换速度。评价时应把搜索近似造成的误差与统计模型误差分开。

KNN 的价值是把归纳偏置暴露得非常直接：相近输入应有相近输出。困难也同样直接——相近究竟由谁定义，以及现有样本是否足以让局部真正局部。
