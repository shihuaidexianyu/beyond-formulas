机器学习最容易从算法名称开始：先问该用线性模型、树还是神经网络，再把数据交给优化器。这个顺序把真正决定结论含义的选择藏了起来。模型训练之前，必须先说清预测发生在谁身上、当时能看见什么、输出将触发什么行动，以及错误由谁承担。

本单元把一项学习任务拆成三个不能交换顺序的问题：

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  从现实决策到学习问题：先固定预测单位、可用信息、输出与行动，再定义数据、假设空间和部署分布。算法只能在这个问题内部求解，不能替代问题定义。
\item
  损失、风险、目标与指标：区分单个样本的代价、未来总体上的平均代价、训练时实际优化的表达式和报告结果时使用的摘要。四者可以相关，却通常不是同一个函数。
\item
  目标函数与噪声模型的对应：为什么回归用 MSE、分类用交叉熵——噪声模型给出似然，似然给出目标函数；形状决定优化几何，统计性质决定泛化。
\item
  数据划分、泄漏与分布偏移：解释独立测试为什么能够估计未来风险，验证数据怎样因参与选择而失去独立性，以及训练分布与部署分布不同时旧分数为什么不再回答新问题。
\end{enumerate}

这一单元规定后续所有模型必须遵守的评价纪律，暂不涉及具体模型的选择。读完后，应能把``模型在测试集上达到某个分数''改写成一条带条件的陈述：在哪个采样机制下、对哪个预测单位、用什么损失、经过多少次选择，估计了哪个目标分布上的表现。
