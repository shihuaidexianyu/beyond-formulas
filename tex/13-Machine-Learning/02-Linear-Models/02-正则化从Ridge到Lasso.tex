\section{正则化从 Ridge 到 Lasso}
\label{sec:13-Machine-Learning/02-Linear-Models/02}

\begin{quote}
\textbf{先抓住这一点：}Ridge 连续收缩所有系数，Lasso 的尖角还能把部分系数推到精确零。两者都用偏差换稳定性，但结果对特征尺度和相关性十分敏感。
\end{quote}

上一节留下了一个刻意的失败：两列特征几乎相同，最小二乘仍能把训练数据拟合得很好，系数却可能随着一点测量噪声剧烈变化。这是因为数据几乎没有提供足够信息去区分这两个方向，不能简单归咎于算法。若继续追求训练误差的最小值，模型会把噪声也解释成精确结构。

正则化的第一性问题是：当数据不足以唯一、稳定地决定参数时，我们愿意额外偏好什么样的解？``选 Ridge 还是 Lasso''要在回答这个问题后再决定。

\subsection{Ridge：不让弱信息方向无限放大}

Ridge 回归求解

\[
\widehat\beta_\lambda
=\argmin_\beta
\left\{
\frac12\|y-X\beta\|_2^2+
\frac{\lambda}{2}\|\beta\|_2^2
\right\},
\qquad \lambda\ge 0.
\]

第一项要求解释数据，第二项不允许系数过大。其一阶条件为

\[
(X^{\trans} X+\lambda I)\widehat\beta_\lambda=X^{\trans} y.
\]

当 \(\lambda>0\) 时，即使 \(X\) 不满列秩，目标也会在被惩罚坐标上变成强凸，因此解唯一。

SVD 最能揭示它究竟做了什么。若 \(X=U\Sigma V^{\trans}\)，则

\[
\widehat\beta_\lambda
=V\diag
\left(
\frac{\sigma_j}{\sigma_j^2+\lambda}
\right)U^{\trans} y.
\]

普通最小二乘会沿奇异方向除以 \(\sigma_j\)。当 \(\sigma_j\) 很小时，噪声被巨大放大；Ridge 把这一因子改为 \(\sigma_j/(\sigma_j^2+\lambda)\)，对信息最弱的方向收缩得最强。它以一点偏差换取更小方差，目标是让未见数据上的预测更稳定，即使训练拟合因此略差一些。

如果两个温度传感器几乎记录同一个量，OLS 可能给出 \(8.2x_1-7.3x_2\) 这样的抵消组合。下一批数据只要略有偏差，预测就会大幅波动。Ridge 更倾向把作用平滑分配给两个传感器。单个系数不再追求无偏，但整体预测可能可靠得多。

在高斯噪声模型下，如果给系数一个零均值各向同性高斯先验，Ridge 也可解释为后验众数。不过这个解释依赖尺度：把某个特征从``米''改成``毫米''，对应系数会缩小一千倍，同一惩罚便不再代表同一先验。因此除非单位本身承载有意的先验信息，通常应在每个训练折内部标准化特征，并通常不惩罚截距。

\subsection{Lasso：允许某些坐标恰好消失}

若我们不仅想收缩，还希望得到稀疏表示，可以把二次惩罚换成 \(\ell_1\) 惩罚：

\[
\widehat\beta_\lambda
=\argmin_\beta
\left\{
\frac12\|y-X\beta\|_2^2+\lambda\|\beta\|_1
\right\}.
\]

\(|\beta_j|\) 在零点不可微，这恰恰是 Lasso 能产生精确零系数的原因。以一维问题为例，

\[
\min_\beta\ \frac12(\beta-z)^2+\lambda|\beta|.
\]

分别考察三种情形：当 \(\beta>0\) 时，一阶条件给出 \(\beta=z-\lambda\)，仅当 \(z>\lambda\) 时自洽；当 \(\beta<0\) 时对称地给出 \(\beta=z+\lambda\)，仅当 \(z<-\lambda\) 时自洽；当 \(|z|\le\lambda\) 时，零落入目标在 \(\beta=0\) 处的次微分，最优解恰为 \(\beta=0\)。合起来就是软阈值解

\[
S_\lambda(z)
=\sign(z)(|z|-\lambda)_+.
\]

在正交设计下，每个坐标都独立执行这一操作。几何上，\(\ell_1\) 约束集合在坐标轴上有尖角，损失等高线较容易在尖角处相切；优化上，近端算子直接把一段范围内的坐标送到零。

但``系数为零''只表示在当前样本、表示方式和惩罚强度下，这个变量没有被选入预测模型。若两个特征高度相关，Lasso 可能任意选择其中一个；换一批样本，入选者便可能互换。稀疏预测不等于找到了唯一的科学机制，更不等于得到因果变量。

\subsection{两种惩罚应怎样选择}

Ridge 适合许多变量都贡献少量信号、且相关特征应共同保留的情形；Lasso 适合相信表示在当前坐标系中近似稀疏、并重视紧凑模型的情形。若既希望成组保留相关变量又希望稀疏，可以考虑 elastic net，但它仍不能替代对稳定性和选择偏差的检查。

真正困难的是选择 \(\lambda\)。如果先用全部数据标准化、筛变量或挑好 \(\lambda\)，再报告交叉验证误差，验证折已经参与了模型选择。正确做法是把所有数据驱动的预处理放入每个训练折；若还要无偏估计整个调参流程的泛化误差，则用外层验证评估、内层验证选 \(\lambda\)，或保留从未参与选择的测试集。

除了最低验证误差，还应看整条正则化路径：

\begin{itemize}
\tightlist
\item
  \(\lambda\) 增大时，训练误差怎样上升、验证误差怎样先降后升；
\item
  Ridge 系数是否平滑收缩；
\item
  Lasso 的非零集合在相邻折和 bootstrap 样本中是否稳定；
\item
  选择的模型是否对真正部署分布仍然适用。
\end{itemize}

数值上，Ridge 可用分解后的线性求解；Lasso 常用坐标下降或近端梯度，并以 KKT 残差或对偶间隙判断是否真正收敛。不能因为目标值变化很小，就假定非光滑问题已得到足够精确的解。
