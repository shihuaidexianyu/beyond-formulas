\section{从二分类到 Softmax 分类}

若响应只有``发生''和``未发生''两类，直接套用线性回归会遇到两个问题：预测值可以小于 0 或大于 1，而误差的波动也不会像常方差高斯噪声那样。我们需要重新回答``数据怎样产生''和``什么损失与这个概率模型一致''；强行截断输出绕过了这两个问题。

\subsection{Logistic 回归把线性结构放在 log-odds 上}

令 \(Y\in\{0,1\}\)，并设

\[
p(x)=P(Y=1\mid x).
\]

概率必须位于 \((0,1)\)，但 odds \(p/(1-p)\) 位于 \((0,\infty)\)，log-odds 则可取任意实数。于是可以把线性结构放在

\[
\log\frac{p(x)}{1-p(x)}=x^\top\beta
\]

上。解回 \(p\) 得 sigmoid：

\[
p(x)=\sigma(x^\top\beta)
=\frac{1}{1+\exp(-x^\top\beta)}.
\]

这意味着 \(\beta_j\) 描述其他特征固定时 log-odds 的变化；\(\exp(\beta_j)\) 是 odds 的倍数变化，不是概率增加了 \(\beta_j\)。当基准概率不同时，同一 odds ratio 对应的概率差也不同。

对独立 Bernoulli 样本取负对数似然，得到二元交叉熵

\[
L(\beta)
=-\sum_{i=1}^n
\left[
y_i\log p_i+(1-y_i)\log(1-p_i)
\right].
\]

若 \(p=(p_1,\ldots,p_n)^\top\)，则

\[
\nabla L(\beta)=X^\top(p-y),\qquad
\nabla^2L(\beta)=X^\top WX,
\]

其中 \(W_{ii}=p_i(1-p_i)\ge0\)。Hessian 半正定，所以线性 Logistic 回归的负对数似然是凸函数。这个性质保证局部极小点也是全局极小点，却不保证一定存在有限的无正则解。

\subsection{完全分离揭示``可分类''不等于``可估计''}

假设一条直线把训练集的正负样本完全分开。沿这个分离方向不断放大 \(\beta\)，正类概率会趋近 1，负类概率趋近 0，似然不断改善，但参数范数趋向无穷。训练准确率看起来完美，有限最大似然估计却不存在。

这是很有教学价值的冲突：分类边界已经足够好，概率模型反而无法确定置信程度。加入上一节的 Ridge 类惩罚会得到有限解；贝叶斯先验也能约束参数。无论采用哪种方法，都应认识到正则化不仅是``防过拟合''，有时还在修复估计问题。

数值实现应直接从 logits 计算稳定损失。例如 \(\log(1+\exp z)\) 应改写成稳定形式 \(\max(z,0)+\log(1+\exp(-|z|))\) 再求值，不能在极大 \(|z|\) 时先算 sigmoid 再取对数，否则会溢出或得到 \(\log 0\)。

\subsection{多分类需要相对分数}

若类别互斥且共有 \(K\) 类，可为每类给一个 logit

\[
z_k=w_k^\top x
\]

并用 softmax 归一化：

\[
p_k=\frac{\exp(z_k)}{\sum_{j=1}^K\exp(z_j)}.
\]

对 one-hot 标签 \(y\)，单样本交叉熵为

\[
\ell=-\sum_{k=1}^Ky_k\log p_k.
\]

对 logits 求导会得到极其简洁的结果

\[
\nabla_z\ell=p-y.
\]

这不是需要死记的巧合。交叉熵中的 \(\log p_k\) 展开为 \(z_k-\log\sum_j e^{z_j}\)；第一项贡献真实标签，第二项的梯度正是 softmax 概率。

softmax 只依赖相对分数。给所有 logits 加同一个常数 \(c\)，概率完全不变，因此参数存在共同平移的冗余。可以固定一个参考类，也可以用正则化选择代表解。计算时正利用这种不变性，令 \(m=\max_k z_k\)：

\[
\log\sum_k e^{z_k}
=m+\log\sum_k e^{z_k-m}.
\]

例如 logits 为 \((1000,1001,999)\) 时，直接指数运算会溢出；减去最大值后变成 \((-1,0,-2)\)，概率不变，计算却稳定。

softmax 适合``恰有一个类别''的任务。若一张图片可以同时含有``室内''\,``人物''和``夜景''，标签彼此不互斥，应为每个标签使用独立 Bernoulli 概率，而不是迫使它们在同一个 simplex 上竞争。

\subsection{概率输出之后才是决策}

模型输出 \(P(Y=1\mid x)=0.3\) 不等于必须判为 0。若漏诊代价是误报的九倍，那么即使概率低于 0.5，也可能值得进一步检查。阈值应由行动成本、容量约束和基准率决定，而不是由 sigmoid 的中点替代。

评估也需区分几件事：

\begin{itemize}
\tightlist
\item
  log loss 衡量整套概率分布；
\item
  calibration 询问预测为 0.7 的样本是否约有 70\% 为正；
\item
  ROC 或 PR 曲线描述阈值改变时的排序表现；
\item
  准确率只评价某一个阈值下的硬决策，并可能在类别不平衡时极具欺骗性。
\end{itemize}

如果训练时人为过采样正类，模型见到的 class prior 已改变。排序可能仍有价值，但原始概率通常需要校正。若部署人群的基准率或条件分布继续变化，历史验证集上的校准也不能永久保证未来可靠。

\subsubsection{一次完整的小任务}

为一个三类风险分流问题建立线性 softmax 模型。先写清类别是否确实互斥，再实现稳定的 log-softmax；用自动微分或有限差分核对解析梯度 \(p-y\)；最后分别报告交叉熵、混淆矩阵和校准结果。然后给所有 logits 加同一常数，解释为什么预测不变、参数却并非完全可识别。若把任务改成多标签，你需要修改模型的哪一部分？

概率评分与信息量的关系将在``交叉熵与 KL 差异''中继续展开。
