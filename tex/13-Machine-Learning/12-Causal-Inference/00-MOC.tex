预测问题问的是：看见某些特征后，结果通常怎样变化。因果问题进一步问：若主动改变一个条件，结果会怎样变化。两者可能使用同一张数据表、同一个回归模型，答案却依赖不同的假设。相关性由联合分布描述；因果性还要说明变量由什么机制产生、哪些变量可以干预，以及哪些替代世界能够比较。

本单元不把因果推断写成一种新算法，而把它拆成三个层次：先定义希望回答的因果问题，再判断该目标能否由现有数据识别，最后才选择估计方法。若识别所需的无混杂、重叠或一致性条件不成立，增加样本、换更复杂的模型或把标准误算得更精确，都不能修复目标与数据之间的断裂。

六节依次完成这条链：

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  观察、干预与反事实回答不同问题：区分条件概率、干预分布和个体反事实，先固定 estimand 再谈计算。
\item
  结构因果模型把干预写成机制替换：用结构方程和因果图表达生成假设，并由截断分解得到干预分布。
\item
  混杂、碰撞点与后门调整：辨认哪些路径应阻断、哪些变量不该控制，以及 Simpson 反转为何可能出现。
\item
  潜在结果把处理效应定义在替代世界之间：说明随机分配、条件可忽略性、重叠与一致性怎样识别平均处理效应。
\item
  回归、加权与双重稳健估计各自依赖什么：把 outcome model、propensity score 与增广估计放到同一公式中，并讨论极端权重和机器学习 nuisance model。
\item
  可识别不等于可外推也不等于可决策：处理隐藏混杂、干扰、异质效应、时间变化和目标人群转移，给出因果结论的报告边界。
\end{enumerate}

这里主要讨论二元处理与平均处理效应，因为它足以暴露因果推断的核心结构。连续处理、纵向干预、工具变量、断点回归和因果发现都可在同一框架中继续展开，但每一种设计都需要自己的识别论证，不能只把二元公式换一个符号。
