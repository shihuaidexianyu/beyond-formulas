\section{关系与等价类}

\subsection{关系只是被选中的有序对}

从集合 \(A\) 到集合 \(B\) 的二元关系 \(R\)，说穿了就是 \(A\times B\) 的一个子集。如果 \((a,b)\in R\)，通常写成 \(aRb\)。

比如学生集合 \(S\) 和课程集合 \(C\)，``学生 \(s\) 选了课 \(c\)'' 就构成 \(S\times C\) 的一个子集；整数集上的``\(a\) 整除 \(b\)'' 也是 \(\Z\times\Z\) 的子集。关系可以画成箭头、列成表格、写成条件，但本质上都是同一堆有序对的不同表示方式。

如果关系是从 \(A\) 到自身的，那就可以研究三个常见性质：

\begin{itemize}
\tightlist
\item
  自反性：每个 \(a\in A\) 都满足 \(aRa\)；
\item
  对称性：只要 \(aRb\) 就有 \(bRa\)；
\item
  传递性：\(aRb\) 且 \(bRc\) 的话，一定有 \(aRc\)。
\end{itemize}

这三条性质是互相独立的。比如``认识''这个人与人之间的关系，大体对称但不一定传递；``小于''传递但既不自反也不对称；``等于''则三条全满足。

\subsection{等价关系是在选择一种``忽略差异''的方式}

同时满足自反、对称、传递三条的关系，叫等价关系。它不代表两个对象完全一样，只是说在当前关心的标准下，可以把它们当成一类来看。

固定一个正整数 \(m\)，在整数上定义：

\[
a\sim b
\quad\Longleftrightarrow\quad
m\mid(a-b).
\]

意思就是 \(a\) 和 \(b\) 除以 \(m\) 余数相同。这里 \(m\mid(a-b)\) 读作``\(m\) 整除 \(a-b\)''，也就是 \(a-b\) 是 \(m\) 的倍数。整除关系后面讲偏序的时候还会再遇到。这个 \(\sim\) 确实是等价关系：\(m\mid0\) 保证了自反；\(m\mid(a-b)\) 自然也有 \(m\mid(b-a)\)，所以对称；传递性也不难，\(a-b\) 和 \(b-c\) 都是 \(m\) 的倍数，加起来 \(a-c\) 也是。

对任意一个元素 \(a\)，它的等价类是：

\[
[a]=\{x\in A:x\sim a\}.
\]

拿模 3 举例，整数被分成了三类：

\[
[0],\quad[1],\quad[2],
\]

比如 \([1]=\{\ldots,-5,-2,1,4,7,\ldots\}\)。注意 \([1]\) 和 \([4]\) 不是两个有重叠的类，它们完完全全是同一个类。

\subsection{为什么等价类只能相等或不交}

假设 \([a]\) 和 \([b]\) 有一个公共元素 \(x\)。那 \(x\sim a\) 且 \(x\sim b\)。由对称性得 \(a\sim x\)，再跟 \(x\sim b\) 一起用传递性，就推出 \(a\sim b\)。这样一来，任何跟 \(a\) 等价的元素也都跟 \(b\) 等价，所以 \([a]\subseteq[b]\)；反过来也一样，于是两个类就相等了。

所以等价类有这么几个特点：

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  每个元素至少属于自己的等价类；
\item
  任意两个等价类，要么完全一样，要么彻底不交；
\item
  所有等价类拼起来就是整个原集合。
\end{enumerate}

这其实就是集合的一个划分。反过来，你随便给集合一个划分，也能自然定义出一个等价关系：两个元素等价当且仅当它们在同一块里。所以``等价关系''和``把集合分组''，说的是同一个结构，只是语言不一样。

所有等价类放在一起构成的集合叫商集，记作 \(A/{\sim}\)。商集的元素不再是原来的单个点，而是一整类一整类的对象。比如有理数就可以从整数对 \((p,q)\)（\(q\ne0\)）出发来构造：规定 \((p,q)\sim(r,s)\) 当且仅当 \(ps=qr\)，这样表示同一个比值的整数对就归到同一个等价类里了——为什么用乘法判定而不用除法？就是为了不预先假设``比值''这个东西已经存在。构造新的数学对象的时候，等价关系就是用来声明``哪些不同的表示其实是同一个东西''的。

\subsection{传递性是最常被漏掉的条件}

在实数上定义 \(xRy\) 表示 \(|x-y|\le1\)。这个关系自反、对称，但不传递：\(0R0.8\) 且 \(0.8R1.6\)，但 0 和 1.6 就不满足了。所以``距离足够近''一般不能直接当等价关系用，因为一连串的局部接近可以累积成很远的距离。

\subsection{闭包是在原关系上补齐一条规则}

有时手里的关系还不具备我们需要的性质，但又不想删掉已经确认的有序对。这时可以在原关系上补进一些有序对，直到某条规则成立。如果存在一个包含 \(R\)、满足指定性质的最小关系，就称它为 \(R\) 关于该性质的\textbf{闭包}。这里的``最小''是按集合包含来讲的：除了为了满足规则而不得不补的有序对，什么也不多加。下面用到的自反、对称与传递三种闭包都存在。

以传递性为例。设集合里有 \(a,b,c\)，原关系只有

\[
R=\{(a,b),(b,c)\}.
\]

它还不传递，因为已经有 \(aRb\) 和 \(bRc\)，却没有 \(aRc\)。把 \((a,c)\) 补进去以后得到

\[
R^+=\{(a,b),(b,c),(a,c)\},
\]

这就是这个例子的传递闭包。若关系更复杂，就继续补上所有能沿关系链到达的有序对：只要存在

\[
x=x_0Rx_1R\cdots Rx_n=y,
\]

传递闭包里就令 \(xR^+y\)。换成图的语言，原关系记录直接相连的边，传递闭包记录``能不能沿一条或多条边走到''。因此可以简写为

\[
R^+=\bigcup_{n\ge1}R^n,
\]

其中 \(R^n\) 表示连续走 \(n\) 步能够连接的有序对。如果还把每个 \((x,x)\) 都补进去，就得到自反传递闭包，常记作 \(R^*\)。类似地，自反闭包会补齐所有 \((x,x)\)，对称闭包会在有 \((x,y)\) 时补上 \((y,x)\)。所以``闭包''不是一种固定操作，必须说明究竟要补齐哪种性质。

闭包只是在形式上补全规则，并不保证补出来的有序对仍符合最初那句话的现实含义。对前面的``距离不超过 1''做传递闭包之后，\(0\) 可以经过 \(0.8\) 连到 \(1.6\)，再继续一段段走到任意远的实数；得到的已经不是日常所说的``彼此接近''。

客户记录去重会遇到同一个陷阱。若规定``姓名和地址的相似度超过阈值''就算同一个人，那么记录 \(A\) 可能像 \(B\)，\(B\) 也像 \(C\)，而 \(A\) 与 \(C\) 已经差得很远。直接把相似关系的传递闭包当成身份分组，会让一条条局部匹配把不同的人逐渐并到一起。要得到真正的客户身份分组，必须再引入稳定标识、可核查的合并规则，或者明确接受这种``桥接''后果；相似关系本身不会自动变成等价关系。

类似的，``拥有共同的朋友''也不传递。看到``相似''``接近''``相关''这类词的时候，别想当然地觉得是等价关系，三条性质得一条一条过。
