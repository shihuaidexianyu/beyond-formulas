\section{偏序与可比较性}

\subsection{``有先后''不代表任何两个对象都能比较}

平时说``大小关系''，我们很容易默认随便拿两个出来都能比个高低。但很多关系不是这样的：集合包含、任务依赖、整数整除——\(\{1\}\) 和 \(\{2\}\) 谁也不包含谁；任务 A 和任务 B 可能互不依赖；2 不整除 3，3 也不整除 2。

集合 \(P\) 上的关系 \(\preceq\)，如果满足下面三条：

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  自反性：\(x\preceq x\)；
\item
  反对称性：\(x\preceq y\) 且 \(y\preceq x\) 的话，一定有 \(x=y\)；
\item
  传递性：\(x\preceq y\) 且 \(y\preceq z\) 的话，一定有 \(x\preceq z\)，
\end{enumerate}

就叫偏序，\((P,\preceq)\) 叫偏序集。

注意反对称不是``不对称''。它允许自己跟自己有关系，只是禁止两个不同的对象互相排在对方前面。等价关系靠对称性把对象聚成类，偏序则靠反对称性给对象排出层次结构。

\subsection{两个典型例子}

幂集 \(\mathcal P(S)\) 上的包含关系 \(\subseteq\) 就是一个偏序。如果 \(A\subseteq B\) 且 \(B\subseteq A\)，按外延性自然有 \(A=B\)。但一般来说两个集合不一定可比。

正整数上的整除关系 \(a\mid b\) 也是偏序：\(a\mid a\) 没问题；如果 \(a\mid b\) 且 \(b\mid a\)，因为都是正整数，所以 \(a=b\)；传递性来自乘法的性质。但如果把论域换成全体整数，反对称就不成立了——\(1\mid-1\) 且 \(-1\mid1\)，但 \(1\ne-1\)。所以一个关系是不是偏序，不光看符号怎么写，还得看论域是什么。

如果偏序集中随便两个元素都能比较，也就是总有 \(x\preceq y\) 或者 \(y\preceq x\)，那就叫全序或者线性序。实数上的 \(\le\) 就是全序，而集合包含一般只是偏序。

\subsection{最大、极大、上界不是同一个概念}

设 \(A\) 是偏序集里的一个子集。几个容易搞混的概念：

\begin{itemize}
\tightlist
\item
  最大元 \(m\in A\)：对每个 \(a\in A\) 都有 \(a\preceq m\)。如果存在的话是唯一的——假设有两个最大元 \(m_1,m_2\)，那 \(m_1\preceq m_2\) 且 \(m_2\preceq m_1\)，由反对称性它们相等。
\item
  极大元 \(m\in A\)：\(A\) 里没有比它严格更大的元素。可以同时有好几个互不可比的极大元。
\item
  上界 \(u\)：不一定在 \(A\) 里面，只要所有 \(a\in A\) 都满足 \(a\preceq u\) 就行。
\item
  最小上界：所有上界里面最小的那个，也叫上确界。
\end{itemize}

拿正整数按整除排序，看子集 \(A=\{2,3,6\}\)：6 是最大元；2 和 3 都不是最大元，因为它们各自都压不住所有元素。如果换成 \(A=\{2,3\}\)，那 2 和 3 都是极大元，但没有最大元。注意这里比的不是数字大小，是整除关系。

对集合族按包含排序的话，并集通常就是最小上界，交集就是最大下界。实分析里的上确界、算法里的依赖图、优化里的可行集包含，用的其实都是同一套偏序语言。

\subsection{Hasse 图只画必要的比较}

有限偏序集可以用 Hasse 图来画。先记 \(x\prec y\) 表示严格偏序（\(x\preceq y\) 且 \(x\ne y\)）；如果 \(x\prec y\)，而且中间找不到一个 \(z\) 满足 \(x\prec z\prec y\)，那就把 \(y\) 画在 \(x\) 上面连一条线。自反的边和能靠传递性推出来的长边都省掉不画。

Hasse 图不是随便把有向图简化一下。看图的时候，只要有向上的路径就连通着偏序关系；两个点之间找不到上下路径，就说明它们不可比。像``有多个极大元但没有最大元''这种事，看一眼图就清楚了。

数据处理流水线可以把这幅图变成实际的调度表。设清洗原始数据之后，一路生成模型特征，另一路生成质量报告；两条任务都依赖清洗，却互不依赖，所以它们不可比，也可以并行。最终发布同时依赖特征和报告，于是位于两者上方。Hasse 图只需画这些直接依赖：``清洗早于发布''已经能沿路径推出，不必再补一条长边。这里把不可比误解成``顺序尚未决定''会多加无谓约束；它真正表示的是，当前依赖关系没有要求谁先谁后。
