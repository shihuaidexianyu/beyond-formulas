\section{MDP 把决策问题写成随机过程}

一台机器人站在仓库通道口。向左可能更快到货架，却有拥堵风险；向右稳定但绕路；原地等待也许能避开即将经过的叉车。若只比较眼前一步，``向左少走两米''似乎最好，但这一步会把机器人带到新的位置，改变下一步能选什么，也改变未来会遇到怎样的风险。决策问题的核心不是挑一个动作，而是挑一条会不断改变后续处境的规则。

Markov 决策过程（MDP）把这条因果链压成五个对象。它不是一种算法，而是对``状态怎样演化、什么算好''的数学约定；模型写错，后面的 Bellman 方程算得再精确也只会精确回答错问题。

\subsection{五个对象与一条交互链}

一个折扣 MDP 记为

\[
(\mathcal S,\mathcal A,P,R,\gamma).
\]

\begin{itemize}
\tightlist
\item
  状态空间 \(\mathcal S\)：当前决策所需的信息；
\item
  动作空间 \(\mathcal A\)：智能体可以选择的输入；
\item
  转移核 \(P(s'\mid s,a)\)：在状态 \(s\) 做动作 \(a\) 后到达 \(s'\) 的概率；
\item
  奖励 \(R\)：一次转移带来的即时评价，可写成 \(R(s,a,s')\) 或其条件期望；
\item
  折扣率 \(0\le\gamma<1\)：未来奖励相对当前奖励的权重。
\end{itemize}

交互在离散时间上循环：

\[
S_t
\xrightarrow{\ A_t}
(R_{t+1},S_{t+1})
\xrightarrow{\ A_{t+1}}
\cdots .
\]

这里奖励记为 \(R_{t+1}\)，因为它由时刻 \(t\) 的状态与动作触发，并与下一状态一起返回。文献也常写 \(R_t\)；只要整篇索引一致，两种约定没有本质差异。

若去掉动作，转移只剩 \(P(s'\mid s)\)，MDP 就退化为``Markov 性质、转移矩阵与多步演化''中的 Markov 链。动作的加入使转移矩阵不再固定：策略每选一次动作，就选择了下一步概率规律的一部分。

\subsection{Markov 性质不是``世界没有记忆''}

MDP 要求

\[
\Prob(S_{t+1},R_{t+1}\mid S_0,A_0,\ldots,S_t,A_t)
=
\Prob(S_{t+1},R_{t+1}\mid S_t,A_t).
\]

它说的是：给定当前状态与动作后，更早的历史不再额外改变下一步分布。世界当然可以有记忆，但状态必须把对未来仍有影响的那部分历史携带进来。

恒温控制里若状态只记当前温度，却不记加热器内部仍在升高的热惯性，同样温度下``刚关机''与``刚开机''的未来不同，Markov 性质便失败。修复方式不是假装历史无关，而是把加热器温度、上一动作或必要的历史摘要加入状态。状态是为预测未来而选的信息压缩，不是传感器读数的随意列表。

若真实状态无法完整观测，智能体只看到观测 \(O_t\)，问题成为部分可观测 MDP。此时可用整个历史、循环状态或对隐藏状态的信念分布作为决策依据。把非 Markov 观测直接当状态，标准 Bellman 递归的前提就已经破坏。

\subsection{策略把状态变成动作分布}

策略 \(\pi(a\mid s)\) 给出在状态 \(s\) 选择动作 \(a\) 的概率。确定性策略是特殊情形，可写成 \(a=\pi(s)\)。策略一旦固定，动作的不确定性被吸收到转移中：

\[
P^\pi(s'\mid s)
=
\sum_a \pi(a\mid s)P(s'\mid s,a),
\]

于是受控过程重新变成一条普通 Markov 链。评价策略，就是评价这条由策略诱导出的随机过程。

策略依赖当前状态而不显式依赖时间，称为平稳 Markov 策略。对无限时域折扣 MDP，有限状态动作下存在最优的确定性平稳策略；有限时域问题则通常需要把剩余时间写进策略，或把时间本身并入状态。不能把无限时域结论不加说明地搬到所有任务。

\subsection{从即时奖励到长期回报}

只最大化下一步奖励会变得短视。定义从时刻 \(t\) 起的折扣回报

\[
G_t
=R_{t+1}+\gamma R_{t+2}+\gamma^2R_{t+3}+\cdots
=\sum_{k=0}^{\infty}\gamma^kR_{t+k+1}.
\]

睡前决定``再看一集''还是``现在休息''，就有即时奖励与延迟后果的冲突：多看一集立刻带来放松，少睡的代价却要到起床和第二天工作时才出现。若评价只停在下一步，未来疲劳根本进不了今天的选择；回报 \(G_t\) 则把这些随后发生的后果沿时间轴带回当前。折扣率也因此不是纯粹的计算旋钮：把 \(\gamma\) 调小，等于在评价标准里真的减轻了明天相对于现在的分量，而不只是让算法跑得快一些。

若奖励有界且 \(\gamma<1\)，这个级数绝对收敛。折扣有三种常见解释：

\begin{itemize}
\tightlist
\item
  时间偏好：越早得到的奖励越重要；
\item
  随机终止：每一步以概率 \(1-\gamma\) 结束，\(\gamma^k\) 是任务活过 \(k\) 步的概率；
\item
  数学正则化：它使 Bellman 算子成为压缩映射，保证值函数唯一并让迭代收敛。
\end{itemize}

折扣不是无害的技术参数。把 \(\gamma\) 调小会真的改变最优策略，使它更重视近处；接近 \(1\) 则延长有效规划时域，但让远期估计误差传播更久、收敛更慢。对会自然终止且期望长度有限的 episodic 任务，可以取 \(\gamma=1\)；对持续任务则常改用长期平均奖励，理论与算法需要重新表述。

\subsection{两个值函数回答两种问题}

策略 \(\pi\) 的状态值定义为

\[
V^\pi(s)
=
\E_\pi\left[G_t\mid S_t=s\right],
\]

表示``到达状态 \(s\) 后一直遵循 \(\pi\)，预期能得到多少回报''。动作值定义为

\[
Q^\pi(s,a)
=
\E_\pi\left[G_t\mid S_t=s,A_t=a\right],
\]

表示``当前先强制做动作 \(a\)，之后再遵循 \(\pi\)''的预期回报。二者关系为

\[
V^\pi(s)=\sum_a\pi(a\mid s)Q^\pi(s,a).
\]

状态值适合评价处境，动作值适合直接比较选择。若模型已知，从 \(V\) 可用一步前瞻比较动作；若模型未知，\(Q\) 的优势是选动作时无需显式知道转移概率——这会成为 Q 学习的入口。

\subsection{奖励设计会改变问题，而不只是加速学习}

仓库机器人若每走一步奖励 \(-1\)、到达目标奖励 \(0\)，最大化回报等价于偏好更短路径；若碰撞罚得不够重，机器人可能选择高风险捷径；若只在到达时给正奖励而没有步数代价，所有最终能到达的路线可能在 \(\gamma=1\) 下同样好。奖励不是模型的装饰，它定义了最优。

奖励塑形可以缓解稀疏反馈，却可能产生钻空子行为：若按``接近目标的距离减少''给奖励，智能体也许在两个位置之间来回刷分；若只奖励点击率，推荐系统可能牺牲长期满意度。只有特定的势函数塑形

\[
F(s,a,s')=\gamma\Phi(s')-\Phi(s)
\]

能在标准条件下保持最优策略不变。任意加奖励项都可能重写任务，不能把``学得更快''与``目标没变''混为一谈。

\subsection{模型已知与模型未知的分界}

若 \(P\) 与期望奖励都已知，可以对所有下一状态精确求和，直接解 Bellman 方程，这属于动态规划。若它们未知但可以交互采样，便只能观察

\[
(S_t,A_t,R_{t+1},S_{t+1})
\]

这样的单步样本，用样本平均或随机逼近替代模型期望，这才进入强化学习。两者优化的是同一个 MDP，差别在信息条件与计算方式。

库存系统说明``可以观察单步样本''也需要解释。设 $I_t$ 是现有库存，$q_t$ 是订单，潜在需求为 $D_t$，则下一期库存近似满足

\[
I_{t+1}=(I_t+q_t-D_t)_+.
\]

数据库真正记录的销量却只有

\[
S_t=\min\{D_t,I_t+q_t\}.
\]

订单 $q_t$ 不仅改变下一状态，还决定缺货时有多少需求无法被观察。若状态没有记录缺货标志、在途订单和历史促销，转移样本就不能支持所写的 Markov 模型。称算法为 model-free 并不会让删失需求或遗漏状态自动消失；它只表示更新时没有显式使用一张已知转移表。

MDP 还默认转移与奖励规律在学习期间大致稳定。若环境因其他学习者、季节或设备老化持续变化，过去样本与当前规律不再同分布，固定 Bellman 方程本身就在移动。此时需要在线追踪、显式建模非平稳性或重新定义状态；简单地让算法训练更久不能修复一个正在改变的问题。
