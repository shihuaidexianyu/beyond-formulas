\section{最大值原理：把最优轨线变成两点边值}
\label{sec:16-Control-and-Reinforcement-Learning/01-Optimal-Control/02}

\begin{quote}
\textbf{先抓住这一点：}最大值原理把``整条控制函数不可被局部扰动改进''变成状态、协态与逐时刻最优条件。它通常给必要条件，不自动给全局最优性。
\end{quote}

上一节解决了``稳住''，留下的是``最优''：到达目标的方式有无穷多种，哪一种代价最小？本节把这个问题形式化，并用变分思想推出最优解必须满足的条件——Pontryagin 最大值原理。结论的形态出人意料：一个``在函数空间上寻优''的问题，最终被化成一组微分方程加两端的边界条件。

\subsection{把``控制好''写成目标泛函}

最优控制问题由四个成分构成：

\begin{itemize}
\tightlist
\item
  状态方程 \(\dot x=f(x,u)\)，初值 \(x(0)=x_0\) 给定；
\item
  控制约束：每一时刻 \(u(t)\in U\)（例如功率上限 \(\abs{u}\le u_{\max}\)）；
\item
  终端代价 \(\Phi\bigl(x(T)\bigr)\)：对``终点停在哪''的奖惩；
\item
  运行代价 \(L(x,u)\)：对``过程中怎样走''的计费。
\end{itemize}

骑车赶一班固定时间发车的列车，就已经包含这四部分。状态可以记录离车站还有多远和当前速度，控制 \(u\) 是蹬车或刹车的力度，体力、制动能力与路面条件共同给出控制上限。到发车时还没抵达、或冲到站前仍来不及停下，可以写进终端代价 \(\Phi\)；一路消耗的体力与过猛加减速造成的不适，则写进运行代价 \(L\)。若只在乎``别误车''，最优方案可能一路猛骑；若还重视体力和安全，同一段路就会得到另一条轨线。数学不会替人决定什么叫好，它只在评价标准写清后寻找最优。

目标是在满足约束的输入中，使总代价

\[
J(u)=\Phi\bigl(x(T)\bigr)+\int_0^T L\bigl(x(t),u(t)\bigr)\,dt
\]

最小。注意未知量是一整条函数 \(u(\cdot)\)，远比一个数或一个向量复杂——这是无穷维空间上的优化，微积分里``导数为零''的套路不能照搬。

\subsection{扰动最优轨线：变分给出必要条件}

替代思路是变分。设 \(u^\ast\) 最优，给它施加一个小扰动 \(u^\ast+\varepsilon\eta\)。轨线随之偏离 \(x^\ast+\varepsilon\,\delta x+o(\varepsilon)\)，其中一阶偏差满足线性化的变分方程

\[
\dot{\delta x}=f_x\,\delta x+f_u\,\eta,\qquad \delta x(0)=0,
\]

下标表示偏导数（沿 \(u^\ast,x^\ast\) 取值）。最优性要求任何扰动都不能让 \(J\) 变小：一阶变分 \(\delta J\ge0\)。

直接计算 \(\delta J\) 的麻烦在于 \(\delta x\) 要解一个微分方程才能得到。技巧来自一个看似多此一举的观察：沿任何轨线，\(\dot x-f(x,u)=0\) 恒成立，乘上任何函数 \(\lambda(t)\) 再积分，加进 \(J\) 都不改变它的值。取这个自由乘子，用分部积分把 \(\delta x\) 的导数项搬掉，再专门选择 \(\lambda\)，使 \(\delta x\) 的各项系数恰好为零——剩下的只有 \(\eta\) 的项。这就是协态（costate，也称 adjoint）的由来：它是为``解耦''状态偏差而引入的拉格朗日乘子。

\subsection{Hamiltonian、协态与最大值原理}

把上述计算的结果集中陈述。定义 Hamiltonian

\[
H(x,u,\lambda)=\lambda\cdot f(x,u)-L(x,u).
\]

\textbf{Pontryagin 最大值原理}（必要条件的核心内容）：若 \(u^\ast\) 最优、\(x^\ast\) 为对应轨线，则存在协态函数 \(\lambda(\cdot)\)，满足伴随方程与终端条件

\[
\dot\lambda=-\frac{\partial H}{\partial x},
\qquad
\lambda(T)=-\frac{\partial \Phi}{\partial x}\bigl(x^\ast(T)\bigr),
\]

并且在每一时刻 \(t\)，最优控制使 Hamiltonian 在约束集上取到最大：

\[
u^\ast(t)\in\argmax_{u\in U}\,H\bigl(x^\ast(t),u,\lambda(t)\bigr).
\]

符号约定要交代一句：另一种常见写法是 \(H=L+\lambda\cdot f\)、逐时刻取\textbf{最小}，协态差一个符号，两种陈述完全等价。本节沿用``最大值''的原始命名；下一节采用后一种约定，那时协态正好是值函数的梯度。

最大值原理把``整段时间轴上的全局寻优''压缩成``每一时刻在 \(U\) 上的逐点寻优''——后者通常只是有限维的代数问题，有时甚至可以直接读出答案（见下文 bang-bang）。代价是必须同时解出 \(x^\ast\) 与 \(\lambda\)，而它们互相缠绕。

\subsection{协态是状态的边际价值}

协态不是纯粹的技巧产物，它有清晰的经济学读法。终端条件里那个负号来自本节的最大化约定；换到 \(H=L+\lambda\cdot f\) 的最小化约定（协态变号）后，终端条件成为 \(\lambda(T)=\partial\Phi/\partial x\)，读法立刻干净：\(\lambda(t)\) 是``在时刻 \(t\) 给状态一个微小推动，最优总代价改变多少''——状态对剩余目标的边际价值，正如约束优化中拉格朗日乘子的影子价格。

伴随方程 \(\dot\lambda=-\partial H/\partial x\) 则说：边际价值沿最优轨线按``当前状态对未来代价的边际影响''的速率折旧。

于是最大化 \(H\) 的第一项 \(\lambda\cdot f\) 可读作：选运动方向时，不只看眼前的运行代价 \(L\)，还要看它把状态推向边际价值度量下的何处。

\subsection{落点：两点边值问题}

把三部分拼起来，看方程的结构：

\begin{itemize}
\tightlist
\item
  状态方程 \(\dot x=f\) 从\textbf{左端}初值 \(x(0)=x_0\) 向前走；
\item
  伴随方程 \(\dot\lambda=-\partial H/\partial x\) 从\textbf{右端}终端条件 \(\lambda(T)=-\partial\Phi/\partial x\) 向后走；
\item
  每个时刻的 \(u(t)\) 由逐点最大化与两者耦合。
\end{itemize}

条件分散在时间区间的两端：\(\lambda(0)\) 未知，无法单向积分；\(x\) 与 \(\lambda\) 互相依赖，也无法分别求解。这正是 \hyperref[sec:08-Numerical-Analysis/06-ODE/04]{``边值问题：从打靶到全局离散''} 研究的对象：打靶法猜一个 \(\lambda(0)\)、积分到 \(T\)、按终端残差修正猜测；多重打靶与配置法则把整条轨线离散后一次求解。最优控制把``寻优''的困难转化成``解方程''的困难，再转交给数值方法——每一步转化都有代价，但都换来了可计算性。

\subsection{Bang-bang：结构性结论的例子}

最大值原理有时不求解方程就能给出最优解的\textbf{形状}。经典例子是最速停车：\(\ddot y=u\)，\(\abs{u}\le1\)，要求从初始位置以最短时间到达原点并停住。运行代价 \(L\equiv1\)（按时间计费），Hamiltonian 为

\[
H=\lambda_1\,\dot y+\lambda_2\,u-1.
\]

\(H\) 对 \(u\) 是线性的，在 \([-1,1]\) 上的最大值必在端点取得：

\[
u^\ast(t)=\sign\,\lambda_2(t),
\]

控制只取 \(\pm1\) 两个极端值——要么全力加速，要么全力刹车，这就是 bang-bang 控制。再看伴随方程：\(\dot\lambda_1=0\)，\(\dot\lambda_2=-\lambda_1\)，故 \(\lambda_2\) 是时间的线性函数，至多过零一次——切换至多发生一次：先满推，再满刹。没有解出任何轨线，最优控制的完整结构（只取极端值、至多切换一次）已经从最大值原理中读了出来。这种``解的形状先于解本身''的结论，是逐点最大化带来的独有红利。

协态之所以从 \(t=T\) 倒着走，是因为它衡量``当前状态对未来剩余代价的边际影响''：未来终端代价先给出边界，再把价值向现在传播。若终点自由且 \(\Phi\equiv0\)，相应的自然终端条件是 \(\lambda(T)=0\)；若终点被硬约束，终端条件还会出现新的乘子，不能把自由终点公式原样套用。最大值原理给的是必要条件，不保证满足方程的驻定轨线就是全局最优；非凸动力学或代价下，还需比较候选解或补充充分性条件。
