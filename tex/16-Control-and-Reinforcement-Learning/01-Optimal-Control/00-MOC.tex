本单元先站在最强假设上：系统的动力学 \(\dot x=f(x,u)\) 与代价函数都已知，问题只剩一个——怎样把``控制好''写成数学，并解出来；最后再撤掉``完整状态可见''与``模型天然已知''两项假设。六篇按从反馈直觉、最优性条件到可计算算法和数据辨识的顺序组织：

\begin{enumerate}
\tightlist
\item
  从开环到闭环反馈：为什么事先算好的计划对模型误差与扰动脆弱，反馈怎样用误差信号稳住系统，PID 三项各治什么病、各带什么副作用。
\item
  最大值原理：把最优轨线变成两点边值：从``最优轨线不能被扰动改进''的变分思想出发，引出 Hamiltonian 与协态，把寻优问题化为微分方程的两点边值问题。
\item
  HJB 方程与动态规划：定义``从任意状态出发的最优剩余代价''为值函数，推出 Hamilton--Jacobi--Bellman 方程，在 LQR 上完整算到底，并撞见维数灾难。
\item
  LQG 把状态估计与线性反馈分开：在线性 Gaussian 条件下，用 Kalman filter 从带噪观测估计状态，再用 LQR 反馈估计状态，并说明分离原理的边界。
\item
  DDP 用局部二次模型改进整条轨迹：沿名义轨迹展开动力学与代价，通过一次反向递推得到局部反馈律，再用前向 rollout 检验改进。
\item
  系统辨识从输入输出数据估计动力学：由持续激励、闭环偏差与潜在状态说明为什么``有许多数据''仍不等于模型可辨认，并用滚动预测和控制结果验证模型。
\end{enumerate}

记号在全单元统一：状态记 \(x\)，控制输入记 \(u\)，值函数记 \(V\)。HJB 的维数灾难不是故事的终点：LQG 利用线性 Gaussian 结构获得有限维递推，DDP 只在一条轨迹附近建立局部模型，系统辨识则先从实验数据取得动力学。当状态空间大到无法逐点解方程、模型又只能通过交互学习时，才进入下一单元的强化学习。

\subsection{怎么读这一章}

首次阅读先抓``反馈、目标、价值''三层：第一篇建立反馈直觉，第二与第三篇对照``一条候选最优轨线的必要条件''和``所有状态的最优剩余代价''。LQG、DDP 与系统辨识是三种结构化出路，分别依赖线性 Gaussian、局部二次近似与有信息量的数据。公式可后置，但每种方法的假设、局部性和可辨认性不能后置。
