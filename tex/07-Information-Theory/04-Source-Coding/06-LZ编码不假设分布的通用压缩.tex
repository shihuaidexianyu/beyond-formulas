\section{LZ 编码：不假设分布的通用压缩}
\label{sec:07-Information-Theory/04-Source-Coding/06}

\begin{quote}
\textbf{先抓住这一点：}数据里如果出现过一段相同内容，与其再写一遍，不如告诉解码器``回到前面某处复制这么长''。LZ 一边读取一边建立这种共享记忆，因此不必事先拿到真实分布。
\end{quote}

前面几篇的理想构造大多从已知信源模型出发：Huffman 与算术编码需要概率，率失真函数也依赖信源分布。但现实中的文本、代码和日志往往只能边读边摸索规律。LZ（Lempel--Ziv）编码直接利用已经出现的字符串。DEFLATE、LZW 等具体家族被 gzip、PNG、GIF 以及部分 ZIP 格式采用，但这些文件格式还包含其他协议与编码步骤。

\subsection{先感受困难：分布未知，码表从何而来}

Huffman 编码的第一步是统计每个符号的频率，然后构造最优码表。若分布未知，这个流程失效：没有频率表，码表无从谈起。

更根本的问题是：许多数据的统计规律不是``符号频率''，而是``子串重复''。英文文本中``the''反复出现，代码中``for (''反复出现，日志中时间戳格式反复出现。这些模式跨越多个符号，单符号频率无法捕捉。

LZ 编码的突破口是：不显式估计分布，而是把数据中出现的重复子串替换为``指向之前出现位置的指针''。重复模式被压缩，分布知识隐含在指针结构里。

\subsection{LZ77：滑动窗口中的指针替换}

LZ77 维护一个滑动窗口：已编码的历史数据（搜索缓冲区）与待编码的前瞻数据。编码时，在当前位置寻找窗口内最长的已出现子串，输出三元组

\[
(\text{偏移量},\text{长度},\text{下一个新符号}).
\]

例如编码``abcabcX''：前三个符号``abc''尚无可用历史匹配，可以作为字面量输出；到第二个``abc''时，可用偏移 3、长度 3 指回前一段，再把随后出现的新符号 ``X'' 一并写入。不同 LZ77 变体对匹配与新符号有不同格式，三元组只是便于理解的一种版本。

LZ77 的压缩率依赖窗口大小与重复模式的距离：重复越近、越长，压缩越高。文本、代码、日志的重复模式通常在几千字节内，所以几 KB 到几十 KB 的窗口效果显著。

\subsection{LZ78 与 LZW：字典生长}

LZ78 把窗口换成显式字典：从零开始，每遇到一个``已见前缀+新符号''，就把新组合加入字典。编码输出``字典索引+新符号''，解码端同步重建同一本字典。

LZW（LZ78 的改进）进一步优化：字典初始包含所有单符号，每次遇到已见子串就延伸，输出最长已见子串的索引，把``子串+下一个符号''加入字典。编码只输出索引序列，解码端用同样的规则重建字典并还原数据。

GIF 与早期 ZIP 用 LZW，gzip 用 LZ77+Huffman（DEFLATE）。两者都是 LZ 思想的工程实现。

\subsection{通用性：不假设分布，渐近最优}

LZ 家族的重要性质是通用性：对平稳遍历的有限字母表信源，经典 LZ 方案在相应定理条件下可使渐近码率趋近熵率，而无需事先知道分布。这里的限定词不能删；对非平稳、非遍历或有限长度数据，结论需要另外分析。

直觉：LZ 的字典就是信源的``经验模型''，重复子串的指针就是``最优码''的近似。随着数据增多，字典覆盖越来越多的常见模式，码率逼近理论极限。

但通用性不免费：LZ 对短序列或高熵数据（接近随机）效果差——没有足够的重复模式可学。加密数据、随机数、已压缩数据的 LZ 输出可能比输入还长（指针开销超过压缩收益）。

\subsection{LZ 与熵编码的分工}

实际压缩器常把 LZ 与熵编码串联：

\begin{enumerate}
\tightlist
\item
  LZ 阶段：去除重复模式，输出``字面符号+指针''序列；
\item
  熵编码阶段：对 LZ 输出再做 Huffman 或算术编码，利用符号频率进一步压缩。
\end{enumerate}

gzip 的 DEFLATE 就是这个结构：LZ77 去重，Huffman 编码字面量与指针长度。LZ 处理``结构冗余''（重复子串），熵编码处理``统计冗余''（符号频率不均）。两者互补，覆盖不同类型的冗余。

\subsection{边界清单}

\begin{itemize}
\tightlist
\item
  LZ 假设数据有重复模式；对真正随机的数据（加密输出、白噪声），LZ 可能膨胀而非压缩。
\item
  LZ77 的窗口大小是硬约束：重复距离超过窗口，模式无法捕捉。大窗口提高压缩率但增加内存与搜索时间。
\item
  通用性是渐近性质：无限长序列上 LZ 码率趋近熵率，但有限序列的冗余（指针开销、字典建立成本）可能显著。短文件用 LZ 可能不如直接存储。
\item
  基础 LZ 解析不会像 Huffman 那样先建立单符号概率表，但高频符号会形成重复串，仍可能被字典或指针间接利用。实际系统再对字面量、长度和距离做熵编码，是为了继续利用输出符号的不均匀频率。
\end{itemize}

\subsection{落点}

LZ 编码把压缩从``已知分布的最优码''推进到``未知分布的通用码''。它放弃了显式概率模型，用重复模式的指针结构隐式学习分布，实现了不假设分布的渐近最优压缩。这是 gzip、PNG、ZIP 的理论基础，也是``先估计分布、再设计码表''与``边编码边学习''两条路线的分野。
