\section{AEP、典型集与信源编码定理}

\subsection{为什么长块能突破逐符号的整数障碍}

回到上一节末尾的 Bernoulli\((0.99)\) 信源：单符号熵约为 \(0.081\) bit，但任何二元单符号前缀码都要给两个结果各至少一 bit。若把 \(n\) 个符号一起看，可能序列共有 \(2^n\) 个，似乎问题反而更大了。

关键转折是：这 \(2^n\) 个序列并不平等。绝大多数概率集中在一小群统计行为稳定的序列上。长块压缩先找出承载几乎全部概率的``有效世界''，再只为这个世界分配短索引，无须对每个序列逐一讨价还价。

\subsection{块自信息就是样本均值}

设 \(X_1,\ldots,X_n\) 独立同分布。块概率分解为

\[
p(X^n)=\prod_{i=1}^np(X_i).
\]

因此单位符号块自信息为

\[
-\frac1n\log p(X^n)
=\frac1n\sum_{i=1}^n[-\log p(X_i)].
\]

右边是 i.i.d. 自信息随机变量的样本均值，其期望为 \(H(X)\)。当熵有限时，大数定律给出

\[
\boxed{-\frac1n\log p(X^n)\longrightarrow H(X)}
\]

几乎必然成立。这就是渐近等分性质（AEP）。所谓``等分''表示绝大部分实际出现的序列，在一阶指数尺度上都有

\[
p(x^n)\approx2^{-nH(X)}.
\]

\subsection{典型集把极限定理变成可数的对象}

给定 \(\varepsilon>0\)，定义弱典型集

\[
A_\varepsilon^{(n)}
=\left\{x^n:
\left|-\frac1n\log p(x^n)-H(X)\right|<\varepsilon
\right\}.
\]

AEP 立即说明

\[
P\bigl(X^n\in A_\varepsilon^{(n)}\bigr)\to1.
\]

而每个典型序列的概率满足

\[
2^{-n(H+\varepsilon)}
<p(x^n)<
2^{-n(H-\varepsilon)}.
\]

总概率不超过 1，所以典型序列数至多约为 \(2^{n(H+\varepsilon)}\)；典型集又承载接近 1 的概率，而每个元素概率至多 \(2^{-n(H-\varepsilon)}\)，所以其数量至少约为 \(2^{n(H-\varepsilon)}\)。在指数阶上，

\[
|A_\varepsilon^{(n)}|\doteq2^{nH(X)}.
\]

这里的 \(\doteq\) 表示一阶指数率相同，不是有限 \(n\) 时严格相等。

\subsubsection{典型不等于``最可能''}

对偏置硬币，全部为 0 的序列可能是单个最可能序列，却通常不典型，因为它的经验频率离真实比例很远。典型集关心的是一个整体：每个成员未必排名最前，但整个集合承载几乎全部概率。

公平硬币则是特殊情形。每个长度 \(n\) 的序列概率都为 \(2^{-n}\)，熵为 1 bit，所以所有序列都是弱典型的；没有可压缩的低概率区域可以舍弃。

\subsection{可达性：高于熵的速率足够}

现在允许一个很小的块错误概率。编码器给每个典型序列分配一个不同索引，并用一个特殊码字处理非典型序列。因为典型集大小至多约为 \(2^{n(H+\varepsilon)}\)，只需大约

\[
n(H+\varepsilon)
\]

bit 就能索引它们。非典型事件的概率随 \(n\) 增大趋于零。

因此对任意速率 \(R>H(X)\)，只要块长足够大，就存在每符号不超过约 \(R\) bit、错误概率任意小的固定长度近无损码。

上一节的算术编码给出另一种更具操作性的理解：典型序列概率约为 \(2^{-nH}\)，对应区间宽度也约为 \(2^{-nH}\)，定位该区间自然需要约 \(nH\) bit。

\subsection{不可能性：低于熵的速率装不下典型世界}

若码率 \(R<H(X)\)，长度 \(nR\) 的二进制索引最多区分 \(2^{nR}\) 个序列。但典型世界有约 \(2^{nH}\) 个近等概率序列。即使精心挑选最有利的 \(2^{nR}\) 个典型序列，它们能覆盖的概率至多大致为

\[
2^{nR}\,2^{-n(H-\varepsilon)}
=2^{-n(H-R-\varepsilon)}.
\]

取 \(\varepsilon<H-R\)，这个量趋于零；再加上概率趋于零的非典型部分，也无法让成功概率接近一。于是低于熵的固定码率不可能实现渐近无误恢复。

这给出离散无记忆信源的 Shannon 信源编码定理：

\[
\boxed{H(X)\text{ 是近无损压缩的临界速率。}}
\]

它不是说每个有限文件都恰好能压到 \(nH(X)\) bit，也不是说任何数据都能压缩；它说在信源模型、长块和允许错误概率趋零的渐近设定下，高于熵可达，低于熵不可达。

\subsection{零错误、近无损与模型假设}

不同编码结论不要混在一起：

\begin{itemize}
\tightlist
\item
  逐符号前缀码要求零错误，平均长度满足 \(H\le L<H+1\)；
\item
  长块变长码可把每符号平均冗余压到零，同时保持唯一可译；
\item
  典型集构造通常表述为固定长度近无损码，允许非典型块产生小概率错误；
\item
  实际通用压缩还要学习未知模型，并支付模型描述和有限样本成本。
\end{itemize}

若信源有依赖，不能把边缘熵 \(H(X_1)\) 直接当作极限。应使用熵率

\[
\bar H=\lim_{n\to\infty}\frac1nH(X_1,\ldots,X_n),
\]

并在适当的平稳、遍历条件下由 Shannon--McMillan--Breiman 定理获得相应 AEP。若信源非平稳或非遍历，单一典型集图景可能需要修改。
