\section{算术编码：从符号码到序列区间}

\subsection{不再给每个符号单独发码字}

Huffman 编码必须为每个符号安排一个整数长度的码字。算术编码改为把整段序列表示成 \([0,1)\) 中越来越小的一个区间，不再逐个输出符号码。分数 bit 可以在许多符号之间累积，整数舍入只在整段末尾发生一次。

设字母表为 \(A,B,C\)，概率分别为

\[
P(A)=0.5,\qquad P(B)=0.3,\qquad P(C)=0.2.
\]

先按累积概率把 \([0,1)\) 分为

\[
A:[0,0.5),\quad B:[0.5,0.8),\quad C:[0.8,1).
\]

若第一个符号是 \(B\)，当前区间缩到 \([0.5,0.8)\)。第二个符号是 \(A\)，就在这个宽度为 \(0.3\) 的区间内部按同样比例细分，选取前一半：

\[
[0.5,0.65).
\]

第三个符号是 \(C\)，再取当前区间最后 \(20\%\)，得到

\[
I(BAC)=[0.62,0.65).
\]

区间宽度为

\[
0.65-0.62=0.03=0.3\times0.5\times0.2=P(BAC).
\]

这不是例子的巧合。每一步把宽度乘以当前条件概率，因此对一般序列

\[
|I(x^n)|=\prod_{i=1}^nP(x_i\mid x^{i-1})=P(x^n).
\]

模型可以随着历史改变，算术编码并不要求符号独立；编码器和解码器只需在每一步使用完全一致的条件概率。

\subsection{区间怎样变成有限比特串}

编码器最终要发送一个能够唯一定位到目标区间的二进制前缀。长度为 \(k\) 的比特前缀对应一个宽度为 \(2^{-k}\) 的 dyadic 区间。

在上例中，二进制前缀 \(101000\) 对应

\[
[0.101000_2,0.101001_2)
=[0.625,0.640625),
\]

它完全落在 \([0.62,0.65)\) 内，所以在已知消息终止规则的前提下，可用这个前缀表示 \(BAC\)。

为什么要求 dyadic 小区间完全包含于目标区间，而不仅是随便找一个落在其中的点？因为一个有限前缀代表的是所有以它开头的二进制小数，也就是一整个区间。完整包含才能保证无论后续如何补零或处理边界，都不会落入另一个序列区间。

对宽度为 \(w=P(x^n)\) 的任意非空区间，总能找到一个长度不超过

\[
-\log_2w+2
\]

的二进制前缀，其 dyadic 区间完全落在其中。理由是：取 \(k=\lceil-\log_2w\rceil+1\)，则 \(k<-\log_2w+2\) 且 \(2^{-k}\le w/2\)；宽度 \(2^{-k}\) 的 dyadic 区间把 \([0,1)\) 密铺成相邻格子，目标区间若连一个完整格子都不包含，至多横跨两个格子的各一段，总宽不足 \(2\cdot2^{-k}\le w\)，矛盾。因此算术码的长度只比序列的理想自信息

\[
-\log_2P(x^n)
\]

多一个常数量级，而不是每个符号都多不到一 bit。

\subsection{为什么它逼近交叉熵}

若数据实际来自 \(P\)，编码模型使用 \(Q\)，序列理想长度是

\[
-\log_2Q(X^n).
\]

对 i.i.d. 情形取期望并除以 \(n\)，得到

\[
\frac1n\E_P[-\log_2Q(X^n)]=H(P,Q).
\]

终止造成的常数 bit 除以 \(n\) 后趋于零。因此长序列上的算术编码把模型的交叉熵直接转化为实际压缩率：模型越接近真实条件分布，码越短；模型漏掉真实可能符号并给零概率时，编码过程会直接中断。

这也解释了现代压缩系统中``概率模型''和``熵编码器''的分工。模型负责给出条件概率，算术编码或范围编码负责把这些概率近乎无损地兑换成比特。压缩效果的主要智能往往在模型，而不在最后的区间算术。

\subsection{解码为什么可逆}

解码器收到一个位于最终区间内的数，并知道同一个概率模型与终止规则。它查看该数落在初始哪个子区间，得到第一个符号；然后把该子区间重新缩放到 \([0,1)\)，再判断落在哪个条件子区间，逐步恢复序列。

上例中 \(0.625\) 位于 \(B\) 区间；在 \(B\) 区间内的相对位置为

\[
\frac{0.625-0.5}{0.3}\approx0.4167,
\]

落在 \(A\) 的部分；再归一化得 \((0.4167-0)/0.5\approx0.833\)，落在 \(C\) 的 \([0.8,1)\) 内，于是恢复 \(BAC\)。

编码端和解码端只要有一次概率取整、符号顺序或更新状态不一致，之后所有边界都会错位。这是算术编码实现中最关键的同步要求。

\subsection{纸面实数与真实实现之间}

实际程序不会保存无限精度实数区间，而通常用固定宽度整数维护低端点和高端点。当二者共享某些最高位时，就输出这些已确定的位并重新缩放；还需处理区间落在中部导致的 underflow、进位传播和终止。

因此理论描述与工程实现之间有几条边界：

\begin{itemize}
\tightlist
\item
  浮点原型不能自动成为跨平台可复现的编码格式；
\item
  概率频数必须量化为双方一致的整数累计表；
\item
  需要明确 EOF 符号、消息长度或其他终止协议；
\item
  任何零概率事件都必须由模型逃逸机制或重新建模处理；
\item
  码长还包含头部、模型和状态初始化开销。
\end{itemize}

范围编码是同一思想的一种整数化实现；不同实现细节不改变``序列概率对应区间宽度''这条主线。
