压缩并不是简单地``给高概率符号短一些的字符串''。短码之间会争夺同一棵码树，编码必须能够被唯一拆开；逐符号编码还受整数码长限制，即使知道真实概率也可能离熵很远。只有把可译码性、构造方法与长序列的统计结构放在一起，才会看见熵为何成为压缩率的边界。

本单元依次解决六个问题：

\begin{enumerate}
\tightlist
\item
  什么样的变长码能够即时、唯一地译码？
\item
  在逐符号前缀码中，怎样把平均长度做到最小？
\item
  怎样摆脱``每个符号必须占整数 bit''的浪费？
\item
  为什么长序列最终只需要约 \(nH(X)\) bit？
\item
  允许失真时，压缩极限变成什么？
\item
  分布未知时，怎样边编码边学习重复模式？
\end{enumerate}

\subsection{怎么读这一章}

第一次读，可以先抓住一条主线：短码是一种有限资源，Kraft 不等式管资源够不够，Huffman 决定怎样分配，算术编码与长块编码则把逐符号的整数浪费摊薄。如果更关心实际压缩，可先读算术编码和 LZ；如果想知道``理论上还能压多短''，再读 AEP 与率失真。
