\section{二元对称信道：从翻转概率到容量}
\label{sec:07-Information-Theory/05-Channel-Coding/02}

\begin{quote}
\textbf{先抓住这一点：}随机翻转会混淆 0 和 1，但把许多比特一起编码，可以让不同消息在高维空间里离得很远。BSC 的容量 \(1-h_2(p)\) 说的是这种长块可靠通信的最高渐近速率，不是单次发送的正确概率。
\end{quote}

\subsection{重复发送为什么不是最终答案}

若每个比特以概率 \(p\) 独立翻转，最直接的纠错方法是把同一比特发送三遍并做多数表决。其译码错误发生在至少两次翻转时：

\[
P_e=3p^2(1-p)+p^3.
\]

当 \(p=0.1\) 时，错误率从 \(0.1\) 降到 \(0.028\)，但速率也从每次使用 1 bit 降到 \(1/3\) bit/use。继续重复能减少错误，却会把速率推向零。

冗余确实能减少错误，更关键的问题是：在错误概率趋近零的同时，速率最多能保持多高？二元对称信道让这个问题有一个可精确计算的答案。

\subsection{BSC 的模型}

二元对称信道 BSC\((p)\) 可写成

\[
Y=X\oplus Z,
\]

其中 \(Z\sim\operatorname{Bernoulli}(p)\) 与 \(X\) 独立，\(\oplus\) 表示模 2 加法。等价的转移矩阵是

\[
W=\begin{pmatrix}
1-p&p\\
p&1-p
\end{pmatrix}.
\]

``对称''指 0 翻成 1 与 1 翻成 0 的概率相同。给定 \(X\) 后，\(Y\) 的不确定性完全来自噪声比特 \(Z\)，所以

\[
H(Y\mid X)=H(Z)=h_2(p),
\]

其中 \(h_2(p)=-p\log_2p-(1-p)\log_2(1-p)\) 是二元熵函数。因而

\[
I(X;Y)=H(Y)-h_2(p).
\]

\subsection{先算任意输入，再优化}

令 \(q=P(X=1)\)。输出为 1 的概率是

\[
r=P(Y=1)=q(1-p)+(1-q)p=p+q(1-2p).
\]

于是

\[
I(X;Y)=h_2\bigl(p+q(1-2p)\bigr)-h_2(p).
\]

第二项由信道噪声决定，无法通过输入改变；要最大化互信息，只需让输出熵尽可能大。二元变量的熵至多为 1 bit，而 \(q=1/2\) 会让 \(r=1/2\)，达到这个上界。因此

\[
\boxed{C_{\mathrm{BSC}(p)}=1-h_2(p)\text{ bit/use}.}
\]

容量由公平输入达到。这个结论来自信道的对称结构：公平输入把输出空间用得最充分，与``公平输入最自然''的偏好无关。

\subsubsection{\texorpdfstring{\(p=0.1\) 时公式在说什么}{p=0.1 时公式在说什么}}

\[
h_2(0.1)\approx0.469,\qquad C\approx0.531\text{ bit/use}.
\]

它不表示单次发送有 \(53.1\%\) 的概率正确，也不表示存在长度很短、速率恰为 \(0.531\) 且零错误的码。它表示：对任意 \(R<0.531\)，存在越来越长的编码方案，使块错误概率趋于零；而可靠速率不能长期超过这个值。

三重重复码的速率 \(1/3\) 低于容量，但它没有充分利用块空间。更好的码会把许多消息映射到高维二元空间中相隔足够远的码字，让典型噪声图样仍能被区分；逐比特重复无法充分利用这种高维结构。

\subsection{三个边界值修正直觉}

当 \(p=0\) 时没有噪声，\(C=1\)。当 \(p=1/2\) 时，输出与输入独立，\(C=0\)。

当 \(p=1\) 时，每一位都确定翻转，接收者再翻转一次即可完全恢复，所以容量重新回到 1。一般地，BSC\((p)\) 与 BSC\((1-p)\) 只差一个确定的输出翻转，容量相同：

\[
C(p)=C(1-p).
\]

因此通常只研究 \(0\le p\le1/2\)。错误多不一定信息少；真正破坏信息的是输出不可预测地混淆输入。确定的反转是可逆变换，不损失信息。

\subsection{BSC 模型的适用边界}

BSC 适合独立、同概率的硬判决位翻转，但以下情况不能直接套 \(1-h_2(p)\)：

\begin{itemize}
\tightlist
\item
  0 到 1 与 1 到 0 的概率不同；
\item
  错误成串出现，具有时间相关性；
\item
  接收器保留了连续幅度，而不是只保留硬判决比特；
\item
  信道状态随时间变化或只有接收端知道；
\item
  编码器受到输入代价或组成约束。
\end{itemize}

把连续观测先硬判决为 0/1 会形成一个近似 BSC，但根据数据处理不等式，这一步通常丢失软信息，所得容量只是原信道可达能力的下界。
