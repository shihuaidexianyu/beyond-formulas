\section{高斯信道：功率约束与连续容量}

\subsection{连续输入若没有代价，问题就失去边界}

实加性白高斯噪声信道写成

\[
Y=X+Z,\qquad Z\sim\mathcal N(0,N),\qquad Z\perp X.
\]

如果允许发送者任意增大 \(X\) 的幅度，就能把不同码字拉得越来越远，相比固定噪声也越来越容易区分，容量没有有限上界。因此连续信道的容量公式必须连同输入约束一起陈述。

最常见的是平均功率约束

\[
\E[X^2]\le P.
\]

这里 \(P/N\) 是每个实维度的线性信噪比。遗漏约束后直接写容量，是把问题中最关键的物理条件删掉了。

\subsection{从互信息推导容量}

由于 \(Y=X+Z\) 且噪声独立，以微分熵 \(h\) 代替离散熵写出互信息：

\[
I(X;Y)=h(Y)-h(Y\mid X)=h(Y)-h(Z).
\]

噪声熵固定：

\[
h(Z)=\frac12\log_2(2\pi eN).
\]

由独立性，输出方差满足 \(\Var(Y)=\Var(X)+N\)；而方差不超过二阶矩，

\[
\Var(X)=\E[X^2]-(\E[X])^2\le\E[X^2]\le P,
\]

所以 \(\Var(Y)\le P+N\)，等号在输入零均值且功率用满时成立。给定方差的连续随机变量中，高斯分布具有最大微分熵，所以

\[
h(Y)\le\frac12\log_2\bigl(2\pi e(P+N)\bigr).
\]

相减得到

\[
I(X;Y)\le\frac12\log_2\left(1+\frac PN\right).
\]

取

\[
X\sim\mathcal N(0,P)
\]

时，\(Y\) 也是方差 \(P+N\) 的高斯变量，上界取等。因此实标量 AWGN 信道容量为

\[
\boxed{
C=\frac12\log_2(1+\mathrm{SNR})
\quad\text{bit/real use}.
}
\]

这个推导把``高斯分布最大化给定方差下的微分熵''从一个孤立定理变成了容量最优输入的原因。

\subsection{\texorpdfstring{为什么有时公式里没有 \(1/2\)}{为什么有时公式里没有 1/2}}

一个复基带符号包含两个实维度。若总功率与噪声归一化按复符号定义，常写成

\[
C=\log_2(1+\mathrm{SNR})
\quad\text{bit/complex use}.
\]

而连续时间带宽受限 AWGN 信道常写为

\[
C=B\log_2\left(1+\frac SN\right)
\quad\text{bit/s},
\]

其中 \(B\) 已经计入每秒可用自由度。三个公式描述的计量单位不同，不能只看有没有 \(1/2\)。每次使用、每个实维度、每个复符号和每秒，是四种必须明确的口径。

\subsection{数值例子与增长速度}

当 SNR 为 \(0\) dB 时，线性值为 1，

\[
C=\frac12\log_2 2=0.5\text{ bit/real use}.
\]

当 SNR 为 \(10\) dB 时，线性值为 10，

\[
C=\frac12\log_2 11\approx1.73\text{ bit/real use}.
\]

注意 dB 数值不能直接代入公式，必须先用 \(10^{\mathrm{dB}/10}\) 转为功率比。

高 SNR 下容量只随 \(\log\mathrm{SNR}\) 增长：要让高信噪比区间的容量再增加约 1 bit/real use，功率大致要乘 4。低 SNR 下

\[
\log_2(1+s)\approx\frac{s}{\ln2},
\]

容量对 SNR 近似线性。功率与带宽之间因此存在不同工作区间的权衡。

\subsection{高斯输入与实际星座}

连续高斯输入达到理论容量，但实际发送器常使用有限星座，如 BPSK、QPSK 或 QAM，并受到峰值幅度、功放非线性和量化精度限制。有限星座的输入熵至多为 \(\log_2M\)（\(M\) 为星座点数），高 SNR 时互信息会饱和，而高斯输入容量仍对数增长。

这不意味着实际系统``违背''容量公式，而是它解决了带不同输入约束的问题。若加入峰值幅度约束，容量达到分布一般不再是简单高斯；若噪声非高斯、存在衰落或信道未知，最优输入与容量表达式也会改变。

\subsection{容量仍然是渐近边界}

\(\frac12\log_2(1+P/N)\) 与离散信道容量具有同样的操作意义：低于它的速率可用足够长的码实现任意小错误，高于它则不能可靠通信。它不是未编码 QAM 的误码率公式，也不保证有限块长或特定调制能达到上界。

工程计算时至少要逐项核对：

\begin{itemize}
\tightlist
\item
  \(P,N\) 是否采用相同带宽和实/复归一化；
\item
  SNR 是线性值还是 dB；
\item
  约束是平均功率、峰值功率还是二者都有；
\item
  输出单位是 bit/real use、bit/complex use 还是 bit/s；
\item
  接收端是否保留软输出，还是先做了有损硬判决。
\end{itemize}

\subsection{最后一题：把单位说完整}

某实 AWGN 信道的 \(P/N=3\)。

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  求每个实使用的容量；
\item
  若每秒有 \(2B\) 个独立实自由度，写出 bit/s 容量；
\item
  若系统只允许 BPSK 输入，解释为什么高 SNR 下每个实使用的互信息不可能超过 1 bit，即使高斯输入容量可以超过 1。
\end{enumerate}
