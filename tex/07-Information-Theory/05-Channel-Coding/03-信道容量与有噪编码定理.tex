\section{信道容量与有噪编码定理}
\label{sec:07-Information-Theory/05-Channel-Coding/03}

\begin{quote}
\textbf{先抓住这一点：}容量是一条渐近分界线。速率低于它，可以用更长的码把整块消息的错误概率压得任意小；速率高于它，再聪明的编码也不能实现可靠通信。它不直接告诉我们某个有限长度的码表现怎样。
\end{quote}

\subsection{从单次互信息到长块可靠通信}

对离散无记忆信道 \(W(y\mid x)\)，容量定义为

\[
C=\max_{p_X}I(X;Y).
\]

这个公式看上去只是一个单次概率优化，Shannon 定理却赋予它操作意义：它恰好等于长块可靠通信的最高渐近速率。连接这两种说法，是信息论最重要的一次跨越。

一个 \((n,M)\) 块码包含：

\begin{itemize}
\tightlist
\item
  编码器，把消息 \(m\in\{1,\ldots,M\}\) 映射为长度 \(n\) 的信道输入 \(x^n(m)\)；
\item
  译码器，根据输出 \(y^n\) 猜测消息 \(\hat m\)；
\item
  速率
\end{itemize}

\[
R=\frac1n\log_2M\quad\text{bit/use};
\]

\begin{itemize}
\tightlist
\item
  平均块错误概率 \(P_e^{\mathrm{avg}}=P(\hat M\ne M)\)，这里通常令消息均匀分布；最大块错误概率则是 \(P_e^{\max}=\max_mP(\hat M\ne m\mid M=m)\)。
\end{itemize}

``可靠''不指每个信道符号都正确，而指存在一列块长 \(n\to\infty\) 的码，使整个消息的错误概率趋于零。

\subsection{可达性：随机码字为什么不会挤在一起}

固定一个输入分布 \(p_X\)。随机生成约 \(2^{nR}\) 个长度 \(n\) 的码字，每个坐标按 \(p_X\) 抽取。发送某个码字后，信道输出通常与它构成联合典型对。

译码器寻找唯一一个与输出联合典型的码字。错误主要有两类：

\begin{enumerate}
\tightlist
\item
  真正发送的码字与输出不典型；由 AEP，这个概率趋于零；
\item
  某个未发送码字碰巧也与输出典型。
\end{enumerate}

独立的错误码字与给定输出偶然匹配的概率，在指数阶上约为 \(2^{-nI(X;Y)}\)。共有约 \(2^{nR}\) 个竞争者，所以并合界给出总误匹配规模

\[
2^{nR}2^{-nI(X;Y)}=2^{-n(I(X;Y)-R)}.
\]

当

\[
R<I(X;Y)
\]

时，它也趋于零。再选择使互信息接近最大值的输入分布，就得到：对每个 \(R<C\)，至少存在一列码使块错误概率趋于零。

随机编码证明是存在性论证：随机码本的平均表现足够好，因此其中必有具体码本达到相应性能。它并不保证随手生成的一个短码一定好。

\subsection{不可能性：Fano 不等式把低错误变成信息需求}

反向假设消息 \(M\) 在 \(2^{nR}\) 个值上均匀，并且能从 \(Y^n\) 以很小的平均错误概率 \(P_e=P_e^{\mathrm{avg}}\) 恢复。Fano 不等式给出

\[
H(M\mid Y^n)\le h_2(P_e)+P_e\log_2(M-1).
\]

右端第一项不超过 1，第二项中 \(\log_2(M-1)<nR\)，所以 \(H(M\mid Y^n)\le1+P_e\,nR\)；只要 \(P_e\to0\)，这一项相对 \(n\) 就是 \(o(n)\)，即接收输出必须保留几乎全部消息熵：

\[
\begin{aligned}
nR=H(M)
&=I(M;Y^n)+H(M\mid Y^n)\\
&\le I(X^n;Y^n)+o(n).
\end{aligned}
\]

这里使用了 Markov 链 \(M\to X^n\to Y^n\) 的数据处理不等式。再展开 \(I(X^n;Y^n)\)：无记忆性给出 \(H(Y^n\mid X^n)=\sum_iH(Y_i\mid X_i)\)，结合熵的次可加性 \(H(Y^n)\le\sum_iH(Y_i)\)，

\[
I(X^n;Y^n)
=H(Y^n)-\sum_{i=1}^nH(Y_i\mid X_i)
\le\sum_{i=1}^n\bigl[H(Y_i)-H(Y_i\mid X_i)\bigr]
=\sum_{i=1}^nI(X_i;Y_i)
\le nC,
\]

最后一步是因为每个 \((X_i,Y_i)\) 都是同一信道 \(W\) 在某种输入分布下的单次使用，其互信息不超过 \(C\)。两边除以 \(n\) 并取极限，得到可靠速率不可能超过 \(C\)。

于是有限离散无记忆信道的有噪编码定理可概括为：\(R<C\) 时渐近可靠通信可达；\(R>C\) 时不可达。

标准有限 DMC 还有强逆结论：固定在容量以上的速率时，即使采用最优码，错误概率也会趋向 1，而不只是不能趋向零。

\subsection{容量是阈值，不是短码性能表}

基本定理没有回答：

\begin{itemize}
\tightlist
\item
  给定块长 \(n=1000\) 时最低错误率是多少；
\item
  容量边界 \(R=C\) 上是否能按某种方式达到目标；
\item
  编码和译码的计算复杂度多大；
\item
  延迟、峰值功率、误码地板和实现精度如何影响系统；
\item
  零错误通信能达到多高速率。
\end{itemize}

现代编码理论和有限块长信息论研究的正是这些问题。接近容量往往需要更长块、更多计算或更精细的软信息；Shannon 定理保证极限存在，却不替工程师构造有限资源下的最佳方案。

\subsection{输入约束必须进入容量定义}

若不同输入代价不同，不能在所有分布上无约束最大化。给定代价函数 \(c(x)\) 与预算 \(\Gamma\)，容量是

\[
C(\Gamma)=
\max_{p_X:\,\E[c(X)]\le\Gamma}I(X;Y).
\]

离散对称信道常由均匀输入达到容量，但一般信道未必如此。固定信道时，互信息关于 \(p_X\) 是凹函数（\(p_Y\) 与 \(H(Y\mid X)\) 都关于 \(p_X\) 线性，而熵是凹函数），有限字母表下可以用凸优化思想或 Blahut--Arimoto 算法求解。

\subsection{信源与信道如何接起来}

若要求无损恢复，一个离散无记忆信源每个样本含 \(H(S)\) bit，而每个信源样本可使用信道 \(\kappa\) 次，那么分离思想要求

\[
H(S)<\kappa C.
\]

在标准无记忆和长块条件下，可以先把信源压缩到接近熵，再用信道码以低于容量的速率可靠发送。若允许平均失真 \(D\)，相应比较应改成

\[
R(D)<\kappa C,
\]

而不是继续使用 \(H(S)\)。在这些经典单用户条件下，信源编码与信道编码可以分别设计而不损失渐近最优性。

但分离定理也有条件。短时延、多用户网络、联合语义目标或模型不匹配时，联合设计可能更合适。不要把单用户渐近定理扩张成所有通信问题的无条件原则。
