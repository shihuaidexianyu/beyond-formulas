\section{Jensen--Shannon 散度与尺度选择}
\label{sec:07-Information-Theory/03-Divergences/03}

\begin{quote}
\textbf{先抓住这一点：}把来自 \(P\) 和 \(Q\) 的样本混在一起，只看一个样本时，你有多大把握猜出它来自哪边？两个分布越容易分辨，JS 散度越大。因为双方地位相同，它天然是对称的。
\end{quote}

\subsection{当方向性成了任务的障碍}

假设要比较两个新闻语料库的词分布。某个专有名词只出现在第一个语料库，于是第二个分布在该词上给出零概率。直接计算任一方向的 KL，都可能因为支持不匹配而成为无穷——正是第一节强调的支持条件，在这里直接决定方法是否可用；而``两个语料库有多不同''听起来又是一个没有天然方向的问题。

一种常见但粗糙的修补是把双向 KL 平均：

\[
\frac12D(P\|Q)+\frac12D(Q\|P).
\]

它确实对称，却仍会在支持不重合时发散。真正的问题在于，两个分布仍直接把对方当作参考。Jensen--Shannon 散度换了一个思路：先构造双方都能覆盖的共同参照。

\subsection{先混合，再分别比较}

令

\[
M=\frac12(P+Q).
\]

等权 Jensen--Shannon 散度定义为

\[
\operatorname{JS}(P,Q)
=\frac12D(P\|M)+\frac12D(Q\|M).
\]

由于 \(M\ge P/2\) 且 \(M\ge Q/2\)，在相应支持上有密度比 \(dP/dM\le2\) 与 \(dQ/dM\le2\)。所以两个 KL 都不超过 \(\log 2\)，不再因双方支持不同而爆炸；这个论证也适用于连续分布，不依赖逐点概率。

定义立刻给出三个性质：

\begin{itemize}
\tightlist
\item
  对称：交换 \(P,Q\) 不改变 \(M\) 和结果；
\item
  非负：两个 KL 都非负；
\item
  有界：以 2 为对数底时，\(0\le\operatorname{JS}(P,Q)\le1\) bit。
\end{itemize}

等号左端在 \(P=Q\) 时取得；若 \(P,Q\) 支持完全不相交，则观察一个样本即可确定来源，达到右端的 1 bit。

\subsubsection{为什么上界恰好是一 bit}

引入一个公平来源标签 \(Z\in\{0,1\}\)：

\[
X\mid Z=0\sim P,\qquad X\mid Z=1\sim Q.
\]

忽略来源后，\(X\) 的边缘分布正是 \(M\)。把 \(I(Z;X)=D(P_{ZX}\|P_ZP_X)\) 中的联合概率拆成 \(P(z)P(x\mid z)\) 展开，公共因子 \(P(z)\) 在对数中消去，便得到互信息的条件 KL 表示：

\[
\begin{aligned}
I(Z;X)
&=\sum_zP(z)D(P_{X\mid Z=z}\|P_X)\\
&=\frac12D(P\|M)+\frac12D(Q\|M)\\
&=\operatorname{JS}(P,Q).
\end{aligned}
\]

因此 JS 的含义是：看到样本 \(X\) 后，我们获得了多少关于``它来自 \(P\) 还是 \(Q\)''的信息。来源标签本身只有 \(H(Z)=1\) bit，所以

\[
\operatorname{JS}(P,Q)=I(Z;X)\le H(Z)=1.
\]

这比记忆一个数值范围更有用。若两分布相同，样本对来源毫无帮助，JS 为零；若支持不交，样本完全暴露来源，JS 为一 bit。

\subsection{两个极端例子}

第一种情形是完全相同：\(P=Q\)。此时 \(M=P=Q\)，两个 KL 都为零。

第二种情形是

\[
P=(1,0),\qquad Q=(0,1).
\]

双向 KL 都是无穷，但 \(M=(1/2,1/2)\)，于是

\[
D(P\|M)=D(Q\|M)=1\text{ bit},
\]

从而 \(\operatorname{JS}(P,Q)=1\) bit——其中 \(D(P\|M)\) 正是第一节用 \(P=(1,0)\) 与 \((1/2,1/2)\) 算出的那 1 bit。这里的有限上界来自来源标签最多只携带一 bit 信息，并非数值计算技巧的产物。

\subsection{非等权混合揭示了更一般的结构}

若来源先验不是一半一半，令 \(P(Z=0)=\pi\)，混合分布为

\[
M_\pi=\pi P+(1-\pi)Q.
\]

相应的加权 JS 为

\[
\operatorname{JS}_\pi(P,Q)
=\pi D(P\|M_\pi)+(1-\pi)D(Q\|M_\pi)
=I(Z;X).
\]

此时上界变成来源标签的二元熵 \(h_2(\pi)\)，不再固定为一 bit。这再次说明：散度数值只有连同权重、对数底和采样机制一起说明才有完整含义。

\subsection{JS 也不是万能距离}

JS 本身一般不满足三角不等式，所以严格说仍是散度；但 \(\sqrt{\operatorname{JS}}\) 是一个度量——这是已证明的定理，此处只引用结论。实际工作中还会遇到总变差距离、Hellinger 距离、Wasserstein 距离以及更一般的 \(f\)-散度。这些量以不同方式观察误差，选择时应根据任务，不存在``谁更高级''的统一排行。

选择比较量前，可以依次问：

\begin{enumerate}
\tightlist
\item
  是否存在明确的真实分布与近似分布？若有，KL 的方向可能正是问题的一部分；
\item
  支持是否可能不重合？若会，直接 KL 可能成为无穷；
\item
  是否需要对称性、有限上界或三角不等式？
\item
  我们关心的是编码冗余、分类可辨性、概率质量差，还是样本空间中的搬运成本？
\end{enumerate}

例如，比较概率预测与真实生成机制时，\(D(P\|Q)\) 有清楚的负对数损失解释；比较两个地位相同的数据集时，JS 更自然；若两个图像分布只发生轻微空间平移，Wasserstein 距离可能更能反映几何邻近性。脱离任务说``某个散度最好''，通常没有意义。

\subsection{估计散度时的现实困难}

公式假设我们知道 \(P,Q\)，但数据分析中常只有有限样本。直接把频率代入可能产生许多零概率，使 KL 极不稳定；随意加平滑又会改变目标分布。高维连续情形还要面对密度估计误差。

因此报告一个经验散度时，至少要说明：

\begin{itemize}
\tightlist
\item
  分布是已知模型、直方图还是密度估计得到的；
\item
  如何处理未观察类别与零概率；
\item
  使用哪个方向、哪个对数底以及何种混合权重；
\item
  数值是否带有抽样不确定性。
\end{itemize}

数学定义给出比较尺度，但不会替我们解决统计估计问题。这个边界在后面的数理统计部分会再次出现。
